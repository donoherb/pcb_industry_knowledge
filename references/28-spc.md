# SPC与过程能力

来源更新：2026-04-17。本文沉淀统计过程控制（SPC）和过程能力的稳定知识、PCB/载板场景化方法与常见误区，不替代AIAG SPC手册、客户CSR、内部程序文件和已批准的控制计划。凡涉及客户门槛、特殊特性、控制图规则、能力指数要求和提交口径，必须以客户文件与当前有效标准为准。

## 一句话理解

SPC回答的问题不是“产品有没有超规格”，而是“过程是否处于统计受控状态，波动是否可预测，以及这种波动是否足以长期稳定满足客户要求”。

## 与APQP、MSA、Control Plan、PPAP、8D的接口

SPC不是单独的一张控制图，而是六大质量工具中承接“量产过程稳定性和能力证明”的那一段。没有可信的测量系统，SPC无法成立；没有明确的特殊特性和反应计划，SPC也只是记录数据。

| 工具/文件 | 与SPC的关系 | PCB/载板落地重点 |
| --- | --- | --- |
| APQP | 决定哪些特性需要在样件、试产和量产阶段被持续监控 | 从VOC、QFD、特殊特性和风险项进入SPC计划 |
| PFMEA | 决定为什么监控、监控什么、失控后怎么反应 | 将高风险过程参数和产品特性转成SPC项目 |
| Control Plan | 决定抽样频率、方法、反应计划和责任人 | 控制图类型、抽样单位、升级路径要写入Control Plan |
| MSA | 决定SPC数据是否可信 | 分辨率、GRR、稳定性、边界判定能力要先成立 |
| PPAP/Run at Rate | 用SPC和能力研究证明真实量产条件下过程稳定 | 试生产数据来源、批次、设备和人员必须可追溯 |
| 8D/ECN | 决定SPC异常和客户问题是否真正回流到系统 | 异常处置后要更新PFMEA、Control Plan、SOP和SPC规则 |

典型链路：

`客户要求/特殊特性 -> 过程流程图/PFMEA -> Control Plan -> MSA -> SPC判稳 -> 能力研究 -> PPAP/量产控制 -> 异常反应/8D/ECN -> 回写FMEA和控制计划`

## SPC的核心边界

做SPC前先把以下概念分开：

| 概念 | 回答的问题 | 常见误用 |
| --- | --- | --- |
| 合格/不合格 | 当前样本是否满足规格 | 用单点合格替代过程稳定 |
| 统计受控 | 波动是否只来自普通原因 | 过程失控时仍计算Cpk |
| 过程能力 | 在稳定状态下过程波动是否足够小 | 把短期好数据当长期能力 |
| 设备能力 | 设备在短时间、受控条件下能做到什么 | 用Cm/Cmk替代整条产线能力 |
| 容量/节拍 | 产出速度是否够 | 产量达成不代表质量稳定 |

两个高频误区：

- 过程可能“在规格内但不稳定”。这时仍有特殊原因，后续随时可能跑偏。
- 过程可能“稳定但能力不足”。这时过程很规律，但规律地做出接近或超出规格边界的产品。

## 什么时候该上SPC

PCB/载板项目通常满足以下三点时适合上SPC：

1. 特性影响客户功能、可靠性、装配或法规要求。
2. 数据可以重复、稳定地测量，且MSA可信。
3. 过程存在连续采样和改进价值，而不是完全依赖终检筛选。

典型适用对象：

- 关键产品特性：线宽线距、孔径、孔铜厚、面铜厚、阻抗、板厚、介质厚度、镀层厚度、翘曲、关键尺寸。
- 关键过程参数：电流密度、药水浓度、温度、喷压、蚀刻速率、压合曲线、激光参数、烘烤条件。
- 关键属性指标：AOI/AVI漏判率、误判率、缺陷率、电测不良率、外观不良率、批次不良率。

不宜机械套用SPC的场景：

- 量极少、频率极低、每次样本结构都不同的项目。
- 测量系统不稳定或边界判定规则不清。
- 混合多种产品族、材料和规格，导致数据不具备同质性。
- 破坏性测试但没有coupon、分层和取样设计支撑。

## 数据分层原则

SPC最怕把不同来源的波动混成一张图。对PCB/载板过程，至少要考虑以下分层：

| 分层维度 | 为什么重要 | 常见例子 |
| --- | --- | --- |
| 产品/料号 | 规格不同，目标值和波动结构不同 | 普通多层板和高速板阻抗不能混图 |
| 工厂/产线/设备 | 设备状态和能力不同 | 不同电镀线、钻机、LDI机台分别建图 |
| 材料批次/供应商 | 材料波动会改变过程中心和宽度 | CCL、ABF、铜箔、药水批次 |
| 班次/人员 | 换班、调机和操作差异可能带来特殊原因 | 夜班电镀厚度偏移、复判标准不一致 |
| 工艺阶段 | 样件、试产、量产、Safe Launch不可混用 | 爬坡初期数据不应用于成熟量产能力 |
| 检测方法/程序版本 | 测量系统变化会造成假趋势 | AOI程序改版、XRF程序更新 |

如果无法保证同质性，宁可拆图，也不要为了图表好看把不同数据硬拼在一起。

## 数据类型与控制图选择

控制图不是越复杂越好，而是要匹配数据类型、抽样方式和现场决策需求。

| 数据类型 | 常用控制图 | 适用条件 | PCB/载板例子 |
| --- | --- | --- | --- |
| 连续型，合理子组 n=2-10 | Xbar-R | 同一时段内可形成短期同质子组 | 每小时抽5个panel的线宽、镀层厚度 |
| 连续型，合理子组 n>=10 | Xbar-S | 子组较大且关注组内标准差 | 大批量厚度或关键尺寸监控 |
| 连续型，单件或低频 | I-MR | 每次只有1个值或破坏性/高混低量场景 | 阻抗coupon、微切片孔铜、回流翘曲 |
| 属性型，不良品数 | p图 | 每批样本量可变，统计不良品比例 | 每班外观不良率、电测不良率 |
| 属性型，不良品数 | np图 | 每批样本量固定 | 每批固定抽样的外观不良件数 |
| 属性型，缺陷数 | c图 | 单位机会固定，统计缺陷个数 | 固定面积panel上的AOI缺陷数 |
| 属性型，单位缺陷率 | u图 | 单位机会可变，统计每单位缺陷数 | 不同面积panel的缺陷密度 |
| 小漂移敏感监控 | EWMA/CUSUM | 需要更早发现缓慢偏移 | 药水化学参数、阻抗中心漂移 |

选择要点：

- 不要用连续型控制图去画纯属性判断数据。
- 不要把不同面积、不同机会数的缺陷数直接放在同一张 `c图` 上。
- 对破坏性或高成本测试，`I-MR` 往往比强行凑 `Xbar-R` 更真实。
- `EWMA/CUSUM` 适合早期预警，但客户若要求传统图型，不能私自替代。

## 合理子组与采样设计

合理子组的目的，是让“组内波动主要反映短期普通原因，组间波动更容易暴露特殊原因”。

| 场景 | 子组思路 | PCB/载板例子 |
| --- | --- | --- |
| 稳定批量生产 | 同机台、同材料、短时间内连续取样 | 电镀线每小时抽5点面铜厚 |
| 班次切换敏感 | 子组内保持同班次，班次之间分开建图或加分层 | 日夜班分别统计阻焊偏位 |
| 破坏性检测 | 单值监控加移动极差，辅以coupon或配对样 | 微切片孔铜厚、剥离强度 |
| 高混低量 | 同族产品归并，必要时用短程SPC或目标归一化 | 相同叠构和线宽族的阻抗数据 |
| 化学连续过程 | 按时间间隔采样，结合过程参数和设备状态 | 铜离子、酸度、温度、ORP趋势 |

抽样时至少明确：

- 抽样单位：件、panel、lot、strip、coupon还是测试点。
- 抽样频率：首件、每小时、每批、每班、每料号切换后。
- 测点规则：位置、数量、方向、是否固定点位。
- 异常反应：停机、加严抽样、隔离、调机、复测、升级。

## 判稳先于算能力

能力指数建立在“过程已稳定”的前提上。没有判稳，Cp/Cpk/Pp/Ppk都容易误导。

典型顺序：

1. 先确认MSA有效。
2. 再用控制图判断过程是否存在特殊原因。
3. 仅在过程稳定、数据同质的前提下计算能力指数。
4. 对不稳定过程先找特殊原因，不要急着汇报Cpk。

常见失控信号包括：

- 点超出控制限。
- 连续多点落在中心线同侧。
- 连续上升或下降。
- 周期性波动、锯齿波动或交替波动。
- 多个点持续靠近控制限。

这些信号即使单点仍在规格内，也说明过程需要分析。

## 常用能力指数

能力指数的意义，取决于时间跨度、样本来源和是否已判稳。

| 指标 | 典型用途 | 关键前提 | 常见误用 |
| --- | --- | --- | --- |
| Cp | 看过程分布宽度相对规格宽度是否足够 | 过程稳定且近似对称 | 不看中心偏移，只报Cp |
| Cpk | 同时看分布宽度和中心偏移 | 过程稳定、规格明确 | 失控过程仍报Cpk |
| Pp | 看总体长期表现相对规格 | 数据覆盖较长周期且口径一致 | 用短期数据算长期Pp |
| Ppk | 看长期整体表现和中心偏移 | 数据包含长期波动 | 与Cpk混用但不解释差异 |
| Cm/Cmk | 设备短期能力 | 短时间、单设备、受控试验条件 | 用来证明整条制程量产能力 |

简单理解：

- `Cp/Cpk` 更偏短期、稳定状态下的过程能力。
- `Pp/Ppk` 更偏长期、实际经营状态下的总体表现。
- `Cpk` 明显高于 `Ppk`，通常说明长期波动、换班、换料、换机或管理波动较大。

## 公式与解释

学习时可用以下口径理解：

| 指标 | 常见表达式 | 解释 |
| --- | --- | --- |
| `Cp` | `(USL - LSL) / 6σ` | 只看过程波动宽度是否塞得进规格带 |
| `Cpk` | `min[(USL - μ)/(3σ), (μ - LSL)/(3σ)]` | 同时看中心是否偏移 |
| `Pp` | `(USL - LSL) / 6s_total` | 用总体长期波动计算宽度 |
| `Ppk` | `min[(USL - μ)/(3s_total), (μ - LSL)/(3s_total)]` | 用长期整体表现看能力 |

但公式只是结果表达。更重要的是确认：

- `σ` 和 `s_total` 的取法是否符合公司程序和客户要求。
- 规格是双边、单边还是目标值加容差。
- 数据是否近似同分布，是否混入不同产品族和不同阶段。
- 是否存在非正态、截尾、分层和强自相关问题。

## 接收准则与风险分层

通用门槛只能作为训练口径，不能机械替代客户CSR和项目风险判断。

| 场景 | 判定重点 | 处理倾向 |
| --- | --- | --- |
| 安全/法规/客户特殊特性 | 判稳、中心偏移、边界风险、长期趋势 | 不宜仅因短期Cpk好看就放行 |
| PPAP/Run at Rate | 数据来源是否真实量产、是否判稳、是否满足客户门槛 | 需要版本、批次、设备和样本来源可追溯 |
| 量产控制 | 控制图是否及时发现异常，反应计划是否执行 | 比“月报Cpk”更重现场反应 |
| 改善项目 | 关注波动收敛、中心拉回和异常减少 | 要看改善前后同口径对比 |
| 客户争议 | 长期表现、边界判定、测量系统和分层证据 | 不能只拿单批最好数据回应客户 |

常见学习口径里，客户可能会要求某些关键特性达到 `Cpk >= 1.33` 或更高，但这不是普适法律。正式应用时必须写明：门槛来自客户CSR、图纸、质量协议还是内部规则。

## PCB/载板常见SPC项目

不是每个项目都要上SPC。优先选择“对客户重要、对过程敏感、可稳定测量、能推动改善”的项目。

| 类别 | 常见SPC项目 | 关注重点 |
| --- | --- | --- |
| 图形 | 线宽、线距、蚀刻补偿、残铜率 | 细线路产品要防中心漂移和尾部风险 |
| 孔与互连 | 孔径、孔位、孔铜厚、填孔凹陷、背钻残桩 | 需结合切片、coupon、设备参数和可靠性 |
| 压合与结构 | 板厚、介质厚度、层偏、翘曲 | 需按叠构、铜分布、材料批次分层 |
| 表面处理 | ENIG/ENEPIG厚度、沉银/沉锡指标、可焊性相关参数 | 注意标准片、XRF程序和批次差异 |
| 电气 | 阻抗、导通电阻、电测不良率 | 阻抗需同步看材料、线宽和压合 |
| 外观与属性 | AOI/AVI缺陷率、漏判率、误判率、外观不良率 | 属性图要考虑面积、机会数和限度样口径 |
| 化学过程 | 铜离子、酸度、温度、比重、ORP、喷压、蚀刻速率 | 参数SPC要与产品结果联动，而非孤立监控 |
| 封装相关 | 载板warpage、共面性、strip缺陷密度、焊盘尺寸 | 需兼顾封装客户来料判定和热态条件 |

## PCB/载板典型场景

| 场景 | 常见控制图/方法 | 重点 | 典型风险 |
| --- | --- | --- | --- |
| 外层线宽 | Xbar-R 或 I-MR | 线宽中心、波动、设备/药水状态 | 换膜、蚀刻补偿、残铜、测量程序变化 |
| 孔铜厚 | I-MR 或分层Xbar-R | 破坏性取样、coupon代表性、批间波动 | 切片制样差异、电镀电流分布、药水失控 |
| 阻抗 | I-MR、Xbar-R | 测试coupon、材料批次、中心偏移 | TDR偏倚、叠构变化、线宽压合联动 |
| 板厚/介质厚度 | Xbar-R | 压合窗口、材料和铜平衡 | 测点不一致、不同叠构混图 |
| ENIG/ENEPIG厚度 | Xbar-R、I-MR | 标准片、XRF程序、批次趋势 | 标准件不匹配、模型错误、点位偏差 |
| AOI缺陷率 | p图、u图 | 缺陷密度、真假点趋势、程序稳定性 | 面积不同、算法改版、复判标准漂移 |
| 电测不良率 | p图 | 开短路趋势、批次和设备关联 | 混合产品、治具老化、程序版本变更 |
| 电镀化学参数 | I-MR、EWMA | 早期发现缓慢漂移 | 采样周期过长、自相关、补加记录不完整 |
| 回流后翘曲/共面性 | I-MR | 热态条件一致性、样本吸湿状态 | 测试曲线不同、支撑方式不同、样本前处理不一致 |

## 特殊过程参数SPC

PCB和载板不是只有产品尺寸能做SPC，很多失效的早期信号藏在过程参数里。

| 工序 | 可监控的关键参数 | 目的 |
| --- | --- | --- |
| 蚀刻 | 温度、喷压、铜离子、蚀刻速率、补偿值 | 稳住线宽线距和侧蚀 |
| 沉铜/电镀 | 电流密度、槽液成分、温度、时间、添加剂、搅拌 | 稳住孔铜、面铜和填孔质量 |
| 压合 | 升温速率、峰值温度、保压时间、真空、冷却曲线 | 稳住板厚、介质厚、层偏、翘曲 |
| 激光钻孔 | 能量、频率、焦距、孔形监控 | 稳住微孔孔径、残胶和孔底质量 |
| 阻焊 | 曝光能量、显影条件、固化曲线 | 稳住开窗、桥连和附着 |
| 表面处理 | 药水浓度、温度、时间、负载量 | 稳住镀层厚度、表面状态和可焊性 |

参数SPC不是越多越好。应优先选那些对客户结果有明确因果路径、且现场能及时调整的参数。

## 属性数据的SPC

很多PCB/载板现场只会给连续数据做SPC，但属性数据同样关键。

适合属性图的项目：

- AOI/AVI缺陷率。
- 外观不良率。
- 电测不良率。
- 客诉PPM或批次不良率。
- 漏判率、误判率、复判差异率。

注意点：

- `p图` 看的是不良品比例，不是缺陷个数。
- `u图` 更适合不同面积、不同panel机会数下的缺陷密度。
- 属性数据要先确认判定标准、限度样、复判规则和属性MSA。
- 仅统计“不良数量”而不分缺陷类型，容易掩盖主因迁移。

## 非正态、单边规格和自相关

实际现场常常不是教科书里的理想正态数据。遇到以下情况要特别谨慎：

| 情况 | 风险 | 处理建议 |
| --- | --- | --- |
| 单边规格 | 普通双边Cpk解释会失真 | 明确只看上限或下限方向的能力 |
| 非正态分布 | 传统能力指数可能误判尾部风险 | 先做分层、变换或采用合适方法，并说明假设 |
| 截尾数据 | 量具分辨率或筛选逻辑造成分布变形 | 先确认测量系统和抽样口径 |
| 强自相关 | 连续化学参数相邻点高度相关，控制限会被误读 | 调整采样间隔或采用EWMA/CUSUM等方法 |
| 多峰分布 | 混入多产品、多机台或多批次 | 先拆层，不要直接求Cpk |

SPC做得扎实的标志，不是公式更复杂，而是知道哪些数据不能直接套公式。

## 量产爬坡与成熟量产的差异

同一个特性，在试产和成熟量产阶段的SPC目标不一样。

| 阶段 | 主要目标 | 重点看什么 |
| --- | --- | --- |
| 样件/工程试验 | 识别波动来源和失效机理 | 异常类型、中心位置、测量可行性 |
| 试生产/Run at Rate | 证明过程在真实量产条件下可判稳 | 控制图异常、换班换料影响、小时产出 |
| 量产爬坡 | 降低特殊原因、稳定班次和设备差异 | 异常频次、Cpk/Ppk趋势、返工和报废 |
| 成熟量产 | 维持控制并持续优化成本和风险 | 长期趋势、中心漂移、客户PPM和COPQ |

不要用工程样或小批调机后的“最好一段数据”去代表成熟量产能力。

## 高混低量环境下的SPC

PCB和载板很多工厂天然是高混低量，若照搬汽车零件的大批量SPC，往往只会制造无效报表。

可行做法：

- 按产品族、叠构、材料体系、目标值相近的特性分层。
- 对低频、破坏性和高成本项目优先用 `I-MR`、短程SPC或阶段性能力研究。
- 把SPC和首件、换线确认、程序版本管理、Set-up审批结合起来。
- 不强求每个料号都单独积累大样本后再行动，必要时用家族化规则和工程判断先控风险。

高混低量不等于不做SPC，而是更强调分层、边界定义和反应速度。

## 现场反应计划

SPC的价值不在于图，而在于失控后现场怎么做。一个可执行的反应计划至少应回答：

| 问题 | 应明确什么 |
| --- | --- |
| 谁负责 | 操作员、线长、QE、PE、实验室、主管谁先行动 |
| 何时停 | 超控制限、连续趋势、关键特性偏移到什么程度必须停机或隔离 |
| 隔离什么 | 在制品、同批料、同机台、同班次、前后批次范围 |
| 查什么 | 设备、材料、程序、治具、药水、环境、测量系统 |
| 如何恢复 | 调机、复测、小批确认、首件批准、客户沟通 |
| 如何回写 | PFMEA、Control Plan、SOP、培训、经验教训是否同步更新 |

关键原则：

- 失控不等于产品已全部不合格，但必须先评估影响范围。
- 超规格不等于一定能靠加严筛选彻底解决，仍要找发生原因。
- 同类异常重复发生时，说明SPC不是缺图，而是系统闭环失效。

## SPC异常的基本闭环

建议用以下顺序处理：

1. 发现异常：控制图判异、趋势异常、中心偏移、能力下降。
2. 暂停放行判断并隔离可疑批次，尤其是关键特性。
3. 复核测量系统和数据完整性，排除量具、程序、录入和抽样错误。
4. 分层分析设备、班次、材料、程序、工装和时间窗口。
5. 采取遏制和纠正措施，并用小批或加强抽样验证。
6. 恢复前确认过程重新受控，不要只看某几个点回到规格内。
7. 更新PFMEA、Control Plan、SOP、培训、经验教训和必要的客户沟通记录。

## 与PPAP、Run at Rate和客户审核的关系

客户在PPAP或审核里看SPC，重点通常不是“你有没有画图”，而是以下问题：

- 选的特性是不是客户真正关心的CTQ/特殊特性。
- 控制图、能力研究、抽样规则和MSA是否相互一致。
- 数据是不是来自量产设备、量产材料、量产人员和真实节拍。
- 失控时有没有明确反应计划并保留执行证据。
- 能力不足时，是不是有客户批准的过渡计划和改善路径。

常见证据包括：

- 控制计划中的SPC项目清单。
- 控制图和原始数据。
- 过程能力研究报告。
- 异常处置记录和升级记录。
- MSA报告、标准件和程序版本记录。
- Run at Rate期间的小时产出、异常和恢复证据。

## SPC报表与审核证据建议

一个可审计的SPC包通常包括：

- 特性定义：规格、目标值、客户等级、特殊特性编号。
- 控制图定义：图型、抽样单位、样本量、频率、测点规则。
- 数据来源：设备、程序、工装、测量系统、批次和时间戳。
- 判异规则：采用哪套规则、谁复核、何时升级。
- 反应计划：停机、隔离、复测、调机、再验证和放行条件。
- 能力分析：窗口、样本量、分层逻辑、Cp/Cpk/Pp/Ppk结论。
- 变更记录：产品、程序、量具、设备和材料变化后的重新确认。

如果控制图不能追溯到同一批次的原始测量值、设备状态和反应记录，这张图对审核和客户争议的价值就很有限。

## PCB/载板场景下的常见错误

- 没有先做MSA，就用测量值画图和算Cpk。
- 把不同料号、不同叠构、不同材料的数据混在一张图里。
- 过程失控时继续报能力指数，且不解释特殊原因。
- 只看月度Cpk，不看日常判异和反应计划执行。
- 只画产品结果图，不监控关键过程参数。
- 用属性数据硬套连续型控制图，或把不同面积panel的缺陷数直接比较。
- 破坏性检测取样过少，却把偶然好数据当成稳定能力。
- AOI/AVI程序改版后不重新确认，导致趋势断裂。
- 只在客户审核前补图，现场并未据此调机、隔离和升级。
- 客诉后只做加严筛选，不更新PFMEA、Control Plan和SPC规则。
- 只看最终出货良率，不看FPY、返工率、缺陷密度和Cpk/Ppk趋势。
- 将 `Cpk >= 某数值` 当成唯一目标，忽略中心偏移、长期漂移和客户实际风险。

## 快速落地流程

`从特殊特性和高风险过程出发 -> 确认MSA -> 选控制图类型 -> 定义分层和抽样 -> 建立原始数据与反应计划 -> 先判稳再算能力 -> 现场执行和异常闭环 -> 与PPAP/审核/经验教训联动`

使用建议：

- 做Control Plan、过程能力、客户审核或PPAP资料时，先用本页明确“哪些特性值得做SPC、用什么图、怎样判稳、何时算能力、异常后如何反应”。
- 做测量系统和判定争议时，先回到 `27-msa.md` 复核数据可信度，再解释SPC异常和Cpk变化。
- 做APQP和量产导入时，结合 `26-apqp.md` 把SPC放进第三到第五阶段的证据链，不要等客户要PPAP时才补。
- 做PCB/载板制程问题分析时，结合 `04-process-quality.md` 的工艺、缺陷和可靠性内容，避免把统计图和物理机理分开看。
- 做AOI、ET、FQC缺陷位置、报废位置、缺陷密度和检测站点流出分析时，结合 `36-defect-scrap-mapping-aoi-et-fqc.md`。
