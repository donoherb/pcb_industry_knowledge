本文沉淀PCB/载板行业缺陷mapping、报废mapping，以及AOI/AVI、ET电测、FQC/终检等检验结果在panel、pcs、strip或unit位置图上的统一呈现方法。不替代企业MES/QMS/SPC系统、客户检验规范、IPC验收标准、设备原始数据格式、报废审批规则和客户指定map格式。涉及客户提交、封装来料map、报废财务确认和系统接口时，应以客户文件、系统主数据和正式授权流程为准。

## 一句话理解

缺陷/报废mapping不是把不良点画在图上，而是把AOI、ET、FQC、复判、MRB、返工和报废结果统一到同一套产品位置坐标和状态口径中，使现场能看清“缺陷在哪里、由哪道工序发现、是否重复、是否流出、是否已处置、对良率和报废的贡献是什么”。

## 适用范围

本专题适用于：

- 内层/外层AOI、AVI、X-ray、FQC外观、FQA/OQA抽检等视觉/属性检验结果。
- ET电测开路、短路、绝缘、导通不稳、四线测试或客户指定电性测试结果。
- FQC/终检发现的外观、尺寸、阻焊、字符、表面处理、包装和标签异常。
- panel级、pcs级、strip级、unit级、coupon级缺陷和报废位置追溯。
- MRB、返工、让步、报废、客诉8D、DMAIC和SPC缺陷密度分析。
- 载板strip map、封装客户来料map、IC substrate unit good/bad map。

## 核心对象

| 对象 | 含义 | mapping重点 |
| --- | --- | --- |
| Lot | 同一批次或工单范围 | 批次良率、材料/设备/班次关联 |
| Panel | 生产panel | panel内坐标、边/中/角差异、设备夹点或流场影响 |
| Pcs | panel内单片或客户单件 | 单片报废、返工和FQC结果 |
| Strip | IC载板或封装条 | strip ID、unit map、封装客户判定 |
| Unit | strip/pcs内单元 | 单元好坏、位置、缺陷类型和客户接收状态 |
| Defect point | 单个缺陷点 | X/Y、层别、面别、工序、缺陷码和图片证据 |
| Scrap unit | 报废单元或报废片 | 报废原因、审批、财务数量和防误用状态 |

## Mapping层级

| 层级 | 适合场景 | 典型输出 |
| --- | --- | --- |
| Lot mapping | 找批次、材料、班次和设备差异 | 批次良率、报废Pareto、趋势图 |
| Panel mapping | 找panel内位置规律 | heat map、边缘/中心/夹点/流向分布 |
| Pcs mapping | 判断单片处置和返工路径 | 单片NG点、返工状态、FQC最终判定 |
| Strip/unit mapping | 载板和封装接口 | good/bad unit map、strip map、客户来料map |
| Defect type mapping | 找主缺陷和迁移 | 缺陷码分布、工序来源、重复缺陷 |
| Detection station mapping | 找流出和检测有效性 | AOI vs ET vs FQC发现差异、漏检/过杀 |

不要只做lot级统计。很多系统性问题只有放到panel或unit位置图上才会显现，例如固定夹点、喷淋死角、曝光对位边缘异常、探针阵列局部失效。

## 数据来源

| 来源 | 常见字段 | 风险点 |
| --- | --- | --- |
| AOI/AVI | 设备、程序、layer/side、缺陷坐标、缺陷码、图片、复判结果 | 过杀、缺陷码不统一、坐标方向与panel不一致 |
| ET电测 | 设备、治具/飞针程序、net、pin、开短路类型、位置、复测结果 | netlist版本、治具磨损、误判、位置转换困难 |
| FQC/终检 | 检验员、工位、缺陷类型、位置、数量、判定和图片 | 手工记录粒度不足、位置描述不标准 |
| MRB/NCM | 处置结论、返工、让步、报废、审批和原因 | 只记录数量，没有位置和原始缺陷 |
| MES/QMS | 工单、lot、panel、pcs、strip、流程和状态 | 主数据版本、拆panel后追溯断裂 |
| SPC/良率系统 | FPY、缺陷率、报废率、返工率、趋势 | 缺少位置维度，无法定位物理原因 |

Mapping要保留原始来源。整合后的map不能替代AOI图片、ET原始测试记录、FQC判定记录和MRB审批记录。

## 坐标和版面规则

缺陷mapping最容易出错的是坐标系统。必须提前定义：

- 原点：左下、左上、设备原点、客户原点还是CAM原点。
- 方向：生产流向、设备X/Y、客户图纸X/Y、上下板面翻转关系。
- 单位：mm、mil、pixel、row/column、unit index。
- 层别和面别：L1/L2、top/bottom、inner/outer、阻焊/字符/表面处理。
- panel布局：pcs编号、旋转角度、镜像、拼板方向、工艺边和coupon位置。
- strip/unit布局：row/column、unit ID、map方向和客户读图方向。
- 拆分规则：panel拆成pcs后如何保持原panel位置和坐标。

任何跨设备整合前，先用黄金样、标准点或已知缺陷板确认坐标转换，不要直接把多个设备导出数据叠加。

## 缺陷编码

缺陷码要同时服务现场处置、工程分析和数据统计。

建议字段：

| 字段 | 说明 |
| --- | --- |
| 大类 | 开路、短路、残铜、缺口、异物、划伤、阻焊、镀层、尺寸、标签 |
| 小类 | 例如细线路缺口、焊盘残铜、阻焊偏位、金面划伤、字符错印 |
| 发现工序 | AOI、ET、FQC、客户来料、SMT/封装、可靠性 |
| 责任工序 | 图形、蚀刻、钻孔、电镀、阻焊、表面处理、成型、检测 |
| 判定状态 | 真不良、过杀、待判、复判合格、漏检、客户争议 |
| 严重度 | 致命、重大、轻微或客户指定等级 |
| 处置 | 返工、返修、让步、报废、继续加工、客户确认 |

AOI、ET和FQC各自的缺陷码可能不同，mapping系统需要一张统一码表，避免同一缺陷被统计成多个名称。

## AOI Mapping

AOI/AVI mapping关注三件事：

1. 缺陷位置：缺陷是否集中在固定边、固定层、固定曝光区域、固定panel位置。
2. 缺陷类型：真不良、过杀、待判是否分开统计。
3. 检测能力：程序改版、复判标准、限度样和人员一致性是否稳定。

常见AOI map看法：

- 固定坐标重复：优先查菲林/LDI资料、曝光、CAM补偿或设备污染。
- panel边缘集中：优先查夹具、喷淋、蚀刻均匀性、搬运和板边保护。
- 某层集中：优先查该层材料、曝光、显影、蚀刻或层别程序。
- 过杀集中：优先查AOI阈值、限度样、复判训练和程序版本。
- 真不良与过杀混在一起：先做复判分层，否则SPC和报废判断会失真。

## ET Mapping

ET mapping要把电性结果尽量转成物理位置和网络关系：

| ET结果 | Mapping关注 |
| --- | --- |
| 开路 | net、断点可能区域、孔/线/焊盘位置、AOI是否曾报警 |
| 短路 | 相邻net、残铜/异物可能位置、AOI/外观证据 |
| 间歇导通 | 探针接触、孔铜裂纹、污染、治具状态、复测稳定性 |
| 绝缘异常 | 污染、CAF风险、细间距区域、湿热/清洁度 |
| 治具误判 | 固定针位、固定区域、复测差异、黄金板验证 |

ET与AOI的坐标不一定天然一致。开短路只有net信息时，需要结合CAM netlist、焊盘/孔位置、AOI图片和切片/FA定位，不能简单把ET不良点当成可见缺陷点。

## FQC Mapping

FQC mapping用于补上最终外观和客户接收视角：

- 将FQC缺陷记录到pcs/panel/strip位置，而不是只写“外观不良1pcs”。
- 区分制造缺陷、搬运损伤、包装损伤、标签错误和前工序漏出。
- 对可拍照缺陷保留图片、位置、面别、缺陷码和判定标准。
- 对人为判定类缺陷做属性一致性和限度样管理。
- 将FQC发现反向追溯到前工序AOI/AVI/ET是否曾经报警。

FQC是最后防线，不应成为主要筛选工序。FQC map若长期显示同类位置缺陷，应回到前工序过程控制。

## AOI-ET-FQC整合

建议建立统一记录结构：

| 字段 | 示例 |
| --- | --- |
| 产品身份 | 客户、料号、版本、工单、lot、panel、pcs/strip/unit |
| 位置 | X/Y、row/column、layer、side、panel位置、unit ID |
| 来源 | AOI、ET、FQC、MRB、客户、复判 |
| 缺陷 | 原始缺陷码、统一缺陷码、严重度、图片/测试记录 |
| 判定 | 真不良、过杀、待判、复判OK、漏检、客户争议 |
| 处置 | 继续加工、返工、返修、让步、报废、客户确认 |
| 状态 | open、contained、MRB pending、scrapped、released、closed |
| 版本 | AOI程序、ET程序/netlist、FQC标准、MI/CAM版本 |
| 责任链 | 发现工序、责任工序、设备、班次、材料批次、操作者 |

整合后可以回答：

- 哪些AOI缺陷最终被ET或FQC证实为真不良。
- 哪些ET不良在AOI阶段没有被发现，可能是检测漏出或不可见缺陷。
- 哪些FQC缺陷本应在前工序被拦截。
- 哪些位置或设备重复贡献报废。
- 哪些缺陷是过杀导致的效率损失，不应计入真实报废原因。

## 报废Mapping

报废mapping要把“数量”拆成“位置、原因和处置证据”：

| 报废维度 | 说明 |
| --- | --- |
| 位置 | lot、panel、pcs、strip、unit、X/Y、row/column |
| 原因 | 原始发现缺陷、最终报废原因、责任工序 |
| 决策 | MRB结论、报废批准人、时间、依据 |
| 数量 | 报废pcs/unit、面积、金额、材料价值 |
| 状态 | 物理报废、系统报废、财务确认、环保/贵金属处置 |
| 防误用 | 实物破坏、隔离、标签、系统锁定 |

报废map尤其适合分析：

- 固定panel位置报废：查治具、夹点、搬运、喷淋、电镀电流分布。
- 固定工序后报废：查该工序参数、设备、人员和检验有效性。
- 固定缺陷码报废：查PFMEA、Control Plan和改善项目。
- 报废原因与原始缺陷不一致：查MRB分类和码表是否失真。

## 指标和看板

Mapping看板建议同时看数量、密度和状态：

| 指标 | 用途 |
| --- | --- |
| Defect density | 不同面积或不同unit数下比较缺陷密度 |
| Scrap rate | 报废率，按pcs、unit、panel、面积或金额定义 |
| FPY / RTY | 看一次通过和累计过程损失 |
| AOI true defect rate | AOI真不良比例，衡量程序有效性 |
| AOI false call rate | AOI过杀比例，衡量复判负担 |
| ET fail rate | 电测不良率，区分真实不良和治具误判 |
| FQC escape from previous process | FQC发现但前工序未拦截的比例 |
| Repeat location rate | 同一位置或同一设备区域重复异常比例 |
| MRB aging | 待判和MRB停留时间，衡量处置效率 |

缺陷密度应考虑机会数。不同面积、不同线密度、不同unit数量的产品，不能只比缺陷个数。

## 与SPC和DMAIC的关系

Mapping用于定位和分层，SPC用于判断稳定性和异常信号，DMAIC用于系统改善。

- Mapping发现固定位置：优先查设备、治具、流场、曝光/蚀刻/电镀均匀性。
- Mapping发现随机分布但密度上升：优先查材料、环境、洁净度、药水和人员。
- Mapping发现某缺陷码突增：做Pareto和SPC判异，再进入8D或DMAIC。
- Mapping发现FQC漏出增加：查前工序检测能力、限度样和反应计划。
- Mapping发现过杀升高：查AOI/AVI程序、属性MSA和复判一致性。

## 客诉和流出分析

发生客户投诉时，mapping可以帮助回答：

- 客户失效位置是否与厂内AOI/ET/FQC记录对应。
- 同panel、同strip、同unit邻近位置是否有类似报警。
- 同一设备、同一程序、同一材料批次是否产生相同map特征。
- FQC是否曾发现相似缺陷但判定为可接受。
- 是否存在AOI报警被复判放行、ET复测放行或MRB让步记录。
- 影响范围应扩大到哪些批次、相邻panel或相似产品。

客诉分析不能只看最终出货记录；要追到前工序mapping和复判历史。

## 系统实现建议

如果要在MES/QMS/SPC系统中落地，建议最小化实现：

1. 统一主键：lot + panel ID + pcs/strip/unit ID + layer/side + X/Y或row/column。
2. 统一码表：AOI、ET、FQC、MRB和客户缺陷码映射到内部标准码。
3. 保留原始数据：设备原始文件、图片、ET log、FQC照片和复判记录。
4. 建立坐标转换：CAM panel布局、设备坐标和客户map方向可互转。
5. 状态联动：检测结果、复判、MRB、返工、报废和恢复放行同步更新。
6. 看板输出：heat map、Pareto、趋势、MRB aging、缺陷密度和报废金额。
7. 权限控制：缺陷码修改、报废状态、复判结论和客户map导出要受控。

系统初期不要追求复杂算法。先保证身份、坐标、缺陷码和状态准确。

## 审核常看点

客户、IATF或内部审核可能会看：

- AOI、ET、FQC结果是否能追到lot、panel、pcs/strip/unit。
- 缺陷map和实物隔离、MRB、报废记录是否一致。
- AOI过杀、漏检、复判和属性一致性是否受控。
- ET治具/程序误判是否有验证和复测规则。
- FQC发现的问题是否回流到前工序控制。
- 报废数量、原因、位置、系统状态和财务/MES记录是否一致。
- 客诉样品能否追到厂内历史mapping和复判记录。

## 常见错误

- 只统计报废数量，不记录报废位置和原始缺陷。
- AOI、ET、FQC各有一套缺陷码，最终无法合并分析。
- 设备坐标方向不同，叠图后误判为固定位置异常。
- AOI过杀和真不良混在一起，导致报废Pareto失真。
- ET复测OK后没有保留原始fail记录，漏掉治具或间歇性问题。
- FQC只记录不良件数，不记录panel/pcs位置和图片。
- panel拆分后追溯断裂，无法判断相邻位置风险。
- map看起来漂亮，但与MRB、报废、返工和出货状态不联动。

## 快速落地流程

`定义产品位置主键 -> 统一AOI/ET/FQC缺陷码 -> 确认坐标和版面方向 -> 导入原始检验结果 -> 复判和MRB状态联动 -> 生成缺陷/报废heat map和Pareto -> 按位置/设备/材料/班次分层 -> 触发8D/DMAIC/SPC反应 -> 回写Control Plan、FMEA和检验程序`
