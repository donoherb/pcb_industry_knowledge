# 审核准备、LPA与过程审核

来源更新：2026-04-17。本文沉淀PCB/载板行业客户审核、内部审核、分层过程审核、过程审核和产品审核的稳定知识，不替代ISO 9001、IATF 16949、VDA 6.3、AIAG核心工具手册、IPC标准、客户CSR、客户审核清单和企业内部程序文件。客户指定审核方法、评分规则、提交时限和证据格式时，应以客户文件为准。

## 一句话理解

审核准备不是临时整理文件，而是证明“客户要求 -> 工程转化 -> 过程控制 -> 现场执行 -> 检测放行 -> 异常闭环 -> 变更管理”这条证据链在真实量产条件下持续有效。

## 审核类型

| 类型 | 目的 | PCB/载板常见关注 |
| --- | --- | --- |
| 客户审核 | 判断供应商是否具备量产、变更和问题闭环能力 | CSR、APQP、PPAP、特殊特性、追溯、客户料号现场一致性 |
| 认证审核 | 评价体系是否满足ISO/IATF等标准 | 体系过程、内审、管理评审、核心工具、纠正措施 |
| 内部体系审核 | 自查体系运行和流程接口 | 程序文件、跨部门接口、记录完整性 |
| 过程审核 | 沿产品实现流程验证过程能力和现场执行 | MI/CAM、压合、钻孔、沉铜电镀、图形、阻焊、表面处理、终检 |
| 产品审核 | 从成品角度验证产品持续满足规格 | 外观、尺寸、镀层、阻抗、电测、包装标签、可靠性抽样 |
| LPA分层过程审核 | 用多层级短频快检查确认关键控制被持续执行 | 首件、点检、SPC反应、药水分析、钻咀寿命、标签扫描 |
| 供应商审核 | 确认关键材料和外协过程受控 | AVL、变更、CoA/CoC、批次追溯、SCAR和二级供应商 |

VDA 6.3常用于汽车供应链过程审核。是否采用、采用哪些章节和评分规则，应按客户CSR、项目合同或内部程序确定。

## 审核主线

一个成熟的PCB/载板审核通常围绕六条线展开：

1. 要求线：客户图纸、规格、CSR、IPC等级、法规和特殊要求如何被识别与转化。
2. 产品线：目标产品的设计、材料、叠构、工艺路线和检验标准是否明确。
3. 过程线：从来料到出货的流程、设备、人员、环境、参数和记录是否一致。
4. 风险线：PFMEA、Control Plan、MSA、SPC、8D和ECN是否围绕真实风险运行。
5. 追溯线：任一出货批次能否追到材料、设备、人员、程序、检验和放行。
6. 闭环线：异常、客诉、供应商问题、审核发现和变更是否回写系统。

## 与APQP、PPAP、Control Plan和8D的关系

| 文件/工具 | 审核时怎么抽 | 常见风险 |
| --- | --- | --- |
| APQP | 抽一个项目，从VOC/CSR追到量产反馈 | 资料存在但阶段输出互不连接 |
| PPAP | 抽PSW或提交包，看现场是否按批准状态生产 | PPAP包和现场工艺、材料、设备版本不一致 |
| Control Plan | 抽一项关键特性，看SOP、检验记录、MSA/SPC和反应计划 | 控制计划写了，但现场未按频率执行 |
| PFMEA | 抽高风险失效模式，看控制和改善是否落地 | 客诉后FMEA没有更新 |
| MSA | 抽关键量具，看方法、人员、GRR/属性一致性和校准 | 用未验证的测量系统做放行 |
| SPC | 抽控制图，看判稳、判异、反应和能力计算 | 只画图，不按异常规则行动 |
| 8D/SCAR | 抽重复问题，看根因、措施、验证和防再发 | 关闭报告但问题重复 |
| ECN/ECR | 抽一次变更，看影响评估、客户批准和切换点 | 材料/程序/地点变化未触发再评估 |

## 审核前准备

审核前不要只准备会议室资料。建议按以下顺序：

1. 确认审核目的：新供应商准入、年度审核、项目导入、客诉后审核、量产过程审核或认证审核。
2. 固定范围：客户、料号、产品族、工厂、产线、班次、外协过程和供应商边界。
3. 读取要求：客户CSR、图纸、质量协议、PPAP等级、变更规则、包装标签和追溯要求。
4. 选样本：选近期量产批、异常批、变更批、客户投诉批或审核方指定批次。
5. 做证据矩阵：把每项客户要求连接到文件、记录、现场位置和责任人。
6. 预走现场：按真实物流和工艺路线从收货走到出货，检查文件和现场是否一致。
7. 复盘历史问题：上次审核发现、客户投诉、重大报废、供应商异常和ECN关闭状态。

## 审核证据矩阵

建议建立一张简单矩阵：

| 字段 | 说明 |
| --- | --- |
| 要求来源 | 客户CSR、图纸、IPC、内部规范、法规、质量协议 |
| 要求内容 | 例如阻抗公差、标签字段、PPAP提交、年度验证、变更提前通知 |
| 内部转化 | MI、SOP、检验规范、Control Plan、PFMEA、包装规范 |
| 现场证据 | 工单、首件、SPC、检验记录、设备点检、标签扫描、出货报告 |
| 责任部门 | 工程、质量、制造、采购、仓储、实验室、物流 |
| 状态 | 已执行、证据缺口、偏差批准、需客户确认 |
| 风险 | 缺证据、版本不一致、执行不稳定、依赖人工记忆 |

这张表不是为了替代正式审核清单，而是帮助内部先发现证据断点。

## LPA分层过程审核

LPA的价值在于用短周期、多层级、固定问题确认关键控制是否每天都在执行。它不是另一次全面体系审核。

### LPA设计原则

- 问题少而关键，优先选特殊特性、高风险工序、历史客诉和客户关注点。
- 以现场行为和记录为准，少问“是否理解”，多看“是否按批准方法做”。
- 层级明确：班组长、工程/质量、部门经理和高层分别承担不同频率。
- 发现异常要有即时反应计划，不能只记录分数。
- 问题要定期刷新，来自8D、SPC异常、内审、客户审核和ECN。

### PCB/载板LPA样题

| 工序/场景 | 样题 |
| --- | --- |
| 来料 | 当前使用材料是否为AVL批准供应商和有效批次 |
| 工程资料 | 现场MI、程序和客户版本是否为当前批准版本 |
| 压合 | 压合程序、叠构、材料批次和首件记录是否一致 |
| 钻孔 | 钻咀寿命、断针记录、孔位首件和程序版本是否受控 |
| 沉铜/电镀 | 药水分析、槽液参数、铜厚SPC和异常反应是否按频率执行 |
| 图形/蚀刻 | 线宽线距首件、AOI参数和复判规则是否受控 |
| 阻焊 | 油墨批次、曝光能量、显影窗口和外观限度样是否可追溯 |
| 表面处理 | 镀层厚度、可焊性、药水状态和外协记录是否符合要求 |
| 电测 | 治具/程序版本、开短路判定和复测记录是否一致 |
| 包装标签 | 客户料号、批号、数量、RoHS/UL、条码扫描和防潮包装是否正确 |

### LPA输出

LPA输出至少包括：

- 审核日期、班次、区域、审核人和被审核过程。
- 问题、证据、风险等级、即时遏制和责任人。
- 关闭证据和效果验证。
- 趋势分析：高频问题、重复区域、责任部门和系统性原因。

## 过程审核

过程审核应沿真实工艺路线抽样，而不是只看文件夹。推荐方式：

`客户要求/工单 -> MI/CAM -> 来料 -> 内层/压合 -> 钻孔 -> 沉铜/电镀 -> 图形 -> 阻焊 -> 表面处理 -> 成型/电测 -> FQC -> 包装出货 -> 异常和变更记录`

每个工序至少看五件事：

1. 输入是否正确：材料、程序、图纸、设备、人员、环境。
2. 过程是否受控：参数、点检、首件、SPC、换线和反应计划。
3. 输出是否可判定：检验方法、量具、限度样、记录和放行权限。
4. 异常是否闭环：隔离、返工、MRB、8D、SCAR和再检。
5. 版本是否一致：MI、SOP、Control Plan、PFMEA、程序和客户批准状态。

## 产品审核

产品审核从客户视角确认成品是否持续满足要求。常见项目：

- 外观：按客户标准、IPC-A-600或内部限度样判定。
- 尺寸：外形、孔径、槽、焊盘、板厚、翘曲和关键公差。
- 电性能：开短路、阻抗、TDR、绝缘电阻、耐压或客户指定项目。
- 镀层和表面处理：厚度、可焊性、污染、镍腐蚀或孔隙风险。
- 可靠性：热应力、切片、热循环、离子污染、CAF或客户年度验证项目。
- 包装标签：客户料号、批号、数量、日期、条码、RoHS/UL、防潮、防静电和运输保护。

产品审核不等于出货检验重复。它更关注系统性：抽到的问题是否说明过程控制、检验策略或客户要求转化存在缺口。

## 客户审核资料包

客户审核资料包建议分成七类：

| 类别 | 典型资料 |
| --- | --- |
| 体系和组织 | 证书、范围、组织职责、培训矩阵、内审和管理评审摘要 |
| 客户要求 | CSR矩阵、图纸版本、质量协议、PPAP等级、变更规则 |
| 项目资料 | APQP计划、过程流程图、PFMEA、Control Plan、Run at Rate、PPAP/PSW |
| 现场执行 | 工单、MI、SOP、首件、SPC、MSA、检验记录、设备点检 |
| 质量闭环 | 客诉8D、内部NCR/MRB、SCAR、重复问题和防再发证据 |
| 变更追溯 | ECR/ECN、客户批准、切换点、旧料消耗、库存隔离 |
| 供应链 | AVL/ASL、关键供应商审核、CoA/CoC、来料趋势、外协控制 |

资料包应能支持抽样追溯。审核员随机抽一个客户料号或出货批次时，文件、记录和现场要能串起来。

## 现场审核行为

现场审核常暴露的问题不是没有文件，而是人员说法和现场状态不一致。建议：

- 让岗位人员按实际流程说明，不要替现场回答所有问题。
- 不确定时查文件和记录，不要现场临时编口径。
- 对版本、客户批准、偏差和让步要讲清状态。
- 看到明显异常先遏制，再解释原因。
- 审核中产生的承诺要记录责任人和日期，避免会后遗忘。

## 审核发现分级

不同客户和认证机构分级不同，内部可用以下风险口径先判断：

| 等级 | 典型情形 | 反应 |
| --- | --- | --- |
| 重大 | 可能影响客户安全/法规/功能/批准状态，或体系失效 | 立即遏制，管理层升级，必要时客户通知 |
| 一般 | 单点未按流程执行，暂未证实影响客户 | 限期纠正，验证类似区域 |
| 观察项 | 风险苗头或改善建议 | 纳入改善计划或LPA刷新 |

关键不是名称，而是反应强度是否与客户风险匹配。

## 审核后纠正措施

审核后建议按8D思路处理：

1. 固定问题：写清要求、事实、证据、地点、料号、批次和影响。
2. 立即遏制：隔离可疑产品、补查相似过程、暂停风险动作。
3. 根因分析：区分发生原因、流出原因和系统原因。
4. 永久措施：更新流程、文件、培训、系统防错、供应商要求或控制计划。
5. 横向展开：检查相似产品、相似工序、其他班次、其他客户和供应商。
6. 效果验证：用后续记录、LPA、SPC、客诉趋势或复审证明措施有效。
7. 关闭回写：把经验写入PFMEA、Control Plan、SOP、审核清单和培训。

## PCB/载板高频审核发现

| 主题 | 高频问题 |
| --- | --- |
| 客户要求 | CSR未转成内部要求，包装/标签/变更/年度验证遗漏 |
| 文件版本 | MI、SOP、程序、Control Plan和客户图纸版本不一致 |
| 特殊特性 | 控制计划有特殊特性，现场没有加严控制或能力证据 |
| MSA/SPC | 关键量具无MSA，控制图异常无反应，能力指数基于失控过程 |
| 追溯 | 出货批次不能完整追到材料、设备、程序、人员和检验记录 |
| 不合格品 | 隔离区、标识、MRB结论和返工再检证据不足 |
| 供应商 | 外协特殊过程未纳入审核，供应商变更没有客户批准评估 |
| ECN | 变更切换点不清，旧版库存和在制品未受控 |
| 培训 | 只保留签到表，未证明关键岗位能力 |
| LPA | 只打勾，不触发反应计划和趋势改善 |

## 常见错误

- 审核前临时补记录，导致日期、批次、版本和签核逻辑不一致。
- 只准备体系文件，不按客户料号和真实批次做追溯演练。
- 把审核发现当作表面纠正，没有做根因和横向展开。
- 过程审核只看车间5S，不抽关键参数、首件、SPC和反应计划。
- 产品审核只复测尺寸，不看包装标签、客户特殊要求和年度验证。
- LPA问题多年不变，没有吸收客诉、8D、ECN和SPC异常。
- 客户审核承诺没有进入CAPA系统，后续无人追踪。

## 快速落地流程

`确认审核范围 -> 建立客户要求矩阵 -> 选样本批次 -> 追溯文件和现场证据 -> 预走过程审核 -> 刷新LPA重点问题 -> 审核中即时遏制风险 -> 审核后按8D/CAPA关闭 -> 回写PFMEA/Control Plan/SOP/培训/供应商要求`
