本文沉淀PCB/载板行业的变更管理稳定知识和ECR/ECN场景化方法，不替代企业PLM/QMS流程、客户质量协议、CSR、客户变更批准规则和法规要求。凡涉及客户批准、再PPAP、年度验证、外协过程和生产地点变化，必须以客户文件与书面批准为准。

## 一句话理解

变更管理回答的问题不是“改了以后怎么通知一下”，而是“任何可能影响产品、过程、客户批准或追溯基线的变化，如何在发生前被识别、评估、批准、切换和追踪，避免未受控变化进入量产”。

## ECR与ECN的边界

| 项目 | `ECR` | `ECN` |
| --- | --- | --- |
| 角色 | 提出变更需求和论证 | 批准后正式发布执行 |
| 核心问题 | 要不要改，为什么改，影响什么 | 何时改、怎么改、从哪一批开始改 |
| 输出 | 变更说明、影响评估、建议方案 | 生效通知、版本切换、执行要求、追溯边界 |

简单说：

- `ECR` 是“提议和评估”。
- `ECN` 是“批准和发布”。

没有完成影响评估的 `ECR`，不应被现场当成已生效变更。

## 为什么PCB/载板行业更需要严变更

PCB与IC载板项目的很多结果高度耦合。一个看似局部的变化，可能同时影响：

- 阻抗、板厚、翘曲和可靠性
- 良率、报废和节拍
- 客户PPAP批准状态
- 标签、包装和追溯
- 下游SMT、封装和整机使用

因此，“换个材料牌号”“调一版程序”“切到另一条线”“临时换外协”“搬一台设备”都不应视作小变更。

## 常见变更类型

| 类别 | 典型内容 |
| --- | --- |
| 设计变更 | 图纸、叠构、线宽线距、孔结构、表面处理、标签格式 |
| 材料变更 | 材料牌号、供应商、批次规则、二供导入、药水体系 |
| 过程变更 | 工艺参数窗口、流程路径、返工工艺、检验方法 |
| 设备/工装变更 | 新设备、设备搬迁、治具改版、AOI/TDR/XRF程序改版 |
| 地点变更 | 工厂切换、外协切换、产线转移 |
| 包装与物流变更 | 标签、MBB/HIC、包装材料、装箱和条码规则 |

## 变更管理的核心原则

1. 先评估，再执行，不允许“先改着试试再补流程”。
2. 变更评估必须覆盖产品、过程、客户批准、库存和追溯。
3. 任何版本切换都必须有清晰的生效切换点。
4. 旧版物料、旧程序、旧标签和旧库存必须有耗尽计划。
5. 重大变更必须验证，不是只批准文档。

## 与APQP、PPAP、8D、不合格品管理的关系

| 文件/活动 | 与变更管理的关系 | PCB/载板落地重点 |
| --- | --- | --- |
| `APQP` | 新项目与重大变更常需重新走部分APQP逻辑 | 风险、验证、Control Plan和特殊特性需重审 |
| `PPAP` | 部分变更会触发再PPAP或客户确认 | 材料、地点、程序、设备和外协变化要评估 |
| `8D` | 永久措施若涉及变更，必须走ECR/ECN | 不能只在8D里写“已修改” |
| `17-nonconforming-product-management.md` | 旧版/新版混用、标签错误、未批准变更常会形成不合格品 | 切换边界和库存隔离要清楚 |
| `FMEA/Control Plan` | 变更后需同步更新风险和控制 | 否则现场按旧逻辑执行新状态 |

典型链路：

`变更需求 -> ECR -> 影响评估 -> 批准 -> 验证/客户批准 -> ECN发布 -> 切换执行 -> 旧版退出 -> 效果追踪 -> FMEA/Control Plan/PPAP更新`

## 什么时候必须走正式变更

通常以下场景不应靠邮件或口头确认替代正式流程：

- 图纸、叠构、特殊特性变化
- 材料牌号或供应商变化
- 新设备、设备搬迁、工装改版
- AOI/AVI/TDR/XRF/电测等程序版本变化
- 工厂/产线/外协地点转移
- 包装、标签、条码和追溯规则变化
- 返工工艺新增或显著修改

## 影响评估至少要覆盖的六个问题

1. 是否影响客户功能、可靠性、装配或法规要求。
2. 是否影响特殊特性、MSA、SPC、能力或检验方法。
3. 是否影响PPAP、客户批准或年度验证状态。
4. 是否影响旧库存、在制品、在途和客户现场产品。
5. 是否需要试样、试产、Run at Rate或补充可靠性验证。
6. 是否需要更新图纸、SOP、FMEA、Control Plan、标签和系统主数据。

## 变更影响评估表的建议字段

| 字段 | 说明 |
| --- | --- |
| 变更编号 | 对应ECR/ECN唯一编号 |
| 变更对象 | 产品、材料、工艺、设备、地点、程序、标签等 |
| 变更原因 | 客户需求、降本、风险关闭、替代、产能切换、供应风险 |
| 影响范围 | 料号、客户、工厂、设备、供应商、库存 |
| 风险评估 | 功能、可靠性、装配、法规、批准状态 |
| 验证要求 | 小样、试产、MSA、SPC、可靠性、客户确认 |
| 文件更新 | 图纸、BOM、SOP、PFMEA、Control Plan、PPAP、标签 |
| 生效切换点 | 时间、批次、数量或订单边界 |

## 生效切换点与旧版退出

变更失控最常见的问题，不是没批，而是“新旧并存没人说清楚”。

### 必须明确

- 从哪一批开始执行新版本。
- 旧版库存能否继续使用，使用到什么时候。
- 在制品如何区分旧版和新版。
- 标签、程序、治具、SOP何时同步切换。
- 客户是否接受过渡期混用。

### 常见切换方式

| 方式 | 适用场景 |
| --- | --- |
| 按日期切换 | 节点明确但库存不复杂 |
| 按lot/批次切换 | 制造追溯更清晰 |
| 按客户订单切换 | 多客户不同批准状态时 |
| 按库存耗尽切换 | 需客户允许且风险可控时 |

## 典型验证要求

| 变更类型 | 常见验证 |
| --- | --- |
| 材料替代 | 等效性验证、可靠性、阻抗/板厚/翘曲、客户确认 |
| 工艺参数调整 | 小批验证、SPC趋势、良率和缺陷结构对比 |
| 设备/程序改版 | 首件、MSA复核、能力或相关性验证 |
| 地点转移 | 过程验证、试生产、Run at Rate、再PPAP评估 |
| 包装标签变更 | 客户系统收货、条码扫描、防护验证 |

## PCB/载板高风险变更场景

| 场景 | 主要风险 |
| --- | --- |
| 低损耗材料替代 | 阻抗、损耗、压合和可靠性联动变化 |
| 电镀/蚀刻药水体系变化 | 线宽、孔铜、表面状态和良率漂移 |
| AOI/AVI程序改版 | 过杀/漏判口径变化，历史趋势断裂 |
| TDR/XRF程序改版 | 测量系统口径变化，能力数据失真 |
| 外协表面处理切换 | 外观、可焊性、镀层厚度和追溯断点 |
| 产线/工厂搬迁 | 人员、设备、治具、节拍和批准状态变化 |

## 与不合格品管理的接口

未受控变更常直接导致不合格品：

- 新旧标签混用
- 旧版程序误上机
- 未批准材料已投产
- 客户尚未批准，新版已出货

因此，若发现变更边界不清、版本混用、客户批准缺失，应立即按不合格品管理处理，而不是只做文件修正。

## 与PPAP的接口

变更后至少要问：

- 是否触发再PPAP或客户确认。
- 当前PPAP基线是否仍有效。
- 试样和量产样是否仍代表同一批准状态。
- 临时偏差、过渡期混用和库存耗尽是否已被客户书面接受。

## 与8D的接口

8D中若永久措施写到以下内容，通常就应进入ECR/ECN：

- 改材料
- 改程序
- 改工艺窗口
- 改设备/治具
- 改标签/包装
- 改图纸或特殊特性

否则8D只是写了“会改”，但系统没有真正切换。

## 变更管理审核常看点

- ECR与ECN是否分开管理。
- 影响评估是否覆盖客户批准、库存和在制。
- 切换点是否明确。
- 旧版物料和旧程序是否有退出证据。
- 图纸、SOP、PFMEA、Control Plan、PPAP是否同步更新。
- 变更后是否有验证和效果追踪。

## PCB/载板场景下的常见错误

- 把程序改版视作“小调整”，不走变更流程。
- 客户已批准旧材料，却内部先切新材料试产。
- 只更新图纸，不更新Control Plan和检验规范。
- 旧标签和新标签混用，追溯断裂。
- 搬线或转厂后，只看产能，不做再PPAP评估。
- 让步或偏差许可结束后，临时状态被默认成永久状态。

## 快速落地流程

`提出变更需求 -> 开ECR -> 做影响评估 -> 判断是否需客户批准/再PPAP -> 做验证 -> 发布ECN -> 明确切换点和旧版退出 -> 更新文件和系统 -> 跟踪效果`

使用建议：

- 做材料替代、程序改版、设备搬迁和工厂转移时，优先先用本页做影响评估，再回到 `30-ppap.md` 看是否触发再提交。
- 做工程资料、CAM/MI、程序版本、标签模板和客户数据变更时，结合 `33-engineering-cam-mi-data-control.md`。
- 做设备、治具、软件升级、大修和搬迁变更时，结合 `34-equipment-maintenance-and-tooling.md`。
- 做药水、化学品、环保处理方式和禁限用物质变更时，结合 `35-environment-ehs-chemical-management.md`。
- 做8D永久措施时，凡涉及“改”，都应回看本页，避免措施停留在纸面。
- 做不合格品管理时，若发现版本混用或未批准变更，直接按 `17-nonconforming-product-management.md` 先隔离再处理。
