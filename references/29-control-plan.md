# Control Plan与现场控制

来源更新：2026-04-17。本文沉淀Control Plan的稳定知识、PCB/载板场景化方法和常见误区，不替代AIAG Control Plan手册、客户CSR、内部程序文件和已批准的现场控制文件。手册版本状态、客户要求和标准更新规则见 `04-process-quality.md` 与 `07-source-standards.md`。

## 一句话理解

Control Plan回答的问题不是“要不要检”，而是“哪些产品特性和过程参数必须在什么工序、用什么方法、按什么频率、由谁、以什么反应计划被控制住”。

## 与APQP、PFMEA、MSA、SPC、PPAP、SOP的接口

Control Plan不是孤立文件，它位于风险识别和现场执行之间，是APQP证据链里最接近量产现场的一层。

| 文件/工具 | 与Control Plan的关系 | PCB/载板落地重点 |
| --- | --- | --- |
| 过程流程图 | 定义有哪些工序、外协、返工和检验点 | 工序编号、流程边界、返工/重工路径必须一致 |
| PFMEA | 定义这些工序如何失效、哪些风险高 | 高严重度、特殊特性、探测弱项要进入控制计划 |
| SOP/检验规范 | 定义控制点如何具体执行 | 参数窗口、测点、限度样、记录方式必须一致 |
| MSA | 决定Control Plan里的测量方法是否可信 | 关键测量系统要有对应MSA状态和使用边界 |
| SPC | 决定哪些特性/参数需要持续统计监控 | 控制图类型、抽样和反应计划应写进Control Plan |
| PPAP | 用Control Plan证明量产控制逻辑已定义且可执行 | Pre-Launch/Production版本及现场执行证据要可追溯 |
| LPA/过程审核 | 验证Control Plan是否真正在现场执行 | 不只看文件存在，还要看现场记录和反应动作 |

典型链路：

`客户要求/特殊特性 -> 过程流程图 -> PFMEA -> Control Plan -> SOP/检验规范 -> MSA/SPC -> PPAP/量产执行 -> LPA/8D/ECN -> Control Plan更新`

## Control Plan不是什么

常见误解要先排除：

| 误解 | 实际上是什么 |
| --- | --- |
| 只是检验项目清单 | 它同时覆盖过程参数、产品特性、抽样、记录和反应计划 |
| 只给质量部用 | 生产、工艺、设备、实验室、供应链都要执行和维护 |
| 做完PPAP就结束 | 它是量产现场运行标准，异常和变更后必须更新 |
| PFMEA的重复粘贴 | 它应把风险转成可执行控制，不是复制风险描述 |
| 只写结果检验 | 对特殊过程和高风险过程，参数控制和防错更关键 |

## 三种控制计划层级

Control Plan通常至少分为三个层级。不同客户命名可能不同，但逻辑相近。

| 层级 | 目的 | PCB/载板典型用法 |
| --- | --- | --- |
| Prototype Control Plan | 样件阶段快速识别设计和工艺风险 | 样件阻抗coupon、微切片、关键尺寸和外观加严确认 |
| Pre-Launch Control Plan | 试生产阶段验证接近量产条件下的控制逻辑 | 提高抽样频率、加严首件、增加班次切换确认和异常评审 |
| Production Control Plan | 正式量产阶段的标准控制文件 | 固化特性、频率、MSA/SPC、记录和反应计划 |

很多PCB/载板项目还会在量产初期附加 `GP-12`、`Safe Launch` 或“早期生产遏制计划”。这通常不是第四种独立Control Plan，而是对Pre-Launch/Production阶段的加严版本。

## 输入来源

好的Control Plan来自多源输入，而不是QE单独填表。

| 输入来源 | 作用 | PCB/载板示例 |
| --- | --- | --- |
| 客户图纸/CSR | 定义必须满足的合同要求 | 特殊特性、标签、包装、PPAP等级、年度验证 |
| 特殊特性清单 | 定义必须重点控制的项 | 阻抗、孔铜、线宽线距、翘曲、镀层厚度 |
| PFMEA | 定义高风险失效和控制薄弱点 | 孔铜不足、分层、ENIG异常、AOI漏判 |
| 过程流程图 | 定义控制点位于哪个工序 | 背钻、激光钻孔、填孔、电测、外协表面处理 |
| 质量历史 | 定义历史逃逸和客户投诉要防再发 | CAF、阻抗偏差、包装划伤、标签错误 |
| MSA/SPC计划 | 定义测量和统计方法是否可行 | XRF、TDR、AOI、微切片、I-MR/Xbar-R |
| 设备和供应链条件 | 定义现场实际能执行的控制方式 | 药水分析频率、治具寿命、材料批次追溯 |

## 哪些项目必须进Control Plan

不是所有工序特性都要用同样深度写入Control Plan，但以下项目优先级通常最高：

1. 客户特殊特性、CTQ、法规/安全相关项。
2. PFMEA中高严重度、高发生率、探测弱的项目。
3. 终检难以完全筛出的特殊过程输出。
4. 历史客诉、重复8D、内部高COPQ项目。
5. 换料、换机、换程序、换厂容易失控的项目。
6. 需要SPC、MSA、能力研究和PPAP证据支撑的项目。

常见类别：

- 产品特性：尺寸、孔位、孔铜、阻抗、镀层厚度、翘曲、外观。
- 过程参数：电流密度、药水浓度、蚀刻速率、压合曲线、激光能量。
- 设备/工装状态：程序版本、治具寿命、点检结果、夹具状态。
- 材料状态：批次、有效期、烘烤、储存、防潮、客户指定料。
- 防错机制：条码扫描、程序锁定、权限、互锁、黄金样。

## 字段设计

Control Plan最重要的是字段能支撑现场执行和审核追溯。

| 字段 | 说明 | PCB/载板注意点 |
| --- | --- | --- |
| 工序编号/名称 | 与流程图和PFMEA一致 | 必须覆盖外协、返工、重工和终检包装 |
| 产品/过程特性 | 要控制什么 | 区分产品结果和过程参数，不要混写 |
| 特殊特性标识 | 是否为客户/内部关键项 | 保持与图纸、PFMEA、PPAP一致 |
| 规格/目标/公差 | 依据什么控制 | 来自客户图纸、内部窗口或标准 |
| 控制方法 | 如何控制 | 首件、巡检、SPC、实验室测试、防错、自动检测 |
| 测量系统 | 用什么设备/程序/夹具 | 设备编号、程序版本、MSA状态要可追溯 |
| 抽样单位/频率/样本量 | 多久测、测多少 | 件、panel、lot、strip、coupon必须写清 |
| 记录方式 | 记录存在哪里 | MES、QMS、实验室报告、纸质点检表 |
| 责任人/部门 | 谁执行、谁复核 | 生产、QE、PE、实验室、设备要分清 |
| 反应计划 | 异常怎么办 | 停机、隔离、复测、升级、客户通知必须具体 |

## 控制方法的类型

现场控制不只有“测量”一种方式。Control Plan通常应组合使用以下方法：

| 控制类型 | 作用 | PCB/载板示例 |
| --- | --- | --- |
| 首件确认 | 防止切换或开机初始错误 | LDI首件线宽、钻孔首件、阻焊首件 |
| 参数控制 | 控住特殊过程窗口 | 电镀电流密度、蚀刻温度、压合曲线 |
| SPC | 识别趋势和特殊原因 | 线宽、阻抗、镀层厚度、药水参数 |
| 自动检测 | 提升检出率和节拍 | AOI、AVI、电测、条码互锁 |
| 实验室验证 | 覆盖破坏性或低频高风险项目 | 微切片、离子污染、可焊性、可靠性 |
| 防错与权限 | 避免混料、错程序、错判 | 材料扫描、程序锁定、权限控制 |
| 审核型控制 | 验证执行一致性 | LPA、过程审核、产品审核 |

原则上，越靠前预防、越自动化、越不依赖最终筛选的控制，价值越高。

## 抽样与频率设计

频率不能拍脑袋。过低发现不了风险，过高现场执行不动。

| 设计维度 | 应考虑什么 | PCB/载板例子 |
| --- | --- | --- |
| 风险等级 | 特殊特性、历史客诉、高成本失效是否存在 | 孔铜、阻抗、warpage、标签追溯 |
| 过程稳定性 | 新项目还是成熟量产 | 试产阶段加严，成熟阶段按判稳结果调整 |
| 检测成本 | 是否破坏性、是否高节拍 | 微切片不能和AOI同频，但需coupon支撑 |
| 批次转换 | 换料、换机、换程序、换班 | 每次切换后首件或首批确认 |
| 客户要求 | CSR、图纸或质量协议是否指定 | 汽车项目常要求加严和保留记录 |

常见表述方式：

- 首件 + 每小时
- 每批/每lot
- 每panel固定点位
- 每班/每次切换后
- 连续监控 + 超限报警

频率变更本身通常也需要经过风险评估，不应现场口头下调。

## 反应计划怎么写才有用

反应计划不能写成抽象口号。至少要回答以下问题：

| 问题 | 反应计划应写明什么 |
| --- | --- |
| 何时触发 | 超规格、SPC判异、首件失败、设备报警、来料异常 |
| 立即动作 | 停机、暂停放行、加严抽样、切换备用设备、封存材料 |
| 隔离范围 | 同机台、同批、同班、前后批次、在制/库存/已出货 |
| 谁升级 | 操作员、线长、QE、PE、主管、项目经理 |
| 如何恢复 | 调机、复测、小批验证、首件批准、客户偏差许可 |
| 如何回写 | PFMEA、SOP、SPC、培训、供应商要求、8D |

低质量写法：

- “通知主管”
- “调整参数”
- “加强检查”

更可执行的写法：

- “停止该机台后续lot放行，隔离最近2小时在制品和同药槽批次，QE复核后方可恢复。”
- “阻抗连续3点偏向同侧时，加抽同panel其余coupon，并由PE确认叠构/线宽/材料批次。”

## 记录与追溯

Control Plan如果没有记录闭环，就只是一张模板。

记录至少应能追到：

- 产品/料号/版本
- lot、panel、strip、batch或样品编号
- 材料批次和供应商
- 设备、程序、夹具、治具和软件版本
- 操作员/检验员/班次
- 测量结果、判异结论和反应动作
- 放行、隔离、复测和恢复批准

对PCB/载板场景，包装标签、外协表面处理、电测治具、AOI程序、微切片位置和实验室报告编号都应纳入追溯链。

## PCB/载板常见控制项目

| 工序 | 常见控制项 | 控制方式示例 |
| --- | --- | --- |
| 开料/前处理 | 材料批次、方向、尺寸、烘烤 | 扫码防错、首件、批次记录 |
| 内层图形/蚀刻 | 线宽线距、补偿、对位、药水状态 | 首件、AOI、线宽SPC、药水分析 |
| 压合 | lay-up顺序、板厚、介质厚、层偏、翘曲 | 扫码、首件切片、板厚/层偏测量、曲线锁定 |
| 钻孔/激光 | 孔径、孔位、钻咀寿命、激光能量 | 程序锁定、寿命管控、X-ray、切片 |
| 沉铜/电镀 | 孔铜、面铜、填孔质量、药水参数 | 微切片、XRF、参数SPC、报警互锁 |
| 外层图形 | 线宽线距、残铜、短路、对位 | AOI/AVI、显影/蚀刻参数、复判规则 |
| 阻焊/字符 | 开窗、桥连、附着、偏位 | 限度样、AVI、首件、固化曲线 |
| 表面处理 | 镀层厚度、可焊性、外观、镍腐蚀风险 | XRF、外观、可焊性、药水分析 |
| 成型/电测/终检 | 尺寸、毛刺、电测、标签、包装 | 治具点检、黄金样、扫码、终检和包装确认 |

## 不同产品的增强项

同样一套Control Plan模板，不同产品要有不同侧重点。

| 产品/项目类型 | 需要强化的控制点 |
| --- | --- |
| 高多层/高速板 | 阻抗coupon、材料批次、压合介质厚度、背钻、损耗相关验证 |
| HDI/mSAP/SLP | 激光孔、填孔电镀、细线路、AOI/AVI阈值、洁净度 |
| FPC/软硬结合 | 弯折区设计限制、覆盖膜开窗、补强、尺寸稳定和贴装适配 |
| 汽车板 | 特殊特性、追溯、年度验证、LPA、外协和供应商变更控制 |
| IC载板/ABF | build-up层、微缺陷、strip级追溯、warpage、洁净环境和封装接口 |

## Control Plan评审与更新触发

以下情况通常应触发Control Plan评审或更新：

- 客户图纸、CSR、标签、包装或特殊特性变化。
- PFMEA风险更新，新增高严重度或高发生问题。
- 工艺窗口、设备、治具、程序、量具、供应商或地点变化。
- SPC长期异常、能力下降、客诉、8D或重复逃逸。
- 样件转试产、试产转量产、Safe Launch退出。
- MSA结果变化，量具失效或判定标准更新。

更新时要检查三件事是否仍一致：

1. 流程图里的工序编号。
2. PFMEA里的风险和现有控制。
3. Control Plan里的控制方法和反应计划。

## 审核常看点

客户审核、IATF审核和内部审核通常会看：

- 文件版本是否与现场一致。
- 特殊特性是否真进了Control Plan。
- 反应计划是否具体且有实际执行记录。
- 记录能否追到同一批样品、设备和程序。
- PFMEA高风险项是否在现场有对应控制。
- 现场是否按批准频率执行，而不是只在审核前补记录。

## PCB/载板场景下的常见错误

- 把Control Plan做成检验项目总表，没有过程参数和反应计划。
- PFMEA、流程图和Control Plan的工序编号不一致。
- 特殊特性只在图纸和PPAP里出现，现场文件没有体现。
- 频率和样本量脱离现场节拍，导致文件能过审但执行不了。
- 只写“通知主管”“调整参数”，没有隔离和恢复规则。
- 设备、AOI程序、治具或量具改版后未同步更新。
- 量产初期仍按工程样控制逻辑运行，未切换到Production版本。
- Control Plan更新后，SOP、培训、限度样和系统字段未同步。
- 只控制结果，不控制对结果有决定性影响的特殊过程参数。
- 客诉后只做加严检验，没有回写PFMEA和Control Plan。

## 快速落地流程

`从客户要求和特殊特性出发 -> 对齐流程图/PFMEA -> 选控制项和控制方式 -> 定义频率/样本量/测量系统 -> 写清反应计划和记录方式 -> 试产验证 -> 与MSA/SPC/PPAP联动 -> 量产审核和持续更新`

使用建议：

- 做PFMEA到现场执行的转换时，先用本页把风险变成可执行控制，再结合 `28-spc.md` 和 `27-msa.md` 设计数据与测量部分。
- 做PPAP、客户审核和量产导入时，把Control Plan作为现场主线文件，不要等提交资料时再补模板。
- 做AOI/ET/FQC检测站点、缺陷map、报废map和反应计划设计时，结合 `36-defect-scrap-mapping-aoi-et-fqc.md`。
- 做客户投诉、8D或量产变更评估时，先回到Control Plan看原控制逻辑，再判断问题出在控制缺失、执行失效还是判定标准失真。
