# DMAIC与六西格玛改善

来源更新：2026-04-17。本文沉淀DMAIC作为稳定知识和PCB/载板场景化方法，不替代企业内部六西格玛程序、财务收益认定规则、客户批准流程和统计软件操作规范。若企业已有Black Belt/Green Belt模板、项目立项规则和财务认证口径，应以企业正式版本为准。

## 一句话理解

DMAIC回答的问题不是“问题出了怎么止血”，而是“如何把一个反复发生或长期存在的绩效问题，定义清楚、测准、找出主因、验证改善并长期守住”。

## DMAIC适合什么，不适合什么

DMAIC适合以下场景：

- 良率长期偏低，但不是一次性偶发事故。
- 报废率、返工率、缺陷密度、周期、过杀率长期偏高。
- 阻抗、孔铜、板厚、镀层等关键特性波动大。
- 多个工厂/班次/机台存在系统性差异，需要结构化改善。
- 管理层要求看到基线、主因、收益和控制计划闭环。

DMAIC不适合以下场景：

- 客户投诉后必须在短时间内正式回复的紧急闭环，优先用 `8D`。
- 新产品开发阶段的前置质量策划，优先用 `APQP`。
- 单次设备故障或一次性异常，优先用日常异常单或短闭环。

## 与APQP、8D、FMEA、MSA、SPC的关系

| 工具 | DMAIC与它的关系 | PCB/载板落地重点 |
| --- | --- | --- |
| `APQP` | APQP偏新项目前置策划，DMAIC偏既有过程绩效改善 | 量产后良率/周期/报废改善常更适合DMAIC |
| `8D` | 8D偏正式问题闭环，DMAIC偏数据驱动持续改善 | 一次重大客诉可先8D止血，再用DMAIC做系统改善 |
| `FMEA` | DMAIC分析出的主因和控制缺口应回写FMEA | 风险假设、现有控制和优先级要同步更新 |
| `MSA` | 测不准就不能进入DMAIC后续分析 | Measure阶段必须先确认量测可信 |
| `SPC` | SPC既是基线识别工具，也是Control阶段守住改善的主工具 | 控制图、能力和异常反应是Control阶段核心 |

典型链路：

`绩效差距识别 -> DMAIC立项 -> MSA/数据分层 -> 主因分析 -> 改善验证 -> Control Plan/SPC/审核固化 -> 经验教训/FMEA回写`

## DMAIC五阶段

| 阶段 | 核心问题 | 典型输出 |
| --- | --- | --- |
| `D` Define | 问题到底是什么，边界和目标是什么 | 项目章程、SIPOC、VOC/CTQ、问题陈述、目标和收益 |
| `M` Measure | 现在到底有多差，数据是否可信 | 基线、MSA、数据定义、分层、过程图和能力现状 |
| `A` Analyze | 真正的主因是什么 | 因果假设、验证分析、显著因子、流出与发生逻辑 |
| `I` Improve | 用什么方案改善，并如何证明有效 | 试验方案、改善措施、对比数据、收益评估 |
| `C` Control | 如何防止回弹 | 控制计划、SPC、审核点、责任人、反应计划 |

## Define阶段

Define最重要的是把“模糊抱怨”变成“可量化问题”。

### 项目章程至少要写清

- 问题对象：哪条线、哪类产品、哪项指标。
- 业务影响：报废、返工、交期、客户投诉、现金损失。
- 范围边界：包含什么，不包含什么。
- 目标：改善到什么水平，何时完成。
- 团队：项目负责人、工艺、质量、设备、生产、财务。

### 常见问题陈述写法

差的写法：

- “电镀很不稳定。”
- “客户总抱怨品质。”

好的写法：

- “2026年Q1，A工厂高多层板孔铜切片不良率为4.8%，高于目标2.0%，主要集中在2号电镀线和夜班批次，导致月均报废成本增加XX。”

### Define常用工具

| 工具 | 作用 |
| --- | --- |
| `SIPOC` | 识别供应商、输入、过程、输出和客户 |
| `VOC/CTQ` | 把客户或内部需求转成关键指标 |
| 项目章程 | 明确问题、目标、范围、收益和团队 |
| 高层流程图 | 确认问题发生在流程的哪一段 |

## Measure阶段

Measure阶段的核心不是收集更多数据，而是收集“可信且可分层”的数据。

### 必做四件事

1. 定义指标：良率、FPY、报废率、缺陷率、周期、Cpk等口径是否一致。
2. 做 `MSA`：量具、程序、复判规则、切片方法是否可信。
3. 做分层：产品、料号、机台、班次、材料批次、人员、时间段。
4. 建基线：知道当前平均值、波动、趋势和损失规模。

### PCB/载板常见基线指标

| 类别 | 指标 |
| --- | --- |
| 质量 | 良率、FPY、报废率、返工率、缺陷密度、客诉PPM |
| 过程 | 电镀厚度、线宽、阻抗、板厚、孔径、翘曲 |
| 运营 | Cycle Time、OEE、停机时间、过杀率、漏判率 |
| 财务 | 单位报废成本、返工成本、降本收益、现金回收 |

### Measure阶段常见错误

- 把不同产品族混成一张报表。
- 没做MSA就直接做回归或能力分析。
- 只拿月平均，不看批次、机台、班次和分布。
- 用最终良率掩盖返工和过杀。

## Analyze阶段

Analyze阶段要回答“主因是什么”，而不是“我们猜什么像主因”。

### 常用分析路径

| 路径 | 用法 |
| --- | --- |
| 数据分层 | 找差异最大的产品、机台、班次、材料或时间段 |
| 因果图/鱼骨图 | 结构化列出人机料法环测假设 |
| `Pareto` | 找出主要缺陷类型或主要损失来源 |
| 假设检验/相关分析/回归 | 验证哪些因子和结果显著相关 |
| 现场复盘 | 用流程、参数、点检、异常记录验证数据线索 |

### PCB/载板分析时特别要分开三类原因

| 原因类型 | 回答什么 |
| --- | --- |
| 发生原因 | 为什么缺陷产生 |
| 流出原因 | 为什么没被发现或拦住 |
| 系统原因 | 为什么长期存在且重复出现 |

很多DMAIC项目只做到“找到了坏机台”，但没有解释为什么检测没拦住、为什么管理系统没触发反应，这样Control阶段很容易回弹。

## Improve阶段

Improve阶段要把“分析结论”转成“被验证过的改进动作”。

### 改善措施常见类型

- 参数窗口优化。
- 设备保养或治具改造。
- 材料切换或批次控制加强。
- 程序和权限锁定。
- 检测规则、限度样、AOI阈值优化。
- 换线/首件/审批机制优化。
- 人员培训和岗位职责重划。

### PCB/载板改善验证重点

- 尽量用对比批、对比机台或前后同口径数据验证。
- 不只看平均值改善，还要看波动有没有收敛。
- 不只看结果指标，还要看上游过程指标是否同步改善。
- 若措施涉及设计、材料、地点、程序或客户批准边界，要同步评估 `ECN/PPAP/客户批准`。

## Control阶段

Control阶段的任务不是“项目结案”，而是“改善结果不反弹”。

### 应固化的内容

| 类别 | 常见动作 |
| --- | --- |
| 文件 | 更新SOP、Control Plan、FMEA、检验规范、参数卡 |
| 数据 | 建立SPC、周报、异常阈值、看板 |
| 责任 | 明确责任人、审核人、升级路径 |
| 现场 | 首件、点检、参数锁定、防错、分层审核 |
| 审核 | 周期复审、Layered Process Audit、项目后评估 |

### Control阶段常见失败

- 改善后不改SOP和Control Plan。
- 没有SPC或预警阈值，只能等异常再发生。
- 责任人不清，换班或换人后回到旧做法。
- 财务收益和质量收益没有持续复盘。

## PCB/载板典型DMAIC项目

| 项目主题 | 典型Y | 常见X |
| --- | --- | --- |
| 电镀孔铜不良率改善 | 孔铜切片不良率、热循环失效率 | 电流密度、槽液、挂具、活化、班次、药水维护 |
| 阻抗波动改善 | 阻抗均值和标准差、Cpk | 线宽、介质厚度、材料批次、压合窗口、TDR程序 |
| AOI过杀率改善 | 过杀率、复判工时、真实流出率 | 程序阈值、限度样、光照、复判规则 |
| 压合翘曲改善 | warpage、板厚偏差、分层率 | 叠构、铜平衡、压合曲线、烘烤、材料吸湿 |
| FPC补强贴合不良改善 | 贴装不良率、补强偏位率 | 治具、贴合参数、材料尺寸稳定性、操作方法 |
| Cycle Time改善 | 总周期、等待时间、在制品周转 | 工程释放、瓶颈机台、复判、切片实验室、排产 |

## DMAIC与8D的边界

| 维度 | `DMAIC` | `8D` |
| --- | --- | --- |
| 主要目的 | 改善长期绩效 | 闭环重大问题和客户投诉 |
| 节奏 | 可按周/月推进 | 通常需要更快响应 |
| 证据 | 强调基线、统计、收益和控制 | 强调遏制、根因、永久措施和客户接受 |
| 输出 | 项目章程、分析、改善验证、控制方案 | 8D报告、遏制、根因、验证、防再发 |

实际工作中，两者可串联：

- 先用 `8D` 处理急性客诉和流出。
- 再用 `DMAIC` 解决背后的系统性波动和长期损失。

## DMAIC与APQP的边界

| 维度 | `DMAIC` | `APQP` |
| --- | --- | --- |
| 主要对象 | 已有过程或成熟产品改善 | 新项目导入和前置策划 |
| 关注点 | 绩效差距、主因、改善和固化 | 需求转化、风险预防、验证和批准 |
| 时间点 | 多发生在量产或爬坡中后期 | 多发生在立项到量产前 |

如果一个问题本质上是“新项目前期没策划好”，单做DMAIC不够，往往还需要回到APQP、FMEA和Control Plan重建前置逻辑。

## DMAIC项目审核常看点

- 问题定义是否定量且边界清楚。
- Measure阶段是否先做MSA和数据分层。
- Analyze阶段是否有证据证明主因，而不是靠经验判断。
- Improve阶段是否做了对比验证和风险评估。
- Control阶段是否改了文件、SPC和责任机制。
- 财务收益是否与质量/运营指标口径一致。

## PCB/载板场景下的常见错误

- 把DMAIC写成更长的8D报告。
- 只改善平均值，不改善波动。
- 用月度汇总数据做主因分析，忽略机台/班次/批次分层。
- 没做MSA就拿切片、AOI、XRF数据做结论。
- 改善后没接入Control Plan、SPC和审核。
- 项目收益只算理论报废节约，未扣除副作用和维持成本。

## 快速落地流程

`选定长期绩效差距 -> 写项目章程 -> 做MSA和基线 -> 分层和主因分析 -> 试点改善 -> 验证平均值与波动 -> 固化到SPC/Control Plan/FMEA -> 定期复审`

使用建议：

- 做良率、报废、周期、过杀和能力改善时，优先先看本页，再回到 `28-spc.md`、`27-msa.md` 和 `31-fmea.md` 补统计与风险接口。
- 做客户重大客诉时，不要直接拿DMAIC替代8D；应先明确是否需要客户正式闭环。
- 做量产爬坡项目时，可把DMAIC作为APQP第四、五阶段后的持续改善方法使用。
