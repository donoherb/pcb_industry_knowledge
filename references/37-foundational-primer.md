# PCB/载板基础知识补强

## 定位

本文是PCB与IC载板知识库的基础入口，用于帮助新人、研究员、质量工程师、采购、投融资人员和项目团队快速建立共同语言。它不替代 `01-industry-chain.md`、`02-products-tech.md`、`03-materials-and-equipment.md` 和 `04-process-quality.md`，而是把这些文件中的基础概念串成一条可复用的理解路径。

本文只沉淀稳定知识。市场规模、公司份额、产能、报价、客户名单、认证状态和标准最新版本等动态事实，仍按 `07-source-standards.md` 核验后记录。

## 一句话理解PCB和IC载板

PCB是电子系统里的“板级互连平台”：它把芯片、被动元件、连接器、模组、电源和信号路径组织在一起，使整机能够稳定工作。

IC载板是封装里的“芯片级互连平台”：它把裸芯片或芯粒的细小I/O扇出到更适合PCB装配的焊球、焊盘或封装引脚，同时承受封装过程中的热、机械和电气要求。

两者都做互连，但服务层级不同：

| 维度 | PCB | IC载板 |
| --- | --- | --- |
| 系统位置 | 整机、模组或系统板级 | 芯片封装内部 |
| 主要客户链 | 整机厂、EMS、模组厂、系统厂 | 封测厂、IDM、Fabless供应链 |
| 核心挑战 | 成本、交付、可靠性、复杂设计量产 | 细线路、翘曲、洁净度、封装匹配、良率 |
| 判断证据 | 产品结构、量产良率、客户认证、交付表现 | 封装客户认证、strip级良率、warpage、可靠性 |
| 常见误区 | 把层数直接等同于价值量 | 把ABF载板等同于全部载板 |

## 入门学习路径

建议按“对象 -> 产品 -> 制程 -> 质量 -> 商业”的顺序学习，而不是一开始就背术语。

| 阶段 | 先解决的问题 | 推荐阅读 |
| --- | --- | --- |
| 1. 建立边界 | 讨论的是PCB、IC载板、SLP还是中介层 | `00-index.md`、本文 |
| 2. 认识产品 | 板种、层数、材料、孔和线路为什么重要 | `02-products-tech.md` |
| 3. 看懂制造 | 图纸如何变成可出货的产品 | `04-process-quality.md` |
| 4. 连接材料设备 | 材料和设备如何限制性能、良率和成本 | `03-materials-and-equipment.md` |
| 5. 判断公司和项目 | 能力、订单、毛利和扩产风险如何连起来 | `01-industry-chain.md`、`05-financial-and-company-analysis.md`、`09-operations-cost-and-delisking.md` |
| 6. 处理专项问题 | 高速、汽车、FPC、ABF/BT、质量工具等 | 对应专题文件 |

如果只做快速入门，至少先回答六个问题：做什么板、给谁用、关键结构是什么、难点在哪道工序、客户如何认证、盈利是否能覆盖良率和折旧压力。

## 三张地图

分析PCB/载板时，最有用的基础框架是三张地图：产品地图、制造地图和经营地图。

| 地图 | 核心问题 | 典型输出 |
| --- | --- | --- |
| 产品地图 | 这是哪类板，服务什么功能 | 板种、层数、线宽线距、孔结构、材料、表面处理 |
| 制造地图 | 关键工序和瓶颈在哪里 | 流程图、特殊工序、控制点、检测点、返工/报废路径 |
| 经营地图 | 技术复杂度能否转成盈利质量 | ASP、材料成本、折旧、良率、稼动率、客户认证和交期 |

很多错误判断来自只看其中一张地图。例如只看产品地图会高估“能做样品”的意义；只看经营地图会忽略良率爬坡；只看制造地图则可能看不清客户和需求周期。

## 产品分类的四种口径

同一块板可以同时按结构、应用、材料和制程分类。分析时先说明口径，否则容易把不同维度混成一个结论。

| 分类口径 | 常见表达 | 主要用途 | 易错点 |
| --- | --- | --- | --- |
| 按结构 | 单双面、多层、高多层、HDI、FPC、软硬结合 | 判断互连密度和制造难度 | 层数高不必然等于高利润 |
| 按应用 | 汽车、服务器、通信、消费、工控、医疗 | 判断客户要求、认证周期和需求周期 | 同一应用内产品差异很大 |
| 按材料 | FR-4、高Tg、低损耗、PTFE、PI、BT、ABF | 判断电气、热和可靠性边界 | 参数更高不代表一定更适合 |
| 按制程 | 减成法、HDI、mSAP/SAP、build-up | 判断线路精度、设备和良率难度 | 样品能力不能直接代表量产能力 |

口径说明建议写成一句话：本文讨论的是“用于AI服务器交换机的高层数低损耗刚性PCB”，还是“用于高性能处理器封装的ABF FC-BGA载板”。前者仍在PCB板级互连体系内，后者属于封装基板体系。

## 结构基础

PCB/载板结构可以从五个元素理解：导体、介质、孔、表面和外形。

| 元素 | 代表内容 | 影响什么 |
| --- | --- | --- |
| 导体 | 铜箔、线路、焊盘、铜厚 | 电流、阻抗、损耗、发热、蚀刻难度 |
| 介质 | core、PP、树脂、玻纤、ABF/BT/PI等 | 绝缘、Dk/Df、CTE、厚度、翘曲和可靠性 |
| 孔 | 通孔、盲孔、埋孔、微孔、背钻孔 | 层间连接、布线密度、可靠性和钻孔/电镀难度 |
| 表面 | 阻焊、字符、ENIG/OSP/沉银/沉锡等 | 焊接、打线、接触、保存期和外观判定 |
| 外形 | 板厚、尺寸、公差、拼板、工艺边 | 装配适配、成型良率、利用率和运输包装 |

把这些元素连起来看，才能理解为什么一个看似简单的设计变更会影响多个工序。例如线宽变细会影响曝光、显影、蚀刻、AOI、阻抗和良率；板厚变薄会影响压合、翘曲、成型、包装和客户装配。

## 从图纸到出货

制造不是把Gerber直接“印”成板，而是把客户设计资料转成受控流程。

| 阶段 | 关键输入 | 关键输出 | 典型风险 |
| --- | --- | --- | --- |
| 工程评审 | Gerber/ODB++/IPC-2581、钻孔、叠构、规范、BOM | DFM问题、MI、流程路线、特殊特性 | 资料版本混用、隐含要求未识别 |
| 材料准备 | CCL、PP、铜箔、油墨、药水、表面处理材料 | 批次、CoA/CoC、来料检验和储存条件 | 材料替代未批准、吸湿和批次波动 |
| 制造加工 | 内层、压合、钻孔、电镀、外层、阻焊、表面处理、成型 | 在制品、过程参数、检验记录 | 工序窗口漂移、返工路径不受控 |
| 检测验证 | AOI/AVI、ET、切片、阻抗、XRF、可靠性测试 | 合格判定、异常隔离、放行证据 | 漏检、误判、测量系统不可信 |
| 包装出货 | 标签、批次、包装规范、湿敏/防静电/防潮要求 | 出货批、追溯记录、客户收货状态 | 混料、标签错误、运输和储存失效 |

工程资料、程序版本、检验数据和出货标签必须能互相追溯。若只能看到最终合格证，而不能追到材料批次、设备、班次、panel位置和检验记录，后续客诉和失效分析会非常被动。

## 技术难度来自哪里

PCB/载板技术难度通常不是单一指标，而是多个因素叠加。

| 难度来源 | 典型表现 | 需要关注的证据 |
| --- | --- | --- |
| 几何密度 | 细线宽线距、小孔径、微孔、叠孔、via-in-pad | 量产设计规则、缺陷密度、AOI/AVI能力 |
| 尺寸和层间控制 | 高层数、大板、薄板、大尺寸载板 | 层偏、涨缩补偿、翘曲、共面性 |
| 电气性能 | 阻抗、损耗、回损、串扰、电源完整性 | coupon、TDR/VNA、材料批次和stack-up控制 |
| 热和机械可靠性 | 热循环、热冲击、弯折、CAF、分层 | DVP&R、切片、热应力、HAST/CAF数据 |
| 制程窗口 | 电镀均匀性、蚀刻补偿、表面处理、洁净度 | SPC、过程能力、药水分析、设备维护 |
| 客户和认证 | 汽车、服务器、封装、医疗等客户要求 | APQP/PPAP、客户审核、年度验证和变更批准 |

因此，技术能力判断应尽量使用“客户认证 + 量产订单 + 稳定良率 + 可靠性数据 + 追溯证据”的组合，而不是只引用宣传中的最小线宽线距或最高层数。

## 材料理解的基本逻辑

材料选择要服务于应用需求。常见逻辑如下：

| 应用需求 | 材料侧关注 | 制程和质量连带影响 |
| --- | --- | --- |
| 高速低损耗 | Dk/Df、铜箔粗糙度、介质厚度一致性 | 阻抗调试、压合窗口、TDR/VNA验证 |
| 高可靠耐热 | Tg、Td、CTE、CAF、剥离强度 | 热循环、焊锡耐热、湿热偏压、长期验证 |
| 动态弯折 | PI/FCCL、压延铜、胶系、覆盖膜 | 弯折半径、补强、冲切、装配应力 |
| 大电流散热 | 铜厚、金属基、绝缘层热阻 | 蚀刻难度、热阻测试、绝缘耐压 |
| 封装载板 | ABF/BT、吸湿、尺寸稳定、洁净度 | warpage、微孔、细线路、封装良率 |

过度选材也会带来问题：成本上升、加工窗口变窄、良率下降、供应商受限和客户不愿付费。材料评审必须同时看性能、认证、加工性、供应安全和总成本。

## 质量与可靠性的基本区别

质量和可靠性都重要，但问题不同。

| 概念 | 关注点 | 典型问题 |
| --- | --- | --- |
| 质量 | 产品是否满足当下规定要求 | 外观、尺寸、开短路、镀层厚度、阻抗是否合格 |
| 可靠性 | 产品在使用环境和寿命内是否持续满足功能 | 热循环后孔铜裂纹、湿热偏压下CAF、弯折疲劳、封装翘曲 |
| 过程能力 | 制程是否稳定并有余量 | Cpk/Ppk、SPC趋势、批次/设备/班次差异 |
| 测量可信度 | 数据是否能用于判断 | MSA、GRR、属性一致性、偏倚、稳定性 |

终检合格不等于可靠性充分；可靠性样品通过也不等于量产过程稳定。高阶产品要把过程控制、测量系统、可靠性验证和客户场景联动起来看。

## 读资料的最小检查清单

阅读公司材料、客户资料、尽调材料或行业报告时，先抓以下字段。

| 字段 | 为什么重要 |
| --- | --- |
| 产品边界 | PCB、FPC、HDI、SLP、BT、ABF、IC载板不能混用 |
| 应用和客户 | 决定认证周期、价格压力、质量要求和需求周期 |
| 技术指标 | 层数、线宽线距、孔径、材料、表面处理、可靠性条件 |
| 量产状态 | 样品、小批、认证中、量产供货和规模收入含义不同 |
| 良率和稼动率 | 决定技术能否转化为收入和毛利 |
| 产能口径 | 设计产能、有效产能、投产产能、合格产出必须区分 |
| 资本开支 | 设备到厂、调试、认证、折旧和爬坡会影响利润节奏 |
| 变更和认证 | 材料、地点、设备、程序、供应商变化可能触发客户批准 |
| 来源日期 | 动态事实必须核验日期和口径 |

## 常见误判

- 把“可以打样”理解为“可以稳定量产”。
- 把“设备到厂”理解为“产能释放”。
- 把“高层数、细线路、ABF”等标签直接理解为高毛利。
- 把PCB、SLP、IC载板、RDL和中介层混成一条连续赛道。
- 只看最终良率，不看FPY、返工、报废、客户退货和质量成本。
- 用单批可靠性通过结论替代长期过程能力。
- 忽略客户指定材料、指定供应商和变更批准对成本和交付的约束。
- 把公开宣传中的最优指标当成所有产品线的平均能力。

## 基础问题模板

做任何基础研究或项目评审，可先用以下问题开场：

1. 这个产品处在PCB、IC载板还是先进封装的哪一层。
2. 客户真正购买的是面积、层数、可靠性、低损耗、交期还是认证稳定性。
3. 关键材料是否由客户指定，是否有替代和第二来源。
4. 最难的工序是什么，瓶颈是设备、材料、工艺窗口、测量还是人员经验。
5. 量产证据来自样品、试产、客户认证、出货收入还是持续交付记录。
6. 当前良率、FPY、返工、报废和客诉是否支撑目标毛利。
7. 如果材料、设备、地点、程序或供应商变化，是否需要客户批准或再PPAP。
8. 哪些结论是稳定知识，哪些结论必须按日期和来源重新核验。
