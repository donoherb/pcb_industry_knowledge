本文沉淀PCB/载板行业供应商质量管理、来料控制和SCAR闭环的稳定知识，不替代客户质量协议、CSR、采购合同、材料规格书、UL/环保声明、外协特殊过程批准文件和企业QMS程序。涉及供应商变更、二级供应商、外协地点、材料牌号和客户批准状态时，必须以客户文件和书面批准为准。

## 一句话理解

供应商质量不是采购部门的价格管理，而是把外部材料、外协过程和服务输入纳入APQP、Control Plan、变更、追溯和8D闭环，使来料波动不会直接变成制程报废、客户客诉或未经批准的量产变更。

## 适用范围

PCB/载板供应商质量至少覆盖：

- 关键材料：CCL、PP、铜箔、ABF/BT、PI、覆盖膜、补强、油墨、干膜、阻焊、镍金/银/锡材料。
- 化学品和耗材：棕化/黑化、沉铜、电镀、蚀刻、退膜、OSP、清洗药水、钻咀、铣刀、砂带、过滤芯。
- 外协过程：表面处理、特殊检测、可靠性实验室、外形加工、局部电镀、特殊涂覆、包装或物流。
- 设备和治具相关输入：电测治具、飞针程序、AOI/AVI算法参数、夹治具、模具、校准服务。
- 客户指定或客户批准材料：AVL/ASL中的材料牌号、供应商、二级供应商和生产地点。

## 与APQP、PPAP、Control Plan、8D和ECN的关系

| 工具 | 供应商质量接口 | 容易漏掉的点 |
| --- | --- | --- |
| APQP | 关键材料和外协过程在项目阶段完成风险评估、验证和供应商APQP | 等量产异常后才要求供应商补资料 |
| PPAP | 客户要求时，材料证据、供应商批准状态、外协过程能力和变更记录要进入提交包或现场备查 | 只提交厂内资料，忽略外协和二级供应商 |
| Control Plan | 来料检验、供应商CoA/CoC、批次追溯、反应计划和加严/放宽规则要可执行 | IQC项目和控制计划不一致 |
| 8D | 供应商异常要同时处理发生原因、流出原因和系统原因 | 照收供应商8D，不验证措施有效性 |
| ECN/ECR | 材料、供应商、产地、配方、外协地点和检验方法变化要评估客户批准与再PPAP | 把供应商变更当作内部采购替代 |
| 不合格品管理 | 来料异常、在制发现的材料异常和客户退货都要隔离并追溯受影响批次 | 未锁定已用物料和在制/成品库存 |

## 供应商分级

供应商分级要基于产品风险，而不是只看采购金额。

| 等级 | 典型对象 | 管理重点 |
| --- | --- | --- |
| A类关键供应商 | CCL/PP、铜箔、ABF/BT、关键药水、表面处理外协、客户指定材料 | 准入审核、APQP、变更批准、周期审核、批次追溯、加严/恢复规则 |
| B类重要供应商 | 油墨、干膜、钻咀、铣刀、测试治具、包装材料、校准服务 | 来料验证、性能趋势、异常闭环、年度评价 |
| C类一般供应商 | 低风险辅材、通用耗材、非产品接触类服务 | 基础资质、交付稳定性、批次记录 |

判定等级时至少考虑：

- 失效后果：是否影响安全、可靠性、客户装配、长期寿命或召回风险。
- 可替代性：是否客户指定、是否需要长周期认证、是否存在合格二供。
- 过程敏感性：材料小波动是否会放大为阻抗、翘曲、孔铜、分层、CAF或表面处理异常。
- 检出能力：厂内IQC是否能及时发现问题，还是只有制程/客户端才暴露。
- 历史表现：PPM、重复异常、8D准时率、变更纪律和交付稳定性。

## 生命周期管理

推荐把供应商管理拆成八个阶段：

1. 需求定义：明确材料/外协规格、客户要求、法规环保、追溯和变更边界。
2. 寻源与预评估：确认技术能力、体系资质、产能、财务稳定性和客户指定状态。
3. 准入审核：按风险执行文件审核、现场审核、过程审核或样品验证。
4. 样品和工程验证：在目标产品、目标工艺窗口和目标客户要求下验证，而不是只看供应商样册。
5. 供应商APQP/PPAP：对高风险材料和外协过程确认PFMEA、Control Plan、MSA、SPC、追溯和变更控制。
6. 量产监控：用IQC、CoA/CoC、过程表现、客户反馈和供应商KPI持续管理。
7. 异常闭环：来料、制程和客户端异常按NCM/MRB/SCAR/8D联动处理。
8. 变更与退出：任何影响批准状态的变更都走ECR/ECN和客户批准评估；长期不达标供应商降级或退出。

## 准入资料清单

供应商准入资料应按风险裁剪，常见清单包括：

| 类别 | 典型资料 | 审核要点 |
| --- | --- | --- |
| 体系资质 | ISO/IATF证书、范围、有效期、最近审核问题 | 证书范围是否覆盖供货产品和工厂 |
| 产品规格 | TDS、材料规格书、图纸、关键参数范围 | 是否与客户图纸、IPC/企业标准和内部MI一致 |
| 合规资料 | RoHS、REACH、UL黄卡、禁用物质声明、MSDS/SDS | 版本、适用料号、生产地点和有效期 |
| 过程能力 | 工艺流程、关键参数、能力数据、良率趋势 | 数据是否来自目标产品和量产条件 |
| 测量系统 | 检验方法、量具校准、MSA或实验室比对 | 与厂内和客户测量方法是否相关 |
| 追溯 | 批次编码、生产日期、原料批次、CoA/CoC | 能否从成品追到关键原料和生产线 |
| 变更控制 | 变更通知规则、提前期、客户批准路径 | 是否禁止未批准的配方、地点和二供变化 |
| 应急能力 | 二供、库存策略、产能弹性、灾备计划 | 断供时是否会迫使未经验证替代 |

## 材料和外协关注点

| 对象 | 关键风险 | 建议控制 |
| --- | --- | --- |
| CCL/PP | Tg、Td、Dk/Df、流胶、含胶量、铜箔粗糙度、尺寸稳定、吸湿 | AVL控制、批次CoA、压合窗口验证、阻抗和可靠性复核 |
| 铜箔 | 粗糙度、剥离强度、针孔、延展性、厚度均匀性 | 供应商批次趋势、关键产品首批验证、异常批隔离 |
| ABF/BT | 储存条件、吸湿、颗粒、尺寸稳定、warpage、细线路适配 | 温湿度/有效期控制、洁净包装、strip级追溯、客户批准 |
| 油墨/干膜/阻焊 | 黏度、曝光显影窗口、附着力、离子残留、耐化学性 | 保存期限、上线试用、可靠性和外观限度样 |
| 药水 | 浓度、添加剂、污染、寿命、供应批差 | CoA、厂内分析、槽液SPC、批次切换验证 |
| 钻咀/铣刀 | 寿命、偏摆、涂层、尺寸稳定、批差 | 首件、寿命曲线、断针/毛刺趋势、供应商批号追溯 |
| 表面处理外协 | 镀层厚度、孔隙、可焊性、镍腐蚀、污染、混批 | 外协Control Plan、XRF/MSA、可焊性验证、批次隔离 |
| 电测/治具 | 接触可靠性、程序版本、探针磨损、漏测风险 | 程序版本控制、治具点检、GRR/相关性、失效样反查 |
| 包装物流 | 防潮、防氧化、ESD、标签、运输损伤 | 包装验证、标签扫描、湿度指示、运输条件记录 |

## IQC策略

IQC的目标不是把所有供应商问题都检出来，而是用风险分级和趋势监控阻断高风险输入。对关键材料，IQC应与过程表现、客户反馈和供应商KPI联动。

常见IQC策略：

- 资料确认：CoA/CoC、批号、有效期、生产日期、供应商、客户批准状态。
- 外观和包装确认：破损、受潮、污染、标签、混料、储存条件。
- 关键参数验证：按材料类型抽测厚度、树脂含量、铜厚、Dk/Df、黏度、浓度、镀层、尺寸或可焊性。
- 上线首批观察：新供应商、新批次、新配方或异常恢复后，用制程数据验证。
- 趋势监控：把来料数据与压合、钻孔、电镀、阻抗、翘曲、外观和可靠性结果关联。

IQC不应替代供应商过程控制。若缺陷只能在压合、封装、可靠性或客户装配后暴露，仅靠入厂抽样是不够的，应提升为供应商过程审核、APQP验证或客户批准评估。

## CoA、CoC与厂内验证

| 文件 | 用途 | 不能替代什么 |
| --- | --- | --- |
| CoA | 记录供应商对该批材料关键参数的分析结果 | 厂内关键参数验证、客户要求的独立测试 |
| CoC | 声明产品符合约定规格或标准 | 实物一致性确认、追溯和变更批准 |
| SDS/MSDS | 描述化学品安全和储存处置要求 | 产品性能验证和工艺适配 |
| UL/环保声明 | 支撑法规和认证要求 | 当前批次质量、客户批准状态 |

审核CoA/CoC时，要看料号、批号、规格、方法、单位、上下限、签核、日期和供应商工厂是否一致。只有“合格”两个字而没有参数的文件，对工程分析帮助有限。

## 供应商APQP

对A类关键供应商，建议按项目风险要求供应商提交APQP或等效证据：

| 项目 | 关注点 |
| --- | --- |
| 要求传递 | 客户CSR、特殊特性、环保、包装、标签、追溯和变更规则是否传到供应商 |
| 过程流程 | 供应商自己的流程是否覆盖关键工序、检验点、返工和隔离 |
| PFMEA | 是否识别批次混料、配方错误、设备漂移、污染、储存失效、测量误差 |
| Control Plan | 关键参数、频率、反应计划和记录是否明确 |
| MSA/相关性 | 供应商、厂内和客户测量方法是否能对齐 |
| 试产验证 | 是否在目标批量、目标设备和目标规格下验证 |
| 变更控制 | 配方、原料、地点、设备、检验方法和二级供应商变化如何通知 |
| 追溯 | 异常时能否锁定受影响原料、半成品、成品和已交付批次 |

## SCAR触发条件

SCAR（Supplier Corrective Action Request）适用于需要供应商正式纠正和预防再发的异常。常见触发包括：

- 来料批量不合格、混料、标签错误或CoA/实物不一致。
- 制程中确认与供应商材料或外协过程有关的重大异常。
- 客户端问题追溯到供应商输入。
- 重复发生同类来料异常。
- 供应商未经批准变更材料、配方、产地、二级供应商或检验方法。
- 供应商8D延迟、措施无效或拒绝提供必要证据。

不需要所有轻微异常都开SCAR。低风险、一次性、厂内可快速纠正的问题，可通过NCR、退换货、供应商通知或月度绩效处理；但重复、系统性、高风险和客户影响问题应升级。

## SCAR闭环路径

推荐路径：

`异常发现 -> 隔离与风险评估 -> 通知采购/SQE/供应商 -> 开立SCAR -> 供应商遏制 -> 根因分析 -> 永久措施 -> 效果验证 -> 加严/恢复 -> 经验回写和供应商评价`

每一步至少要回答：

| 阶段 | 关键问题 |
| --- | --- |
| 隔离 | 可疑批次、已使用物料、在制品、成品和已出货范围是否锁定 |
| 风险评估 | 是否影响客户、可靠性、法规、交付或批准状态 |
| 供应商遏制 | 供应商库存、在途、已交付和厂内未发货品是否同步处理 |
| 根因 | 是否区分发生原因、流出原因和系统原因 |
| 永久措施 | 是修补个案，还是改变过程、检验、培训、设备或变更规则 |
| 验证 | 是否用后续批次、过程数据、可靠性或客户反馈证明有效 |
| 恢复 | 加严检验何时解除，依据是什么 |
| 回写 | AVL、IQC计划、Control Plan、PFMEA、采购协议和经验教训是否更新 |

## 供应商8D审核要点

审核供应商8D时，不要只看格式完整。重点看逻辑是否成立：

- D2问题描述是否包含料号、批次、数量、失效模式、规格、发现地点和证据。
- D3遏制是否覆盖供应商端、在途、厂内库存、已用物料、成品和客户风险。
- D4根因是否有证据支撑，不能只写“人员疏忽”或“管理不到位”。
- D5/D6措施是否能消除根因，并有责任人、完成日期和验证方法。
- D7防再发是否更新PFMEA、Control Plan、SOP、培训、审核清单或变更流程。
- 效果验证是否覆盖足够批次和时间，而不是单批复测合格。

## 加严、放宽和恢复规则

建议提前定义来料检验状态：

| 状态 | 触发 | 典型动作 |
| --- | --- | --- |
| 正常 | 供应商稳定，历史表现满足要求 | 按控制计划和IQC计划执行 |
| 加严 | 重大异常、重复异常、新供应商、新材料、新工厂、异常恢复期 | 增加抽样、全检关键项、上线首批确认、供应商驻厂或过程数据提交 |
| 暂停 | 高风险不合格、未经批准变更、供应商拒绝闭环、客户风险不可接受 | 停止使用、隔离库存、启动二供或客户沟通 |
| 恢复 | 连续批次合格、措施验证有效、审核问题关闭 | 按批准的恢复条件逐步回到正常检验 |

恢复条件应写清批次数、数量、时间、关键参数、责任人和批准人。不能因为交付压力自动解除加严。

## 供应商绩效看板

供应商评价建议同时看质量、交付、响应和纪律：

| 维度 | 典型指标 |
| --- | --- |
| 质量 | 来料合格率、供应商PPM、重复异常率、重大异常数、批次稳定性 |
| 响应 | 8D准时率、临时遏制时效、永久措施完成率、资料提交及时性 |
| 交付 | OTD、缺料次数、急单响应、最小起订量和长周期材料影响 |
| 变更 | 变更提前通知率、未经批准变更次数、客户批准资料完整性 |
| 成本 | 质量成本、退换货成本、停线/重排产影响、呆滞材料责任 |
| 合作 | 技术支持、现场审核配合、数据透明度、持续改善项目 |

供应商评分的目的不是做年度排名，而是驱动分级管理：保留、辅导、加严、限制新项目、二供导入或退出。

## 追溯要求

关键材料和外协过程至少要能追到：

- 供应商名称、工厂、材料牌号、批号、生产日期、有效期。
- 内部收货批号、检验记录、储存条件和领用记录。
- 使用到的工单、lot、panel、strip、客户料号和出货批次。
- 供应商CoA/CoC、IQC结果、异常记录和放行批准。
- 发生变更时的ECR/ECN、客户批准、切换点和旧料消耗计划。

高端服务器板、汽车板和IC载板项目，追溯粒度通常需要比普通PCB更细。是否要追到strip、位置、设备、班次和药水槽，应按客户要求和风险决定。

## 供应商审核常看点

供应商现场或文件审核通常会看：

- 供应商是否理解传递下来的客户CSR和特殊特性。
- 生产流程、Control Plan、检验规范和实际记录是否一致。
- 关键设备、量具、实验室和人员资质是否受控。
- 批次、混料、标签、储存、有效期和防污染措施是否可靠。
- 异常品隔离、返工、报废和让步流程是否清楚。
- 未经批准变更如何被预防，历史变更是否有证据。
- 供应商自己的二级供应商是否受控。
- 8D和SCAR是否真的关闭并验证有效。

## PCB/载板场景下的常见错误

- 只按价格和交期导入供应商，没有做材料/外协风险分级。
- 以供应商CoA替代厂内关键参数验证。
- 客户指定材料发生供应商、地点或配方变化时，没有评估客户批准和再PPAP。
- 来料检验只看外观，不看与制程和可靠性相关的关键参数。
- 供应商8D写得完整，但厂内没有验证后续批次是否真的改善。
- 加严检验没有退出准则，最后变成长期人工筛选。
- 外协特殊过程没有纳入Control Plan、追溯和审核。
- 供应商绩效只看PPM，不看未经批准变更、响应速度和重复问题。

## 快速落地流程

`供应商分级 -> 准入资料和样品验证 -> 供应商APQP/PPAP评估 -> IQC计划和Control Plan联动 -> 量产KPI监控 -> SCAR/8D闭环 -> 加严/恢复 -> ECN和客户批准管理 -> 年度评价和二供策略`
