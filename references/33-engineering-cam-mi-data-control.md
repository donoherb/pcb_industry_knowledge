本文沉淀PCB/载板从客户设计资料到制造资料放行的稳定知识，不替代客户图纸、工程规范、IPC/客户设计规则、企业工程变更流程、PLM/MES/CAM系统权限规则和客户保密要求。涉及客户设计更改、批准版本、受控输出、软件脚本和客户数据安全时，应以客户文件、企业QMS和书面授权为准。

## 一句话理解

CAM/MI不是把客户文件转成生产文件的文员动作，而是把设计意图、客户要求、工艺能力、测试规则、包装追溯和版本状态转成车间可执行、可追溯、可审核的制造数据。

## 适用范围

本专题覆盖：

- 客户输入资料：Gerber、ODB++、IPC-2581、钻孔文件、netlist、stack-up、图纸、规格、BOM、客户CSR和包装标签要求。
- 工程输出资料：MI、CAM data、拼板、钻孔程序、曝光资料、AOI/AVI程序、电测资料、阻抗coupon、切片位、包装规范。
- 工程放行：DFM评审、资料完整性检查、版本冻结、首件确认、试产反馈和量产发布。
- 数据变更：客户ECN、内部工程优化、程序修订、设备转移、外协数据和旧版退出。
- 数据安全：客户资料权限、加密传输、版本备份、作废控制和外发审批。

## 常见输入资料

| 资料 | 用途 | 风险点 |
| --- | --- | --- |
| Gerber | 图形层、阻焊、字符和外形数据 | 层名不清、单位/零位错误、缺少属性 |
| ODB++ / IPC-2581 | 更完整的制造数据包 | 导入设置、软件兼容和版本解释差异 |
| NC drill | 孔径、孔位、PTH/NPTH和钻孔信息 | 孔表与图纸不一致、控深/背钻信息缺失 |
| Netlist | 电气网络关系 | 客户netlist、CAM提取netlist和电测程序不一致 |
| Stack-up | 层序、材料、介厚、铜厚和阻抗 | 目标阻抗与制造公差不匹配 |
| 图纸/规格 | 尺寸、公差、表面处理、验收标准 | 图纸版本、局部注释和客户补充规范遗漏 |
| 客户特殊要求 | 包装、标签、追溯、PPAP、变更和年度验证 | CSR未转成内部MI和控制计划 |

输入资料缺失时，工程不能靠默认经验直接补齐关键要求。必须把假设、客户确认和内部批准记录留痕。

## 工程资料完整性检查

接单、报价、开案或试产前，至少检查：

- 客户料号、版本、修订历史、单位、坐标零点和层数。
- 层名、层序、正负片、铜层、阻焊、字符、外形和机械层是否清楚。
- 钻孔表、孔属性、孔径公差、PTH/NPTH、控深钻、背钻和树脂塞孔要求。
- 表面处理、镀层厚度、可焊性/打线/接触要求和保存期限。
- 阻抗网络、目标值、公差、coupon位置和测试方法。
- 外观验收标准、IPC等级、客户限度样、可靠性和年度验证要求。
- 包装、标签、条码、追溯、RoHS/UL和客户系统上传要求。
- 是否存在客户指定材料、指定供应商、外协特殊过程或禁用物质。

资料完整性检查应输出问题清单和客户确认状态，而不只是邮件里问过。

## DFM到MI的转换

DFM发现的是设计与制造窗口的冲突；MI要把已批准的制造方法落到现场。

| DFM关注 | MI必须写清 |
| --- | --- |
| 最小线宽线距、蚀刻补偿 | 目标线宽、补偿规则、首件和SPC项目 |
| 孔径、环宽、纵横比 | 钻孔程序、钻咀、叠板数、寿命和孔检要求 |
| 叠构和压合 | 材料牌号、介质、铜厚、压合程序和方向 |
| 阻抗 | 目标、公差、coupon、TDR方法和反应计划 |
| 阻焊桥和开窗 | 曝光显影窗口、限度样和外观判定 |
| 表面处理 | 药水/外协、镀层窗口、XRF测点和可焊性要求 |
| 成型 | 程序、治具、尺寸基准、毛刺和板边保护 |
| 包装追溯 | 标签字段、扫码、防潮、箱装规则和出货检查 |

如果DFM问题需要客户改图或批准偏离，不能只在MI里暗改制造补偿。

## MI最低字段

MI建议至少包含：

- 基本信息：客户、料号、版本、内部料号、工单类型、产品族。
- 材料和叠构：供应商/牌号、PP组合、铜厚、板厚、阻抗结构。
- 工艺路线：每道工序、外协工序、返工路径和禁止事项。
- 关键参数：钻孔、压合、电镀、曝光、蚀刻、阻焊、表面处理和成型窗口。
- 检验要求：首件、巡检、SPC、TDR、XRF、切片、电测、外观和可靠性。
- 特殊特性：客户指定、功能关键、可靠性关键和过程薄弱项。
- 包装标签：客户标签、批次规则、防潮、ESD、数量和箱托盘关系。
- 反应计划：异常隔离、升级、停线、MRB、客户通知和ECN触发。
- 版本和批准：编制、审核、批准、生效日期、适用范围和作废旧版。

## CAM输出控制

CAM输出要受控到程序和数据层级：

| 输出 | 控制重点 |
| --- | --- |
| 拼板/panelization | 拼板利用率、工艺边、fiducial、coupon、条码和客户限制 |
| 钻孔程序 | 孔径映射、钻序、叠板数、控深/背钻、寿命和防呆 |
| 曝光/LDI资料 | 层别、极性、补偿、对位靶、版本和设备适配 |
| AOI/AVI程序 | 判定规则、过杀/漏检风险、限度样和程序版本 |
| 电测程序 | netlist来源、开短路阈值、治具版本和复测规则 |
| 阻抗coupon | 位置、结构代表性、数量、测试方法和客户批准 |
| 成型程序 | 外形、槽、V-cut、倒角、补偿和治具匹配 |
| 包装标签模板 | 字段来源、条码规则、客户模板和打印权限 |

任何CAM输出都应能追到客户输入版本、内部MI版本、创建人、审核人和生效点。

## 版本控制

工程资料版本控制要回答四个问题：

1. 当前批准版本是什么。
2. 哪些批次按旧版生产，哪些按新版生产。
3. 旧版资料、程序、标签和半成品如何退出。
4. 客户是否批准该变更，是否触发再PPAP或偏差。

建议控制点：

- 客户版本、内部版本、MI版本、CAM程序版本和MES/BOM版本要有关联。
- 现场只允许访问当前有效版本；旧版应作废、归档或权限隔离。
- 临时偏差和工程试验版本必须有有效期、范围和退出规则。
- 程序补打、复制、外发和手工修改要有权限和记录。
- ECN后首批要做版本一致性确认。

## 首件和工程放行

新料号、新版本、新工艺或程序修订后，首件确认应覆盖：

- 实物与MI、图纸、CAM输出、程序和标签版本一致。
- 关键尺寸、孔、线宽线距、阻抗、镀层、外观和电测满足要求。
- 首件样品、报告、图片和原始数据可追溯。
- 异常已通过工程、质量、制造和客户批准闭环。
- 量产前的SOP、Control Plan、检验规范和包装规范已同步。

首件合格不能替代过程能力。若关键特性要求能力或SPC，应按量产条件继续验证。

## 外协和多工厂数据

当资料发往外协厂或多工厂生产时，风险会放大：

- 外协是否使用同一版本MI、图纸、程序、检验标准和客户要求。
- 数据外发是否经过客户授权和保密审批。
- 外协程序修改是否允许，修改后如何回传和批准。
- 多工厂设备能力、补偿规则、测试方法和标签系统是否一致。
- 客诉时能否追到具体工厂、设备、程序和外协批次。

客户禁止外协或限制二级供应商时，必须在工程路线和采购流程中防错。

## 数据安全和备份

客户资料通常具有保密要求。基本控制包括：

- 客户资料按项目、客户和权限分级存放。
- 外发资料加密、审批、记录接收方和用途。
- 禁止用个人网盘、私人邮箱或非授权设备传递客户数据。
- CAM脚本、补偿规则和程序库备份并做权限控制。
- 作废版本保留用于追溯，但不允许现场误用。
- 员工离职、供应商退出和项目结束时回收访问权限。

## 审核常看点

客户和内部审核常抽一个料号检查：

- 客户输入、MI、CAM输出、Control Plan和现场记录是否版本一致。
- 工程问题清单、客户确认和DFM回复是否完整。
- 程序修订、补偿变化和ECN是否经过批准。
- 现场是否能访问旧版资料或使用未批准程序。
- 电测、AOI、阻抗、成型和标签程序是否有版本控制。
- 资料外发、外协加工和客户数据安全是否受控。
- 客诉后是否追溯到当时使用的工程资料版本。

## 常见错误

- 报价阶段DFM发现风险，但量产MI没有对应控制。
- 客户图纸版本更新，CAM程序和标签模板没有同步。
- 工程人员口头指定特殊处理，现场没有受控文件。
- 多工厂使用不同补偿规则，客户以为是同一批准过程。
- 电测程序由历史资料复制，未与最新netlist比对。
- 客户确认散落在邮件中，没有进入受控工程资料。
- 临时工程试验版本被现场长期使用。
- 客户资料外发给外协但没有授权、版本和保密记录。

## 快速落地流程

`接收客户资料 -> 完整性检查 -> DFM/DFX评审 -> 客户问题确认 -> CAM转换和MI编制 -> 跨部门审核 -> 首件和试产验证 -> 量产版本冻结 -> ECN/偏差/试验版本控制 -> 客诉和审核时按版本追溯`
