## 定位

本文件补充PCB与IC载板知识库中相对容易被忽略但在工程和尽调中非常关键的三类内容：

- 设计规则如何映射到制造窗口、成本和良率。
- 可靠性与验证如何覆盖真实失效机理，而不是只做形式化测试。
- 先进封装、RDL、中介层、玻璃核心等与有机载板的边界如何划分。

默认讨论稳定知识，不写动态市场和公司事实。

## DFM/DFX基本框架

PCB/载板项目从图纸到量产，至少要经过以下几类“可实现性”检查：

| 维度 | 关注问题 | 典型输出 |
| --- | --- | --- |
| `DFM` | 图纸能否稳定制造 | 设计规则检查、最小线宽线距、孔径、环宽、阻焊桥、拼板建议 |
| `DFA` | 是否适合SMT/封装/组装 | 焊盘设计、共面性、拼板边、fiducial、贴装空间 |
| `DFT` | 是否便于电测、阻抗、可靠性抽样和失效定位 | 测试点、coupon、daisy chain、切片位、条码追溯 |
| `DFR` | 设计是否满足长期可靠性 | 热循环、CAF、弯折、翘曲、湿热、电迁移风险 |
| `DFC` | 成本是否与目标售价匹配 | 层数、材料、拼板利用率、特殊工艺和良率敏感项 |

对PCB/载板制造商来说，真正的能力不只是“按图生产”，而是把 `DFM/DFX` 反馈前置到立项、样品和小批量阶段，避免把系统性问题拖到量产后暴露。

## CAM/MI与工程资料完整性

CAM（Computer Aided Manufacturing）/MI（Manufacturing Instruction）阶段经常被低估，但它决定了车间是否能按统一口径制造。

工程资料、Gerber/ODB++/IPC-2581、CAM输出、MI字段、程序版本、工程放行和数据安全的完整框架详见 `33-engineering-cam-mi-data-control.md`。本节保留其在DFM/DFX与可靠性评审中的接口位置。

最小资料包通常应包括：

- 叠构 `stack-up`：层序、材料牌号、介质厚度、铜厚、目标阻抗。
- 钻孔表：孔径、公差、PTH/NPTH、控深钻、背钻要求。
- 线路规则：线宽线距、公差、阻抗网络和关键区域。
- 阻焊与字符：开窗、桥宽、颜色、局部豁免要求。
- 表面处理：种类、厚度窗口、适用焊接/打线/封装条件。
- 外形和成型：铣边、冲切、V-cut、倒角、毛刺要求。
- 测试和检验：电测规则、coupon、XRF/TDR、外观标准、抽样计划。
- 包装和储存：防潮、真空、烘烤、MSL或客户特殊要求。

资料不完整时，制造风险通常表现为：

- 工程人员按经验补默认值，造成与客户预期不一致。
- 不同工厂、班次或外协厂采用不同版本程序。
- 电测、阻抗、外观和包装标准相互脱节。
- 客诉发生后无法追溯“当时按哪个版本执行”。

## Stack-up与高速设计判断

高速板、高多层板和部分载板问题，本质上都离不开 `stack-up`。

### stack-up的核心作用

- 决定阻抗目标是否可实现。
- 决定串扰、损耗、电源完整性和回流路径。
- 决定压合窗口、板厚公差、翘曲和成本。
- 决定钻孔纵横比、背钻需求和可靠性风险。

### 评审stack-up时应优先问的问题

| 问题 | 为什么重要 |
| --- | --- |
| 目标阻抗是否与材料、铜厚、线宽公差一致 | 图纸阻抗不能脱离制造公差单独成立 |
| 是否区分信号层、参考层和电源层角色 | 回流路径错误会放大串扰和EMI风险 |
| 高速对是否跨分割、跨层或跨参考平面 | 这类结构常导致信号完整性问题 |
| 层数增加是否真的换来性能收益 | 很多设计是为布线便利加层，不一定提高盈利质量 |
| 背钻、stub控制和连接器结构是否闭环 | 服务器/通信板常见失效点不只在线路本身 |

### 对制造端的经营含义

- 更紧的阻抗窗口意味着更严格的材料批次控制、coupon设计和SPC管理。
- 更复杂的叠构意味着压合次数、对位补偿和实验室验证压力上升。
- 高速材料升级往往伴随良率爬坡和库存风险，不能只看ASP上行。

## 热设计与机械设计补充

PCB/载板失效不只来自电性能，热与机械约束同样关键。

| 设计主题 | 典型关注 | 常见失效或成本影响 |
| --- | --- | --- |
| 散热路径 | 铜厚、导热介质、金属基、导热过孔、散热焊盘 | 热阻过高、器件降额、局部热点 |
| 机械刚性 | 板厚、补强、拼板边框、连接器受力 | 板弯、裂纹、装配不良 |
| FPC弯折 | 弯折半径、铜型、补强、开窗位置 | 铜箔疲劳、覆盖膜开裂、焊盘剥离 |
| 大尺寸载板/薄板 | warpage、吸湿、共面性 | 封装贴装、焊球共面和客户端判退 |
| 厚铜/大电流 | 线路截面、热应力、蚀刻补偿 | 线宽失真、局部翘曲、焊接困难 |

## 可靠性验证逻辑

可靠性验证不是“项目末端做几项测试”，而是验证产品在实际使用与失效机理下是否成立。

建议按四层构建验证矩阵：

1. 使用场景：温度、湿度、振动、电压、电流、弯折、装配应力。
2. 关键失效机理：分层、孔铜裂纹、CAF、电迁移、焊接失效、warpage。
3. 验证项目：热冲击、热循环、HAST、PCT、弯折、跌落、插拔、盐雾等。
4. 判定与抽样：样本量、失效定义、切片/FA要求、再验证条件。

## 常见验证项目与适用场景

| 项目 | 主要应力 | 常见适用对象 | 关注失效 |
| --- | --- | --- | --- |
| 热循环 `TC` | 反复冷热循环 | 多层板、HDI、载板、车载板 | 孔铜裂纹、界面疲劳、分层 |
| 热冲击 `TST` | 快速温变 | 高CTE失配结构 | 界面开裂、翘曲突变 |
| 高温高湿偏压 `HAST` / `THB` | 湿热和偏压 | 高可靠板、细间距产品 | CAF、电迁移、绝缘衰退 |
| 压力锅 `PCT` | 高温高湿高压 | 层压结构、吸湿敏感产品 | 分层、界面剥离、吸湿失效 |
| 焊锡耐热/回流模拟 | 装配热应力 | SMT板、载板、FPC | 分层、起泡、表面处理失效 |
| 弯折/动态寿命 | 往复机械应力 | FPC、刚柔板 | 铜疲劳、覆盖膜开裂 |
| 跌落/冲击 | 机械冲击 | 消费电子模组板 | 焊点、连接器和板体裂纹 |
| 振动 | 长期机械振动 | 汽车、工业、航空 | 焊点疲劳、连接器和铜裂纹 |
| 盐雾/腐蚀 | 腐蚀环境 | 汽车、户外、工业 | 金属腐蚀、接触失效 |

验证项目的选择应由失效机理驱动，而不是只照搬客户历史清单。

## 可靠性项目与失效机理映射

| 失效机理 | 典型诱因 | 优先验证/分析 |
| --- | --- | --- |
| 孔铜疲劳裂纹 | Z向膨胀、厚板小孔、高温循环 | 热循环、切片、电阻监控 |
| 分层/起泡 | 吸湿、界面处理不足、回流热冲击 | 回流模拟、PCT、切片、SAM |
| `CAF` | 玻纤界面、污染、偏压和湿热 | HAST/THB、绝缘电阻、切片 |
| 电迁移 `ECM` | 离子残留、湿热偏压、细间距 | HAST/偏压、离子污染分析 |
| warpage失控 | 材料不对称、铜分布不均、吸湿 | 翘曲量测、回流前后比较 |
| 焊接/表面处理失效 | 镍腐蚀、氧化、厚度偏差 | 可焊性、XRF、截面和焊接评价 |
| FPC弯折裂纹 | 铜型不当、弯折区布局不佳 | 动态弯折、金相、应力复盘 |

## 失效分析 `FA` 基本流程

高阶PCB和载板的 `FA` 不能只停留在“哪里坏了”，应追到“为什么会在该处、该批次、该条件下坏”。

典型流程：

1. 明确失效定义：开路、短路、阻值漂移、焊接不良、warpage超限等。
2. 锁定范围：单片、同panel、同lot、同材料批次、同设备、同班次。
3. 保护样品：防止返工、清洗或切割破坏原始证据。
4. 选择分析手段：外观、显微、切片、X-ray、CT、SEM/EDS、离子分析、电性能复测。
5. 建立机理链：设计要求 -> 制程偏差 -> 结构证据 -> 电/热/机械失效结果。
6. 回写控制：PFMEA、Control Plan、检验规范、设备保养、供应商要求。

### 常用FA工具

| 工具 | 作用 | 适用场景 |
| --- | --- | --- |
| 光学显微镜 | 初步定位缺陷和形貌 | 外观、切片、裂纹 |
| X-ray / `CT` | 看内部孔、空洞、层间结构 | BGA、微孔、填孔、空洞 |
| 切片 `microsection` | 看孔铜、界面、镀层、树脂状态 | 孔铜、分层、CAF、表面处理 |
| `SEM/EDS` | 微观形貌和元素分析 | 腐蚀、污染、金属异常 |
| 离子色谱 / `ROSE` | 残留离子和清洁度分析 | ECM、绝缘失效、清洗不足 |
| `SAM` | 层间脱层和空洞探测 | 封装基板、层压结构 |

## 先进封装与载板边界

讨论“载板升级”时，经常把不同技术对象混成一类。至少应区分：

| 对象 | 位置 | 典型工艺 | 与有机IC载板的关系 |
| --- | --- | --- | --- |
| 有机IC载板 | 芯片封装基板 | ABF/BT、build-up、SAP/mSAP | 本知识库默认的“IC载板”主体 |
| `RDL` | 晶圆/面板级重布线层 | 薄膜、光刻、电镀 | 更接近先进封装前段互连，不等同PCB/载板 |
| 中介层 `Interposer` | 芯片与基板之间 | 硅、玻璃或有机中介层 | 是封装架构的一部分，不等同普通载板 |
| `SLP` | 类载板PCB | mSAP、细线路PCB工艺 | 仍属于PCB体系，但向载板靠近 |
| 玻璃核心/玻璃载板 | 新型基板/核心路线 | 玻璃加工、TGV等 | 属于新兴路线，需与现有有机载板分开分析 |

结论上要避免两种错误：

- 把所有先进封装互连都归为“IC载板升级”。
- 把 `SLP`、`RDL`、中介层和有机载板视为完全同一套设备、客户和盈利逻辑。

## 载板与封装协同的关键接口

有机载板量产能力很大程度上取决于它与封装端的接口是否稳定。

重点接口包括：

- bump / pad匹配。
- 表面处理与wire bonding / flip chip兼容性。
- strip尺寸和封装设备夹持、对位规则。
- warpage与底填、回流、焊球植球工艺的兼容性。
- 来料洁净度、颗粒、离子残留和客户端判定口径。

因此，载板厂“自己测得合格”不等于封装端“可稳定使用”。

## 设计评审中的高频红旗

| 红旗 | 常见后果 |
| --- | --- |
| 阻抗目标极紧，但未给材料和coupon策略 | 大批量阻抗偏移、返工或挑选出货 |
| 细线路设计跨越量产窗口，只能样品实现 | 量产FPY低、交付不稳 |
| FPC弯折区仍布孔、布焊盘或做厚补强 | 弯折裂纹和装配失效 |
| 载板/薄板只给常温warpage要求 | 客户回流后判退、共面性不良 |
| 只给最终验收标准，不给失效定义和FA要求 | 客诉后双方长期扯皮 |
| 电性能目标明确，但热设计和装配设计缺失 | 实际应用降额、寿命不足 |

## 工程评审最小清单

对新项目、可研或尽调，建议至少核对以下十项：

1. 产品类别和应用场景是否明确。
2. 关键 `stack-up` 与材料体系是否冻结。
3. 设计规则是否位于量产窗口内。
4. 关键尺寸、阻抗、翘曲和表面处理是否有测量方案。
5. coupon、daisy chain、切片位和追溯方案是否定义。
6. 可靠性矩阵是否覆盖真实使用应力。
7. 失效定义和FA流程是否预设。
8. 封装/装配接口要求是否完整。
9. 拼板、利用率、节拍和特殊工艺成本是否可接受。
10. 客户规范、IPC/JEDEC和内部标准是否已对齐。
