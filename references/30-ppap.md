## 一句话理解

PPAP回答的问题不是“文件交齐了吗”，而是“供应商能否用已批准的设计、材料、设备、过程、测量系统和控制方法，持续量产出满足客户要求的产品”。

## PPAP的目标和边界

PPAP是客户批准量产准备度的证据体系，不是纯行政流程。

| 维度 | PPAP关注什么 | PCB/载板场景 |
| --- | --- | --- |
| 设计 | 当前版本的图纸、叠构、材料和特殊特性是否明确 | Gerber/ODB++、叠构、阻抗、孔结构、表面处理 |
| 过程 | 工艺流程、参数和控制是否与批准状态一致 | 压合、电镀、蚀刻、激光、阻焊、电测、包装 |
| 测量 | 数据是否来自可信测量系统 | XRF、TDR、切片、AOI/AVI、CMM、属性判定 |
| 验证 | 尺寸、材料、性能、可靠性和外观是否满足要求 | 孔位、板厚、阻抗、可焊性、热循环、warpage |
| 量产条件 | 数据是否来自真实量产条件 | 量产设备、量产人员、量产材料、目标节拍 |
| 持续性 | 过程是否可长期稳定维持 | Control Plan、SPC、Cpk/Ppk、反应计划、再验证 |

PPAP不是所有客户、所有项目都用完全相同的格式。是否需要提交、提交级别、系统上传方式、样件数量、外观批准、年度验证和再提交条件，都可能由客户CSR或采购文件定义。

## 与APQP、Control Plan、MSA、SPC、Run at Rate、PSW的接口

| 文件/活动 | 与PPAP的关系 | PCB/载板落地重点 |
| --- | --- | --- |
| APQP | 决定PPAP前要完成哪些策划、验证和风险关闭 | 不要把PPAP当作APQP的替代品 |
| Control Plan | 证明量产控制逻辑已定义并可执行 | Pre-Launch/Production版本要清楚 |
| MSA | 证明关键数据可信 | 没有MSA，尺寸/能力数据说服力不足 |
| SPC/过程能力 | 证明量产过程已判稳并具备能力 | Cpk/Ppk和控制图要来自同一口径 |
| Run at Rate/试生产 | 证明真实量产条件下节拍和质量可达成 | 要有材料、设备、人员和批次追溯 |
| PSW | 通常是客户接收PPAP的核心签署文件 | 料号、版本、偏差和批准状态必须准确 |

典型链路：

`设计记录/特殊特性 -> PFMEA/Control Plan -> MSA/SPC/能力 -> 试生产/Run at Rate -> 尺寸/材料/性能/可靠性证据 -> PPAP包 -> PSW批准 -> 量产监控/再PPAP评估`

## 什么时候需要PPAP或再PPAP

具体触发条件以客户文件为准，但以下场景通常需要提交或至少评估是否需要再提交：

| 场景 | 常见处理 |
| --- | --- |
| 新客户、新平台、新产品导入 | 完整PPAP或客户指定差异包 |
| 新工厂/产线/地点转移 | 再PPAP或过程验证型提交 |
| 关键材料或供应商变化 | 影响评估，小批验证，必要时再PPAP |
| 关键设备、工装、程序或特殊过程变化 | 评估是否改变已批准过程假设 |
| 图纸、叠构、规格、特殊特性变化 | 需评估是否重新提交 |
| 客户投诉、重大可靠性问题、量产长期失控 | 可能触发再验证、偏差批准或重新提交 |

经验上，未经评估的“材料替代、外协切换、工艺窗口改变、地点转移”比单次批量异常更危险，因为它们会直接破坏已批准的PPAP基础。

## 提交等级

很多项目会用Level 1到Level 5的提交逻辑，但必须以客户要求为准。

| Level | 常见理解 | 现场含义 |
| --- | --- | --- |
| Level 1 | 只提交PSW或简化批准文件 | 仍需内部备齐对应证据 |
| Level 2 | PSW加部分样品和有限支持资料 | 客户可能只抽看关键项目 |
| Level 3 | PSW加完整支持资料和样品 | 最常见的标准完整提交逻辑 |
| Level 4 | 客户定义的定制要求 | 提交内容以客户书面要求为准 |
| Level 5 | 资料在供应商现场备查，客户现场审核 | 现场文件、样品、记录和线体状态必须一致 |

同一客户的不同项目也可能用不同级别。不要把历史项目的Level直接套到新项目。

## PPAP证据的基本原则

高质量PPAP包应满足四个原则：

1. 同一版本：图纸、材料、Control Plan、样品和报告指向同一批准状态。
2. 同一来源：样品、能力、MSA和测试报告来自同一批试生产或明确相关的量产条件。
3. 同一口径：时间、单位、规格、特殊特性编号和样本范围一致。
4. 同一逻辑：设计要求、过程控制、测量方法、结果和客户批准路径能连起来。

PPAP资料齐全但证据之间对不上，是最常见的退件原因之一。

## 常见证据包

以下是PCB/载板场景下常见的PPAP证据逻辑，具体删减和补充由客户决定。

| 类别 | 常见证据 | PCB/载板说明 |
| --- | --- | --- |
| 设计与变更 | 图纸、叠构、Gerber/ODB++、工程变更、客户批准 | 需明确特殊特性、表面处理、材料和版本 |
| 风险与流程 | 过程流程图、PFMEA、Control Plan、SOP/检验规范 | 工序、编号和版本要一致 |
| 测量与能力 | MSA、尺寸结果、SPC、Cpk/Ppk、属性一致性 | 测量系统和能力口径要对应 |
| 材料与性能 | 材料认证、环保、可焊性、阻抗、可靠性、外观 | 注意实验室资质、方法和样本来源 |
| 样品与辅具 | 提交样件、Master Sample、限度样、检查辅具/治具确认 | 电测治具、阻抗coupon、标签样式都可能被检查 |
| 客户要求 | CSR矩阵、系统上传、包装、标签、年度验证 | 不能只准备技术文件，忽略物流和系统要求 |
| 核心批准 | PSW或客户要求的等效批准文件 | 批准状态、偏差、有效期和数量要清楚 |

## 试生产和Run at Rate的有效性

PPAP最容易被质疑的不是格式，而是“数据是不是来自真实量产条件”。

有效的试生产/Run at Rate通常要满足：

- 使用量产设备、量产程序、量产工装和量产测量系统。
- 使用量产材料、量产供应商和量产包装逻辑。
- 由量产班组或至少具备量产代表性的人员执行。
- 在客户或内部定义的目标节拍下运行，而不是慢速样件模式。
- 记录异常、停机、返工、切换、SPC判异和恢复动作。

对PCB/载板项目，尤其要关注：

- 样品是否来自代表性panel/lot，而不是挑最好的几片。
- 微切片、阻抗、镀层厚度和可靠性样本是否对应同一批生产条件。
- 外协表面处理、包装和标签是否已按量产状态执行。

## PSW签署前的一致性评审

PSW不是最后一页签字，它是对整包证据的一次确认。签署前建议做“证据一致性评审”。

至少确认：

- 料号、版本、客户编号、修订号一致。
- 设计记录和工程变更已被纳入当前批准基线。
- 提交样品、尺寸报告、材料/性能结果来自同一提交状态。
- Control Plan、PFMEA、SOP、MSA、SPC、能力研究版本一致。
- 任何临时偏差、未关闭问题、客户口头同意事项都有书面痕迹。

如果这些关系说不清，即使文件都在，PPAP仍然不稳。

## PCB/载板项目的特殊关注点

与一般机械件相比，PCB/载板PPAP通常更容易在以下地方出问题：

| 主题 | 为什么高风险 | 典型表现 |
| --- | --- | --- |
| 叠构和材料 | 材料批次、树脂、铜厚和Dk/Df会影响多个结果 | 阻抗、板厚、翘曲和可靠性联动变化 |
| 破坏性检测 | 关键结果依赖微切片、可靠性和实验室方法 | 样本代表性不足、制样差异大 |
| 外协特殊过程 | 表面处理、外部实验室、特殊加工可能不在本厂 | 证据断点、追溯不完整 |
| 高混低量 | 数据量有限且切换频繁 | 过程能力和样本来源不稳定 |
| 包装和标签 | 客户系统、条码和防护要求复杂 | 客户收货/装配端直接退件 |
| IC载板/ABF | 洁净度、strip级追溯、warpage和封装接口更严 | 封装客户端验收与一般PCB口径不同 |

## 量产批准状态

PPAP批准状态通常至少包括以下几类：

| 状态 | 含义 | 管理重点 |
| --- | --- | --- |
| 完全批准 | 客户接受当前提交状态，可以按约定量产 | 继续按批准状态执行Control Plan和SPC |
| 临时批准 | 允许在限定数量/时间/条件下供货 | 必须有关闭计划、有效期和适用范围 |
| 拒收/退回 | 证据不足或结果不满足要求 | 不能当作量产批准使用 |

临时批准最容易被误用。没有到期管理、数量边界和问题关闭计划的“临时批准”，在管理上等于未控风险。

## 再提交评估

不是每次变更都需要完整Level 3 PPAP，但每次重要变更都应保留“为什么需要或不需要再提交”的判断证据。

建议至少评估：

- 图纸、叠构、材料牌号、供应商是否变化。
- 关键设备、夹具、程序、量具、地点是否变化。
- 是否影响特殊特性、MSA、SPC、能力、可靠性、包装或客户系统要求。
- 是否需要客户书面批准、偏差许可或样件确认。
- 现有库存、在制品和已交货是否受影响。

没有评估记录，后续客户追溯时很难证明你在受控状态内变更。

## 内部PPAP评审清单

在正式提交前，建议内部至少做一次跨部门评审。

| 维度 | 核对问题 |
| --- | --- |
| 客户要求 | CSR、标签、包装、系统上传、年度验证、提交等级是否明确 |
| 设计基线 | 图纸、叠构、特殊特性、工程变更是否冻结到同一版本 |
| 过程控制 | 流程图、PFMEA、Control Plan、SOP是否一致 |
| 测量系统 | 关键量具/方法的MSA是否完成且有效 |
| 过程能力 | 关键特性是否先判稳再给Cpk/Ppk |
| 样品来源 | 样品是否来自代表性试生产/量产条件 |
| 实验室与测试 | 方法、样本量、准则、资质是否满足客户要求 |
| 问题闭环 | 未关闭问题是否已分级、遏制、验证并明确客户沟通路径 |
| 签署权限 | PSW和内部批准是否由正确职责签署 |

## 客户退件和失败的高频原因

| 类别 | 常见问题 |
| --- | --- |
| 版本问题 | 图纸、PSW、Control Plan、样品版本不一致 |
| 数据问题 | 过程未判稳就报Cpk，或MSA不成立 |
| 样品问题 | 样件不是量产条件、来源不可追溯 |
| 文件逻辑问题 | 流程图、PFMEA、Control Plan三张文件对不上 |
| 客户要求问题 | 漏掉包装、标签、外观批准、系统上传或CSR细项 |
| 问题管理问题 | 试生产异常未闭环，临时偏差未说明 |

## 客户投诉、变更与PPAP更新

量产后，PPAP不是作废，而是当前批准状态的基线。以下情况通常需要回看PPAP：

- 客户投诉暴露出批准时未识别的风险。
- 关键特性Cpk/Ppk或SPC长期恶化。
- 材料、供应商、工艺、设备、程序或地点变化。
- 包装标签、外观标准或客户系统要求变化。
- 年度验证、过程审核或实验室比对失败。

这时应先判断是：

1. 现场执行偏离了已批准PPAP。
2. 已批准PPAP本身不足。
3. 变更后没有触发应有的再提交评估。

## PCB/载板场景下的常见错误

- 把PPAP当作文控打包任务，而不是量产准备度证明。
- 样件、尺寸、阻抗、微切片、可靠性报告来自不同批次，彼此对不上。
- 工程样或低速样件数据被拿来代表量产能力。
- Cpk/Ppk没判稳就提交，或只报最好的一段数据。
- MSA、AOI/AVI一致性、实验室方法没有准备充分。
- 忽略包装标签、外观Master Sample、客户系统上传和CSR细项。
- 临时批准后未跟踪有效期、关闭计划和供货范围。
- 发生材料替代、程序改版或地点变化后，没有再PPAP评估记录。
- 只保留提交包，不保留现场执行状态和原始数据。
- PPAP批准后，Control Plan、SPC和培训没有按批准状态运行。

## 快速落地流程

`确认客户PPAP要求和等级 -> 冻结设计/过程/特殊特性基线 -> 完成MSA/SPC/能力/试生产 -> 整理尺寸/材料/性能/可靠性证据 -> 做一致性评审 -> 签署PSW和提交 -> 跟踪批准状态 -> 量产阶段维护并做再提交评估`

使用建议：

- 做新项目导入时，先用 `26-apqp.md` 走完整体主线，再用本页检查第四阶段证据是否真正能支撑客户批准。
- 做Control Plan、MSA、SPC和能力研究时，始终考虑这些数据最终是否能进入PPAP，而不是只满足内部报表。
- 做客户投诉、变更、工厂转移或二供导入时，把本页当作“是否需要再PPAP/客户批准”的快速评估框架。
