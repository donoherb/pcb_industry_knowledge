# PCB/载板DMAIC案例库

来源更新：2026-04-17。本文不是记录具体企业机密案例，而是沉淀PCB/载板常见DMAIC项目的案例骨架、典型Y/X关系、常见陷阱和复用要点，便于后续快速套用。

## 使用原则

- 主方法论看 `14-dmaic.md`。
- 真正开项目时，表单和记录结构优先看 `15-dmaic-templates.md`。
- 本页提供“像什么样的项目、一般从哪里入手、常见会踩什么坑”。

## 案例总览

| 案例 | 典型Y | 适用场景 |
| --- | --- | --- |
| 孔铜不良率改善 | 切片不良率、热循环开路率 | 刚性板、HDI、厚板 |
| 阻抗波动改善 | 阻抗均值、标准差、Cpk | 高速板、服务器板、交换机板 |
| AOI过杀率改善 | 过杀率、复判工时 | 内外层AOI、阻焊AOI、AVI |
| 压合翘曲改善 | warpage、板厚偏差、分层率 | 高多层、高速板、载板 |
| FPC补强贴合不良改善 | 偏位率、贴装不良率 | FPC、软硬结合板 |
| Cycle Time改善 | 总周期、等待时间、在制品周转 | 多品种工厂、瓶颈线体 |

## 案例1：孔铜不良率改善

### 典型问题

- 孔铜薄、空洞、切片不良集中在部分机台或班次。
- 热循环后开路率偏高。

### 常见Y

- 孔铜切片不良率
- 热循环开路率
- 孔铜厚度均值和波动

### 常见X

- 电流密度
- 活化/沉铜状态
- 槽液成分和维护频率
- 挂具状态
- 班次和操作方法
- 切片制样一致性

### 常见陷阱

- 没做切片MSA，导致“差异”来自制样和判读。
- 只看平均孔铜厚，不看尾部和空洞。
- 只改电镀参数，不管前处理和挂具。

## 案例2：阻抗波动改善

### 典型问题

- 同一料号阻抗分布宽，Cpk长期偏低。
- 客户端高速裕量不足，但出厂大多仍在规格内。

### 常见Y

- 阻抗均值
- 阻抗标准差
- `Cpk/Ppk`

### 常见X

- 线宽补偿
- 介质厚度
- 铜厚
- 材料批次
- 压合窗口
- TDR程序或测试规则

### 常见陷阱

- 混入不同叠构、不同层别的数据。
- 没确认TDR和coupon口径一致。
- 改善只看均值拉回，没让波动收敛。

## 案例3：AOI过杀率改善

### 典型问题

- AOI/AVI报警量大，复判占用大量人力。
- 真不良率不高，但大量良品被拦截。

### 常见Y

- 过杀率
- 复判工时
- 真实流出率

### 常见X

- 程序阈值
- 图像光照和清晰度
- 缺陷库和限度样
- 复判标准
- 程序版本切换

### 常见陷阱

- 只压低阈值来减少报警，结果漏判上升。
- 没区分缺陷类型做Pareto。
- 没做属性MSA就调整程序。

## 案例4：压合翘曲改善

### 典型问题

- 高多层或高速板翘曲超限。
- 客户装配或连接器区域出现机械不匹配。

### 常见Y

- 常温/热态warpage
- 板厚偏差
- 分层率

### 常见X

- 叠构对称性
- 铜平衡
- 压合曲线
- 材料吸湿
- 烘烤条件
- 冷却和搬运方式

### 常见陷阱

- 只看常温翘曲，不看热态。
- 把不同叠构一起分析。
- 只改压合曲线，不回到叠构和铜平衡。

## 案例5：FPC补强贴合不良改善

### 典型问题

- 补强偏位、起翘、贴装区不平整。
- 客户端SMT或连接器装配不良。

### 常见Y

- 补强偏位率
- 补强起翘率
- 客户端贴装不良率

### 常见X

- 治具定位
- 贴合参数
- 材料尺寸稳定性
- 补强材料批次
- 操作方法

### 常见陷阱

- 只看出厂外观，不看客户端贴装结果。
- 没分静态弯折和动态弯折场景。
- 改善后没有把治具和作业法标准化。

## 案例6：Cycle Time改善

### 典型问题

- 订单周期长，在制品堆积。
- 设备稼动率看起来不差，但交付仍慢。

### 常见Y

- 总周期
- 等待时间
- 在制品周转
- OTD

### 常见X

- 工程资料释放时间
- 瓶颈机台排队
- 复判和切片实验室吞吐
- 急单插单比例
- 换线和首件确认时间

### 常见陷阱

- 把所有时间都归因为设备不足。
- 不拆等待时间和加工时间。
- 改善后没有调整排产和升级规则。

## 如何选案例起点

| 问题特征 | 优先参考案例 |
| --- | --- |
| 切片/热循环/孔结构问题 | 孔铜不良率改善 |
| 高速/阻抗/材料波动问题 | 阻抗波动改善 |
| 复判压力大、报警多 | AOI过杀率改善 |
| 板弯、装配机械不良 | 压合翘曲改善 |
| FPC贴合、补强、客户端贴装问题 | FPC补强贴合不良改善 |
| 交付慢、在制高、等待多 | Cycle Time改善 |

## 复用时的边界提醒

- 案例骨架可复用，但参数门槛、样本量、财务收益口径不能照抄。
- 任何涉及客户批准、材料切换、工厂转移和程序变更的动作，都要回到 `APQP/PPAP/8D/FMEA` 体系确认。
- 相似案例不等于相同主因。先做分层和MSA，再决定是否复用过去结论。
