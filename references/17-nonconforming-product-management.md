## 一句话理解

不合格品管理回答的问题不是“发现不良后怎么处理掉”，而是“如何在识别、隔离、评审、处置、追溯和放行各环节中，确保不合格品不会被误用、误放行或掩盖成正常品”。

## 适用范围

不合格品不仅包括成品，也包括：

- 来料不合格
- 在制品不合格
- 外协返回品不合格
- 终检不合格
- 客户退货和现场不良
- 可疑品和待判品

对PCB/载板行业，还应覆盖：

- panel、strip、coupon、切片样品
- 电测不良、AOI/AVI争议品
- 表面处理异常批
- 返工/返修后的再检品

## 与Control Plan、8D、PPAP、FMEA的关系

| 文件/活动 | 与不合格品管理的关系 | PCB/载板落地重点 |
| --- | --- | --- |
| `Control Plan` | 定义何时触发隔离、复测和升级 | 反应计划必须写明隔离范围和恢复条件 |
| `8D` | 对重大不合格和流出问题做正式闭环 | 不合格品管理先止血，8D再追根因和防再发 |
| `PPAP` | 若不合格涉及批准状态，可能触发偏差/再提交 | 返工样、让步样不能冒充量产批准样 |
| `FMEA` | 重复或重大不合格应回写风险和控制 | 现有控制失效时要更新PFMEA/DFMEA |
| `SPC/MSA` | 用于识别异常和验证恢复条件 | 量测不可信时，不合格判定本身也不稳 |

典型链路：

`发现异常 -> 标识与隔离 -> 范围界定 -> MRB评审 -> 返工/返修/让步/报废 -> 再检与放行 -> 重大问题进入8D/FMEA/PPAP评估`

## 不合格品的基本分类

| 分类 | 含义 | 常见场景 |
| --- | --- | --- |
| 明确不合格 | 已确认不满足要求 | 板厚超差、阻抗超限、孔铜不足 |
| 可疑品 | 暂未确认，但存在风险 | 同机台前后批、程序错版影响批次 |
| 待判品 | 需实验室、工程或客户进一步判断 | 外观争议、轻微尺寸偏差、AOI争议 |
| 流出不良 | 已出货或到客户现场才发现 | 客诉、退货、现场筛选 |

不要把“可疑品”混成“合格待用”。只要风险未排除，就应按受控状态管理。

## 核心原则

1. 先隔离，再判断，不要边生产边讨论。
2. 先保护证据，再返工或切片，不要把证据先破坏掉。
3. 先界定范围，再决定处置，不要只处理眼前几片。
4. 任何例外放行都必须有批准边界、数量边界和有效期。
5. 返工、返修、让步、报废必须分清，不能用一个词代替所有处置。

## 识别与触发来源

不合格品管理的触发来源常见于：

- 来料检验
- 首件失败
- 过程SPC判异
- AOI/AVI/X-ray/电测/终检发现异常
- 实验室切片、阻抗、可靠性验证失败
- 客户投诉、退货和驻厂反馈
- 供应商异常通知
- 程序、材料、设备、标签或追溯错误

### PCB/载板高频触发项

| 触发项 | 常见问题 |
| --- | --- |
| 电测 | 开短路、导通不稳、误判/漏判争议 |
| AOI/AVI | 微短路、残铜、阻焊偏位、过杀争议 |
| 切片 | 孔铜不足、空洞、分层、填孔凹陷 |
| 阻抗/TDR | 阻抗超限、分布漂移 |
| XRF/表面处理 | 金厚、镍厚、镀层异常 |
| 外观/包装 | 划伤、污染、标签错、混料 |

## 标识与隔离

不合格品管理最容易失守的不是技术判断，而是现场混料和误放行。

### 最低要求

- 有清晰状态标识：不合格、待判、隔离、返工中、返工待检、报废待处理。
- 有物理隔离或系统锁定：独立区域、独立料架、系统冻结、条码锁定。
- 有数量和批次记录：lot、panel、strip、机台、班次、材料批次。
- 有责任人和时间记录：谁发现、何时隔离、谁批准移动。

### 隔离范围至少要考虑

- 同机台
- 同班次
- 同材料批次
- 前后批次
- 在制品、库存、在途、客户现场

如果范围界定不清，宁可先扩大隔离，再逐步缩小。

## MRB评审

`MRB` 不是为了合法化放行不良，而是为了用跨职能机制做受控判断。

### MRB典型参与角色

- 质量
- 工艺/工程
- 生产
- 设备
- 项目/销售
- 必要时设计、SQE、实验室、财务

### MRB至少要回答五个问题

1. 这批产品到底哪里不满足要求。
2. 影响范围到哪里。
3. 是否影响客户功能、可靠性、法规、装配或批准状态。
4. 有哪些可接受处置路径。
5. 处置后如何验证和谁批准放行。

## 常见处置方式

| 处置 | 含义 | 使用边界 |
| --- | --- | --- |
| 返工 `Rework` | 通过再加工恢复到原要求 | 必须有已批准工艺和再检标准 |
| 返修/修理 `Repair` | 恢复到可接受状态，但不一定回到原始要求 | 多数情况下需更高层批准，部分客户禁止 |
| 让步接收 `Use-as-is` / `Concession` | 在偏差可接受前提下按现状接收或出货 | 需明确客户/内部批准边界 |
| 降级使用 | 用于较低等级或内部用途 | 不得与原客户要求混用 |
| 退供应商 | 来料或外协异常退回处置 | 需锁定使用范围并跟踪供应商闭环 |
| 报废 `Scrap` | 不可恢复或不值得恢复 | 需物理和系统双重防误用 |

## 返工与返修的边界

这是现场最容易混淆的点。

| 维度 | 返工 `Rework` | 返修/修理 `Repair` |
| --- | --- | --- |
| 目标 | 恢复到原要求 | 恢复到可接受使用 |
| 依据 | 原规范、已批准返工工艺 | 需特别批准的修理准则 |
| 风险 | 过程风险、重复热史或重复加工风险 | 客户接受性和批准状态风险更高 |
| 放行条件 | 再检确认满足原要求 | 需按批准条件和限定范围放行 |

对PCB裸板和载板，很多客户对“repair”比“rework”更敏感，不能内部口头同意后直接出货。

## 让步接收与偏差许可

### 让步接收

适用于已经发现的不合格品，在确认功能、可靠性、装配和法规风险可接受后，经必要批准按限定条件接收。

### 偏差许可

更偏事前或特定批次的受控偏离批准，例如材料临时替代、尺寸轻微偏差、标签格式暂时调整等。

### 使用边界

- 必须明确适用批次、数量、有效期和客户。
- 必须保留书面批准痕迹。
- 不能把一次让步默认为长期标准。
- 不能用让步掩盖系统性失控。

## 报废管理

报废不是“扔掉”，而是一个必须可追溯、可复核、防误用的受控动作。

### 报废至少要满足

- 有报废判定依据。
- 有审批权限。
- 有数量、批次和原因记录。
- 有物理处置和系统扣账。
- 有防止报废品回流产线或流入市场的措施。

### PCB/载板特别注意

- panel拆分后部分报废、部分保留时，要确保追溯不断裂。
- 返工失败转报废时，要保留原始不合格和返工记录。
- 铜、金、贵金属和特种材料报废还可能涉及财务和环保管理。

## 客户通知与放行边界

以下情形通常不应只做内部MRB：

- 涉及客户功能、可靠性、装配或法规风险。
- 涉及已批准PPAP基线的偏离。
- 涉及客户特殊特性。
- 涉及客户已收到或在途产品。
- 涉及repair、让步出货、临时偏差许可。

如果客户文件要求书面批准、系统上传或偏差单流程，必须按客户路径执行。

## 再检与恢复放行

不合格品经过返工/返修/筛选后，必须明确：

- 用什么标准再检。
- 是否需要100%复检或加严抽样。
- 是否需要实验室或可靠性补充验证。
- 谁批准恢复放行。
- 是否需要对前后批次追溯复核。

恢复放行不能只写“复检OK”。应能说明恢复条件已被满足。

## 追溯要求

不合格品记录至少应包含：

- 产品/料号/版本
- lot、panel、strip、batch
- 数量和不合格数量
- 发现工序和发现方式
- 缺陷类型和判定依据
- 隔离范围
- MRB结论
- 返工/返修/让步/报废记录
- 再检结果
- 放行或关闭批准

## PCB/载板高频场景

| 场景 | 常见处置重点 |
| --- | --- |
| 电测不良 | 区分真实不良与治具/程序误判，防止误报废 |
| AOI/AVI争议 | 做属性一致性和复判，防止过杀积压 |
| 切片不良 | 先锁定机台/班次/药水范围，再评估返工可能性 |
| 阻抗偏差 | 区分批次偏移和系统性漂移，谨慎处理让步出货 |
| 表面处理异常 | 确认是否可返工重做及对可靠性的影响 |
| 标签/追溯错误 | 即使产品物理合格，也应按不合格品管理处理 |

## 与8D、DMAIC的分工

| 工具 | 主要目的 | 与不合格品管理的关系 |
| --- | --- | --- |
| `17-nonconforming-product-management.md` | 当下识别、隔离、评审和处置 | 先控制住不良品流动 |
| `8D` | 对重大问题做正式根因和防再发闭环 | 处理造成不合格的系统问题 |
| `DMAIC` | 改善长期绩效和波动 | 处理长期高报废、高返工和高过杀 |

很多现场问题不是缺少分析，而是先前连不合格品都没隔离好，导致后续分析对象和范围都失真。

## 审核常看点

- 不合格品状态标识是否清楚。
- 隔离区、系统状态和现场实物是否一致。
- MRB权限和结论是否合规。
- 让步、偏差许可和客户批准是否有书面记录。
- 报废记录与财务/MES/QMS是否一致。
- 返工后的再检和恢复放行是否有证据。
- 重复不合格是否回写到8D、FMEA、Control Plan。

## PCB/载板场景下的常见错误

- 把可疑品先混入正常品区，等有空再判断。
- AOI/AVI争议品没有复判规则，长期堆积。
- 把返工、返修、让步三个概念混成“特采”。
- 返工后只看最终合格，不记录返工路径和再检依据。
- 报废只有数量，没有原因和批次。
- 客户批准边界不清，却直接做让步出货。
- 供应商来料异常只退料，不追溯已使用范围。
- 标签和追溯错误被当作“小问题”处理，未按不合格品控制。

## 快速落地流程

`发现异常 -> 立即标识和隔离 -> 界定范围 -> MRB评审 -> 选择返工/返修/让步/报废路径 -> 再检和恢复放行 -> 重大问题转8D/FMEA/PPAP评估`

使用建议：

- 做日常质量管理时，把本页作为“先控住实物”的主文件，再配合 `32-8d.md` 做根因闭环。
- 做客户审核、IATF审核和PPAP评估时，重点检查让步、返工、报废和恢复放行是否有受控证据。
- 做高报废、高返工或高过杀项目时，可在本页的实物流控制基础上再启动 `14-dmaic.md`。
- 做AOI、ET、FQC检验结果与报废位置、原因和状态联动时，结合 `36-defect-scrap-mapping-aoi-et-fqc.md`。
