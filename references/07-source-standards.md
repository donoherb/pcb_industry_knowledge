# 来源、引用与更新规则

## 来源分级

| 等级 | 来源 | 用途 |
| --- | --- | --- |
| A | 公司公告、年报、招股书、交易所文件、客户/供应商正式文件 | 公司事实、财务、产能、风险 |
| A | 标准组织、监管机构、行业协会正式文件 | 标准、政策、行业口径 |
| B | 知名行业报告、付费数据库、协会统计、券商深度报告 | 市场规模、竞争格局、趋势 |
| B | 公司官网、投资者交流材料、业绩会纪要 | 产品、客户、扩产、战略 |
| C | 媒体报道、展会信息、招聘信息、供应链访谈 | 线索和交叉验证 |
| D | 社交媒体、未署名转载、二手截图 | 只作线索，不能单独作为结论依据 |

结论涉及投资、供应商选择、项目立项或客户判断时，优先使用A级来源；B级可辅助；C/D级必须交叉验证。

## 引用格式

记录动态事实时使用以下格式：

```text
事实：公司A某工厂计划建设高阶HDI产线。
来源：公司A 2025年年度报告，第X节。
日期：报告披露日YYYY-MM-DD；数据期间FY2025。
口径：计划投资额为含税/不含税；币种为人民币。
备注：是否已投产、是否客户认证完成仍需跟踪。
```

输出给用户时至少说明：

- 来源名称。
- 披露日期或数据期间。
- 数据口径和币种。
- 是否为公司披露、第三方估计或推断。

## 必须更新的内容

以下内容不能长期依赖本地知识库：

- 市场规模、增长率、价格、供需缺口。
- 公司排名、产能排名、客户份额。
- 公司财务数据、扩产进度、投产时间。
- 客户名单、供应商名单、认证状态。
- 管理层、股权结构、并购重组。
- 政策、关税、出口管制、环保要求。
- 标准的最新版本和认证状态。

如果用户问“最新”“今年”“现在”“目前”“最近”，必须先核验当前资料。

## 数据口径规则

- 时间：区分自然年、财年、季度、月度、滚动十二个月。
- 币种：标明人民币、美元、新台币、日元等；换算需说明汇率日期。
- 单位：PCB可能用平方米、平方英尺、片、层数或金额；IC载板可能用片、颗、面积、封装类型或金额。
- 范围：区分全球、中国大陆、中国台湾、日本、韩国、东南亚等区域。
- 产品：区分PCB总量、HDI、FPC、高多层、高频高速、IC载板、ABF、BT。
- 公司：区分集团、上市主体、子公司、工厂和合资公司。
- 产能：区分设计产能、有效产能、投产产能、满产产能和良率后的合格产出。

## 更新流程

1. 收集资料：优先公告、年报、招股书、协会统计和公司正式材料。
2. 统一口径：时间、币种、单位、区域、产品和公司边界。
3. 交叉验证：重要结论至少两个独立来源，或一个A级来源。
4. 记录证据：写明来源、日期、页码或章节。
5. 标注不确定性：如果是估算、推断或媒体线索，明确说明。
6. 回写知识库：稳定结论进入对应参考文件；动态数据另建带日期的数据记录，不覆盖原始口径。

## 标准资料更新规则

IPC、JEDEC、UL、ISO、IATF、AIAG核心工具手册、OEM客户特定要求（CSR）等标准/准标准资料的“最新版本”属于动态事实。更新标准资料时：

- 优先使用标准组织或官方商店页面；第三方标准销售页只能作交叉核对。
- 必须记录核验日期、标准号、修订版、语言、Published/Released日期、是否含amendment、addendum、errata。
- 区分“已发布现行版”“拟发布/ballot/working draft/final draft”和“翻译版”。未发布草案不能写成现行验收依据。
- 不把付费标准的条款表、图片、图示缺陷判定和长段正文写入知识库；只记录标准定位、适用场景、版本状态、与其他标准的配套关系和采购/核验入口。
- 客户图纸、采购文件和质量协议优先于知识库中的通用版本说明；如果客户指定旧版，不能用知识库记录的新版自动替代。
- IATF 16949标准、IATF Sanctioned Interpretations/FAQs、AIAG Core Tools手册和OEM CSR要分开记录。AIAG手册更新不等于IATF标准自动升版，OEM CSR可能规定更早或不同的执行日期。
- 输出或尽调使用时，说明“版本截至YYYY-MM-DD核验”，并提醒正式合规判断需查授权原文。

## 推荐沉淀模板

对后续新增资料，建议在知识库外或项目内使用如下文件名：

```text
YYYY-MM-DD_公司名_资料类型_主题.md
YYYY-MM-DD_行业报告_机构名_主题.md
YYYY-MM-DD_访谈纪要_主题.md
```

单条记录建议包含：

```text
# 标题

来源：
披露日期：
数据期间：
可靠性等级：A/B/C/D
涉及公司/产品：

## 关键事实

## 可用于分析的结论

## 尚需核验

## 原文摘录或页码
```

## 冲突来源处理

当不同资料出现冲突时，先判断冲突的是“事实”“口径”还是“适用条件”。

| 冲突类型 | 优先依据 | 处理原则 |
| --- | --- | --- |
| 客户图纸/质量协议 vs 通用标准 | 客户文件 | 交付和验收按客户文件执行，知识库只解释差异 |
| 已发布标准 vs 草案/解读文章 | 已发布标准 | 草案只能作为趋势线索，不能写成现行要求 |
| 公司公告 vs 媒体/访谈/招聘信息 | 公司公告 | 媒体和访谈可作补充线索，但不能单独定案 |
| 集团口径 vs 工厂或产品线口径 | 与问题最贴近的经营单元 | 讨论产能、良率和客户导入时，优先落到工厂和产品线 |
| 营销表述 vs 验证数据 | 量产和验证证据 | 样品、宣传和路线图不等于量产能力 |

若冲突无法消除，输出中应明确写“存在分歧来源”和“当前更可信的采用口径”，不要把有争议事实写成确定结论。

## 知识库写作规范

- 稳定知识进入 `references/`；动态事实优先写入带日期的资料摘录或项目档案。
- 每条动态事实至少写清来源、日期、对象边界、数据口径和不确定性。
- 推断要显式标注。可以写“根据A和B推断”，不要把推断伪装成来源原句。
- 表格、比较框架和检查清单优先于大段堆砌资料，因为它们更容易复用和更新。
- 基础知识补充优先写清“定义、适用口径、易混点、与其他文件的联动”，避免把基础入口写成过长的百科堆砌。
- 跨skill调用说明应写成稳定接口：触发条件、最小读取文件、知识包输出格式、动态事实规则和调用边界；不要把某个调用方skill的排版或文件生成细节写入本知识库。
- 报价、产能、ASP、客户导入和尽调模板可以沉淀稳定框架；但具体价格、良率、稼动率、账期、返利、索赔、客户认证阶段、NPI状态和量产放行证据必须按客户文件、公司资料或带日期来源核验。
- 涉及客户特定要求、PPAP等级、标准版本和合规边界时，应保留“以客户文件/授权原文为准”的提示。
- 关于 `DFM/DFX`、可靠性矩阵、FA方法、运营与成本框架、客诉闭环、追溯包装、实验室相关性和年度验证，可直接沉淀稳定方法论；但具体客户测试条件、样本量、量产门槛、响应时限、标签字段、包装有效期、报价条款、稼动率和良率阈值若与特定客户或公司绑定，应按动态事实管理并注明适用对象。
- 关于工程资料控制、设备维护、工装治具、OEE、EHS和化学品管理，可沉淀通用方法；但具体客户批准版本、设备稼动数据、故障率、排放限值、排污许可、处罚记录、SDS版本、禁限用物质清单和法规状态必须按日期、属地和客户文件核验。
- 关于缺陷mapping、报废mapping、AOI/ET/FQC结果整合和缺陷密度分析，可沉淀通用方法；但具体客户map格式、设备导出格式、缺陷码、报废金额、客户良率口径和系统接口规则必须按客户文件和企业系统主数据核验。

## 输出约束

- 不把第三方预测写成事实。
- 不把公司规划写成已投产。
- 不把样品突破写成量产能力。
- 不把客户认证中写成已供货。
- 不把行业平均毛利率套到单家公司。
- 不用未注明日期的数据支撑“当前”结论。
