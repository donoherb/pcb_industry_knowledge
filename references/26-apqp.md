# APQP专题：VOC、设计开发、过程设计、量产验证与持续改进

## 定位与来源

本页沉淀APQP主要阶段的实务方法：第一阶段把顾客声音（VOC）、法规/标准、客户特殊要求（CSR）和历史质量信息，转换成可测量、可验证、可追溯的项目质量指标；第二阶段把这些输入继续落到产品设计开发、DFMEA、DVP&R、样件控制计划和设计评审；第三阶段把设计方案转换为稳定、可重复、可控制、可交付的制造过程；第四阶段用真实量产条件验证产品和过程，并形成Run at Rate、MSA/Cpk、产品验证、包装标签验证和PPAP批准证据；第五阶段在量产生命周期内持续执行控制计划、SPC、顾客反馈、供应商质量、ECN和持续改进，并把量产经验教训回流到下一轮APQP。

来源更新：2026-04-17。本文只做知识提炼和PCB/载板场景化，并补充通用APQP实务、汽车电子项目管理、PPAP提交、供应商APQP、量产反馈和变更控制经验；正式APQP、Control Plan、PPAP、FMEA等要求仍以AIAG/IATF手册、客户CSR和项目合同为准。


## 核心原则

- APQP第一阶段的价值不是完成表单，而是把质量风险前移到立项、报价、设计输入和工艺策划阶段。
- 客户语言必须转换成工程语言。诸如“可靠”“耐用”“外观好”“不能失效”等表达，需要被拆解成规格、公差、验证条件、样本量、验收准则和反应计划。
- 输出要满足三点：可测量、可验证、可追溯。每个关键指标应能追溯到VOC/CSR/法规/历史问题，并能进入FMEA、控制计划和验证计划。
- 特殊特性不应等到量产前才识别。高权重VOC、高风险失效后果、法规/安全项、客户CTQ和过程敏感项，都应在APQP早期形成初始特殊特性清单。
- “质量是设计出来的”。如果APQP第一阶段没有把VOC转成工程指标，后续FMEA、Control Plan和PPAP容易变成补文档。
- APQP不是质量部门单独填表，而是销售、项目、研发、工艺、质量、制造、采购、供应商和客户共同管理项目风险的机制。
- 阶段输出要有准入、准出和批准人。未关闭的重大质量风险、客户特殊要求、产能风险和供应商风险，必须通过RYG升级机制显性化。

## 适用范围与裁剪

APQP适用于新产品、新客户、新技术、新材料、新工艺、新产线、新供应商、新生产地点以及客户要求重新批准的项目。PCB/载板行业订单形态差异很大，不能所有项目都做同等深度，但裁剪必须基于风险，而不是基于时间压力。

| 项目类型 | APQP深度 | 重点 |
| --- | --- | --- |
| 全新平台/全新客户 | 完整五阶段APQP | VOC/QFD、DFMEA、DVP&R、PFMEA、Control Plan、MSA、Run at Rate、PPAP全链路 |
| 新料号但沿用成熟平台 | 裁剪式APQP | DFM、特殊特性、过程流程差异、关键尺寸/阻抗/可靠性验证、PPAP差异包 |
| 工艺或材料变更 | 变更驱动APQP | 影响评估、风险更新、小批验证、MSA/能力/可靠性复核、客户批准 |
| 产线/工厂转移 | 过程验证型APQP | 设备/人员/材料/治具一致性、相关性比对、Run at Rate、再PPAP |
| 快速样件/工程样 | 样件控制计划 | 明确非量产差异、样件检验、风险声明和后续量产验证计划 |
| 高可靠/车规/服务器/载板项目 | 强化APQP | 特殊特性、可靠性、供应商APQP、追溯、能力、变更批准和年度验证 |

裁剪时至少保留四件事：客户要求评审、风险评估、控制计划/检验计划、变更和问题闭环。任何影响客户CTQ、特殊特性、可靠性、法规或PPAP批准状态的项目，不应只按普通新料号导入处理。

## 项目治理与RACI

APQP需要清晰的跨职能职责。推荐在项目启动时建立CFT（Cross Functional Team，跨职能团队）、项目章程、里程碑计划、风险清单、行动项清单和升级规则。

| 角色 | 典型职责 | PCB/载板落地 |
| --- | --- | --- |
| 项目/PM | 主计划、节点、客户沟通、风险升级和资源协调 | 样件、试产、PPAP、量产爬坡节点管理 |
| 销售/客服 | VOC、合同、客户CSR、商务和交付承诺 | 客户规范、PPAP等级、标签/包装/系统上传要求传递 |
| 研发/设计 | 设计输入、DFMEA、DVP&R、设计评审 | 叠构、阻抗、材料、可靠性、DFM/DFT和客户图纸评审 |
| 工艺/PE | 流程、参数窗口、PFMEA、控制计划、SOP | 压合、电镀、蚀刻、钻孔、阻焊、表面处理、检测能力 |
| 质量/QE | APQP证据链、MSA、SPC、PPAP、审核和客户质量 | 特殊特性追溯、8D、控制计划执行、PPAP包一致性 |
| 制造/生产 | 人员、节拍、现场执行、试产和量产反馈 | 量产班组培训、Run at Rate、瓶颈和异常记录 |
| 设备/ME | 设备能力、保养、治具、备件和防错 | 钻机、LDI、压机、电镀线、AOI/AVI、电测治具 |
| 采购/SQE | 供应商准入、材料认证、供应商APQP和变更 | CCL/PP/铜箔/油墨/药水/外协特殊过程质量管理 |

RACI建议：每个关键交付物必须明确Responsible（执行人）、Accountable（最终负责人）、Consulted（协商方）和Informed（知会方）。例如PFMEA通常由工艺/质量牵头，生产、设备、设计、SQE参与；客户特殊要求矩阵由质量或项目牵头，销售、工程、制造和供应链共同确认。

## VOC输入来源

| 输入来源 | 作用 | PCB/PCBA/载板示例 |
| --- | --- | --- |
| 市场调研与竞品对标 | 识别客户真实使用场景、竞品性能边界和差异化目标 | 服务器板的高速损耗、汽车板的长期可靠性、消费电子HDI的轻薄化和跌落耐受 |
| 保修、客诉和质量历史 | 把过去的失效教训转成新项目预防要求 | 开短路、孔铜裂纹、CAF、分层、阻抗偏差、离子污染、ENIG镍腐蚀、翘曲、焊点失效 |
| 跨职能团队经验 | 汇集销售、项目、研发、工艺、质量、采购、制造和供应商经验 | 报价阶段DFM风险、关键材料交期、压合/电镀良率、外协特殊过程、客户审核关注点 |
| 客户图纸、规范和CSR | 明确必须满足的合同性要求和提交证据 | IPC等级、客户检验规范、特殊特性符号、PPAP等级、可靠性项目、数据上传系统 |
| 法规和行业标准 | 界定安全、环保、可靠性和合规底线 | IATF 16949、IPC-6012/6013/A-600/A-610、UL、RoHS/REACH、车规客户要求 |

## CSR与需求追溯矩阵

客户特殊要求（CSR）必须从项目启动开始结构化管理。不要把CSR只放在销售邮件或客户手册中；应转成一张可追溯矩阵，并逐条连接到设计输入、过程控制、验证计划和PPAP证据。

建议字段：

| 字段 | 说明 |
| --- | --- |
| 要求编号 | 客户手册、图纸、规范、合同、质量协议或系统条款编号 |
| 要求原文/摘要 | 保留可追溯表述，必要时附原文位置 |
| 要求类型 | 产品、过程、检验、包装、标签、追溯、PPAP、变更、供应商、年度验证 |
| 内部转化 | 转成内部规格、SOP、控制计划、测试项目、系统字段或批准流程 |
| 责任部门 | 工程、质量、生产、采购、物流、IT、供应商等 |
| 证据文件 | 图纸、DVP&R、PFMEA、Control Plan、MSA、SPC、PPAP、培训、审核记录 |
| 状态 | 未评审、适用、豁免、已满足、需客户澄清、需偏差批准 |

PCB/载板常见CSR风险点：

- PPAP提交等级、PSW签署、客户系统上传、年度再验证和特殊特性标识。
- 禁用材料、环保声明、UL/黄卡、RoHS/REACH、冲突矿产或客户绿色供应链要求。
- 可靠性项目、样本量、失效定义、实验室资质和第三方见证要求。
- 包装、标签、条码、批次追溯、先进先出、保存期限和湿敏/防氧化要求。
- 供应商变更、生产地点变更、外协特殊过程变更和再PPAP触发条件。

## 标准化转换路径

典型链路：

`VOC收集 -> VOC分类与权重 -> QFD/质量屋 -> 工程特性 -> 设计/可靠性/质量目标 -> 初始特殊特性 -> DFMEA/DVP&R -> 过程流程图/PFMEA/Control Plan/SOP/MSA/过程能力 -> Run at Rate/产品与过程验证 -> PPAP批准 -> 量产控制/SPC/顾客反馈/供应商质量/ECN/持续改善 -> 经验教训回流`

关键控制点：

| 阶段 | 核心问题 | 主要输出 |
| --- | --- | --- |
| VOC收集 | 客户真正担心什么，使用场景和失效后果是什么 | VOC清单、客户痛点、历史问题、法规/CSR清单 |
| 分类与权重 | 哪些需求影响安全、功能、可靠性、成本和交付 | VOC优先级、CTQ候选项、风险排序 |
| QFD展开 | 客户需求对应哪些工程特性 | 质量屋、工程特性权重、目标值和责任部门 |
| 指标定义 | 指标是否可测量、可验证、可追溯 | 设计目标、可靠性目标、质量目标、DVP&R/PV计划输入 |
| 风险传递 | 指标如何进入失效预防和现场控制 | 初始特殊特性、DFMEA、过程流程图、PFMEA、Control Plan、MSA/SPC计划 |
| 项目监控 | 项目是否按节点、成本、风险和质量水平推进 | Program Metrics、RYG状态、升级机制、行动清单 |

## APQP第一阶段：计划与项目确定

第一阶段的任务是把商业机会和客户输入转化为可执行的项目计划。它既要回答“客户要什么”，也要回答“公司有没有能力、用什么资源、在什么风险下承诺”。

### 第一阶段输入与输出

| 类别 | 内容 |
| --- | --- |
| 输入 | RFQ、客户图纸/规格/CSR、历史客诉、类似产品良率、材料/设备能力、法规标准、供应商状态、成本和交期目标 |
| 输出 | 项目章程、APQP主计划、VOC/QFD、设计/可靠性/质量目标、初始BOM/流程假设、初始特殊特性、初始风险清单、管理层可行性承诺 |
| 评审重点 | 是否有未澄清客户要求、不可实现公差、材料交期风险、设备能力缺口、检测能力缺口、可靠性验证周期不足 |

### 报价和可行性评审

PCB/载板项目在RFQ阶段就应进行可制造性、可测量性、可采购性和可交付性评审。低质量报价会把风险延后到试产或量产，最终表现为延期、良率不足、PPAP退回或客诉。

| 评审维度 | 检查问题 | 常见风险 |
| --- | --- | --- |
| 设计可制造性 | 线宽线距、孔径纵横比、叠构、阻抗、层偏、阻焊桥、表面处理是否匹配能力 | 报价接受了超出现有能力的设计 |
| 材料可得性 | CCL/PP/铜箔/油墨/药水/特殊材料是否有合格供应商和交期 | 材料认证周期长于项目周期 |
| 测量可行性 | 关键特性是否能测、测量系统是否有分辨力和MSA计划 | 规格很严但检测方法不可靠 |
| 可靠性可验证性 | CAF/HAST/热循环/可焊性等验证周期和样本量是否可满足节点 | PPAP前无法完成长期可靠性证据 |
| 产能与节拍 | 瓶颈设备、外协、检测、烘烤、表面处理和电测能力是否足够 | Run at Rate时产能达不到承诺 |
| 供应链与外协 | 外协过程、关键材料和供应商变更控制是否可满足客户要求 | 外协特殊过程没有被纳入APQP |

管理层可行性承诺不应只是签字。若存在能力缺口，应明确投资、外协、临时偏差、客户澄清或技术降阶方案，并在APQP风险清单中跟踪。

## QFD/质量屋用法

QFD的作用是把“顾客想要什么”映射成“工程上控制什么”。在PCB/载板项目中，QFD不要只作为一次性图表，应作为APQP第一阶段到FMEA和Control Plan的桥。

常用做法：

1. 在左侧列出VOC和客户CTQ，例如高速稳定、长期耐湿热、焊接可靠、外观无明显缺陷、装配不翘曲。
2. 在顶部列出工程特性，例如阻抗、公差、Dk/Df、线宽线距、孔铜厚、介质厚度、镀层厚度、离子污染、CAF距离、翘曲、表面处理。
3. 用关系矩阵评估VOC与工程特性的强弱关系，并结合客户权重、失效后果和过程能力计算优先级。
4. 将高权重工程特性转成目标值、验证项目、样本量、测量系统、控制频率和责任人。
5. 把高风险/高权重项纳入初始特殊特性清单，并传递到DFMEA/PFMEA、Control Plan和PPAP证据。

## 三类目标

| 目标类型 | 目的 | PCB/载板落地示例 |
| --- | --- | --- |
| 设计目标 | 定义产品结构、材料和性能边界 | 层数、叠构、板厚、线宽线距、孔径、介质厚度、阻抗、材料牌号、铜厚、表面处理、外形尺寸、成本目标 |
| 可靠性目标 | 证明产品在客户使用条件下不会过早失效 | 热循环、热冲击、HAST/PCT、CAF、焊锡耐热、剥离强度、弯折、跌落/振动、翘曲、离子污染、长期导通稳定性 |
| 质量目标 | 定义量产过程的稳定性和放行水平 | 一次合格率、报废率、PPM、客户投诉率、Cpk/Ppk、AOI/AVI漏检率、阻抗合格率、电测不良率、交付达成率 |

设计目标回答“产品要做到什么”，可靠性目标回答“使用中能否持续满足”，质量目标回答“量产过程能否稳定做到”。

## 初始特殊特性

初始特殊特性来自VOC、法规/安全、客户CSR、QFD高权重工程特性、历史失效和过程能力薄弱点。它应在APQP第一阶段形成候选清单，并在DFMEA/PFMEA、样件验证、试生产和量产准备中持续收敛。

识别准则：

- 安全、法规、环保或客户强制要求相关。
- 直接影响客户功能、装配、可靠性或最终产品失效后果。
- 历史客诉或保修中反复出现。
- 过程窗口窄、能力不足、检测困难或失控后不易返修。
- 对下游SMT、封装、整机装配或终端使用有放大效应。

PCB/载板常见候选项：

| 类别 | 候选特殊特性 |
| --- | --- |
| 电气性能 | 阻抗、开短路、绝缘电阻、耐压、导通电阻、插入损耗 |
| 互连可靠性 | 孔铜厚、微孔填孔、孔壁空洞、内层连接、背钻残桩、CAF间距 |
| 图形与尺寸 | 线宽线距、层偏、焊盘尺寸、外形尺寸、孔位、涨缩补偿 |
| 表面与焊接 | ENIG/ENEPIG镀层厚度、镍腐蚀、可焊性、离子污染、阻焊开窗和桥连 |
| 结构可靠性 | 板厚、翘曲、分层、介质厚度、热膨胀匹配、弯折寿命 |
| 装配相关 | PCBA焊点质量、BGA焊盘共面性、连接器区域尺寸、治具定位孔 |

输出要求：

- 在图纸、规格书、DFM评审、FMEA、Control Plan、作业指导书、检验规范和PPAP资料中保持同一套特殊特性口径。
- 对每个特殊特性定义测量方法、测量设备、MSA要求、抽样频率、控制图/SPC规则、反应计划和升级路径。
- 发生客户投诉、制程异常、工程变更或材料变更后，应回写FMEA和Control Plan。

## Program Metrics与RYG

APQP项目指标不能只跟踪进度，还要跟踪风险、成本、质量基线和闭环效率。可监控的Program Metrics包括：

| 指标维度 | 典型指标 |
| --- | --- |
| 节点与交付 | APQP关口达成率、样件交付、试产准备、PPAP提交、客户批准状态 |
| 成本与资源 | 开发费用偏差、模治具/设备投入、材料认证进度、供应商准备度 |
| 质量与能力 | 样件合格率、试产良率、关键特性Cpk/Ppk、MSA完成率、可靠性通过率 |
| 风险闭环 | FMEA高风险项关闭率、行动项逾期率、问题重复发生率、变更影响评估完成率 |
| 透明度 | CFT行动清单、系统节点数据、客户问题清单、管理层升级事项 |

RYG状态管理：

| 状态 | 含义 | 应对 |
| --- | --- | --- |
| Green | 按计划推进，风险可控 | 按既定节奏评审，保持证据和数据同步 |
| Yellow | 出现潜在风险或轻微偏差 | CFT启动原因分析、纠正措施和资源协调，明确恢复计划 |
| Red | 关键节点、质量、成本或客户批准出现重大偏差 | 管理层介入，升级决策，必要时调整项目范围、资源或客户沟通策略 |

成熟做法是让系统基于节点、行动项、质量数据和风险阈值自动触发RYG状态，而不是依赖项目成员主观填报。

## 阶段关口与准入准出

APQP应按阶段关口管理。关口评审不是会议纪要，而是基于证据决定能否进入下一阶段。

| 关口 | 进入下一阶段前必须确认 |
| --- | --- |
| Gate 1 计划批准 | VOC/CSR评审完成，项目范围、资源、主计划、初始特殊特性和主要风险已批准 |
| Gate 2 设计冻结/样件批准 | 设计记录、DFMEA、DVP&R、DFM/DFT、样件控制计划和设计评审问题已闭环或有批准计划 |
| Gate 3 过程准备批准 | 过程流程图、PFMEA、Pre-Launch控制计划、SOP、设备/治具/人员/材料准备和MSA计划已就绪 |
| Gate 4 PPAP提交批准 | Run at Rate、产品验证、包装标签验证、MSA、能力研究和问题闭环满足客户提交条件 |
| Gate 5 量产批准 | PPAP/PSW获得批准或临时批准，Production控制计划生效，量产KPI和反应计划明确 |

红线项应阻止放行：安全/法规不符合、客户特殊要求未评审、关键可靠性失败、关键测量系统不可信、特殊特性无控制计划、重大问题无遏制、未经客户批准的材料/地点/工艺变更、PPAP样件来源不可追溯。

## APQP主计划和证据管理

APQP主计划应把技术任务、质量任务、供应链任务和客户批准任务放在同一张时间轴上，而不是质量部门单独维护资料清单。

主计划建议包含：

- 客户节点：RFQ、设计冻结、样件、试产、PPAP提交、SOP/量产爬坡。
- 内部节点：DFM、DFMEA、DVP&R、PFMEA、Control Plan、MSA、SPC计划、Run at Rate、PPAP评审。
- 长周期任务：材料认证、可靠性试验、治具制作、设备验收、外协认证、客户系统权限。
- 决策节点：设计评审、工艺评审、供应商评审、试产评审、管理层升级、客户批准。
- 证据状态：未开始、进行中、待评审、已批准、需客户澄清、需偏差批准。

资料管理要做到“三一致”：文件版本一致、样件批次一致、数据来源一致。PPAP包中的设计记录、试产批次、检验报告、MSA、能力研究和PSW应能互相追溯。

| PPAP证据 | 设计记录、过程流程、PFMEA、Control Plan、MSA、初始过程能力、材料/性能测试报告 |

## APQP第二阶段：产品设计和开发

第二阶段承接第一阶段的VOC、QFD、设计目标、可靠性目标、质量目标和初始特殊特性，把“要达到什么”进一步转化为“怎么设计、怎么验证、怎么评审、怎么进入样件控制”。该阶段的核心不是画完图纸，而是在设计方案形成时同步完成质量前置、风险预防和验证策划，避免缺陷在试产或量产后才暴露。

在PCB/载板项目中，第二阶段通常覆盖叠构、材料、线路/孔/阻焊/表面处理设计、阻抗与信号完整性、DFM/DFT、可靠性验证、样件制造路径、样件检验计划和设计评审。若供应商只按客户图纸代工，仍应把客户设计输入转化为DFM风险、过程可行性、样件控制计划和PPAP证据；若供应商承担设计责任，则必须把DFMEA和DVP&R做实。

### 阶段目标

| 目标 | 管理含义 | PCB/载板落地 |
| --- | --- | --- |
| 设计质量前置 | 在设计阶段识别并降低失效风险，而不是靠终检筛出问题 | 叠构、材料、阻抗、孔结构、铜厚、表面处理、焊盘和外形尺寸在设计评审时完成风险闭环 |
| 可制造性确认 | 设计输出要匹配现有或计划中的工艺窗口和量产能力 | 线宽线距、孔径纵横比、层压涨缩、LDI/蚀刻、电镀、阻焊、成型和检测能力提前确认 |
| 可验证性确认 | 每个关键需求都能被测试、测量和判定 | DVP&R定义测试项目、条件、样本量、接收准则、责任人、计划日期和结果闭环 |
| 证据链建立 | 后续PPAP、客户审核和量产批准有可追溯证据 | DFMEA、设计记录、设计评审、样件控制计划、检验计划、可靠性报告和变更记录一致 |
| 数字化闭环 | CAD/CAE/EDA/PLM/MES/QMS数据能支持风险识别和问题回写 | EDA规则、仿真结果、DFM报告、测试数据、FMEA行动项和ECR/ECN在系统中关联 |

### 设计输入完整性检查

第二阶段开始前，应先确认设计输入是否完整。输入不完整时，DFMEA和DVP&R容易变成猜测，后续试产会反复返工。

| 输入类别 | 检查内容 | PCB/载板重点 |
| --- | --- | --- |
| 客户设计记录 | 图纸、Gerber/ODB++、IPC网表、叠构、BOM、材料、表面处理、阻抗表 | 版本、单位、公差、特殊特性、客户批准状态 |
| 功能和使用条件 | 电气、热、机械、装配、环境和寿命要求 | 高速网络、功率/热、回流曲线、车载环境、服务器长期工作 |
| 法规和标准 | IPC、UL、环保、客户CSR、行业可靠性要求 | IPC等级、UL材料、RoHS/REACH、客户禁用物质 |
| 质量历史 | 类似产品失效、客诉、内部报废、供应商异常 | CAF、孔铜裂纹、分层、阻抗偏差、ENIG异常、翘曲 |
| 可制造性约束 | 线宽线距、孔径、纵横比、层数、铜厚、拼板、外形、检测能力 | HDI/mSAP/SAP、背钻、填孔、电镀均匀性、AOI/AVI能力 |
| 供应链约束 | 材料牌号、替代料、外协过程、特殊设备和长周期物料 | CCL/PP/铜箔/ABF/BT、特殊表面处理、第三方实验室 |

若客户只提供加工文件而没有清晰工程目标，应通过DFM问题清单向客户澄清。不能把工程假设直接写入内部文件并默认客户接受。

### DFM、DFT与可测量性评审

PCB/载板设计评审至少覆盖DFM（可制造性）、DFT（可测试性）、DFA（装配适配）和MSA可行性。评审结论应形成问题清单、责任人、关闭日期和客户确认记录。

| 评审项 | 典型问题 | 输出 |
| --- | --- | --- |
| 叠构和材料 | Dk/Df、Tg、CTE、树脂含量、铜厚、压合对称性、翘曲风险 | 叠构批准、材料清单、替代料规则 |
| 图形和阻抗 | 线宽线距、蚀刻补偿、阻抗coupon、关键网络、损耗目标 | 阻抗计算、coupon设计、仿真/测试计划 |
| 孔结构 | 机械孔、激光孔、叠孔/错孔、背钻、纵横比、孔铜目标 | 钻孔能力评审、切片计划、可靠性验证 |
| 表面处理 | ENIG/ENEPIG/OSP/沉锡/沉银选择与应用环境匹配 | 镀层规格、XRF方法、可焊性和金相验证 |
| 检测与测试 | 电测覆盖、AOI/AVI能力、X-ray/切片、尺寸测量和MSA | 测试治具、测量方法、MSA计划 |
| 包装和物流 | 防潮、防氧化、板间隔离、翘曲保护、标签追溯 | 包装规范、标签模板、运输验证计划 |

### 核心方法与工具

| 方法/工具 | 第二阶段作用 | PCB/载板应用要点 |
| --- | --- | --- |
| 项目管理、行动项、检查表 | 驱动设计节点、评审和问题关闭 | 用关口清单确认客户输入、特殊特性、DFM、仿真、样件计划、材料认证和验证计划是否完成 |
| DFMEA | 识别设计失效模式、后果、原因和预防措施 | 关注阻抗偏差、信号损耗、热失效、CAF、孔铜疲劳、分层、翘曲、表面处理失效、焊接不良和装配干涉 |
| 过程流程图 | 提前梳理样件到预量产的制造路径 | 明确样件是否使用量产设备、量产材料和量产工艺；避免工程样件数据误代表量产能力 |
| 样件控制计划 | 将关键特性、检查项目和反应计划前移到样件阶段 | 对阻抗coupon、微切片、孔铜厚、镀层厚度、尺寸、外观、翘曲、电测和可靠性样件定义检验频率 |
| 检验计划 | 把控制计划转为具体测量、功能、材料和可靠性试验 | 明确IPC/客户规范、测试方法、测量设备、样本量、判定准则和数据记录格式 |
| CAD/CAE/PLM/EDA | 支撑数字化建模、仿真、版本和生命周期管理 | 用EDA/仿真做阻抗、SI/PI、热、翘曲和DFM检查；PLM/QMS承接设计评审、变更和质量反馈 |

数字孪生、闭环质量和智能制造平台可以提升协同效率，但不应被当作APQP的替代品。无论工具多先进，最终仍要回到同一条证据链：客户需求 -> 设计输入 -> DFMEA -> 设计验证 -> 样件控制计划 -> 问题闭环 -> PPAP/量产准备。

### 实施路径

1. 数字化建模与仿真：对关键网络、叠构、材料、热、翘曲、装配间隙和可制造性进行建模与规则检查。PCB项目可重点做阻抗/损耗仿真、SI/PI分析、热分析、DFM/DRC、拼板和coupon设计。
2. 风险评估与可靠性分析：用DFMEA把历史客诉、类似产品失效、仿真异常和客户CTQ转为设计风险；对高风险项定义设计更改、试验验证或过程控制措施。
3. 设计与工艺协同：研发、工艺、质量、制造、采购和供应商共同评审设计输出，确认材料可得性、工艺窗口、检测能力、外协能力、良率风险和成本影响。
4. 原型/样件制作策划：从样件阶段开始定义控制计划和检验计划，避免样件只验证功能、不验证可制造性和关键质量特性。
5. 设计评审与变更闭环：用评审记录、行动项、状态报告和ECR/ECN管理决策点；所有设计更改要回写DFMEA、DVP&R、控制计划和BOM/图纸。

### 第二阶段关键交付物

| 交付物 | 用途 | 检查重点 |
| --- | --- | --- |
| 设计记录/图纸/规格 | 固化产品结构、材料、性能和特殊特性 | 版本、客户批准、特殊特性标识、关键公差和标准引用是否清楚 |
| DFMEA | 证明设计风险已系统识别并降低 | 高严重度/高风险项是否有预防措施、验证计划和关闭证据 |
| DVP&R | 规划并记录设计验证活动 | 测试项目、条件、样本量、接收准则、计划日期、责任人、结果和问题闭环是否完整 |
| 样件/原型 | 验证功能、材料、制造可行性和可靠性 | 是否使用代表性材料、设备、工艺和检验方法；样件差异是否被记录 |
| 样件过程流程图 | 描述从来料到样件交付的制造路径 | 是否覆盖外协、特殊过程、检测节点、返工和追溯 |
| 样件控制计划 | 管控样件阶段关键质量特性 | 特殊特性、检测方法、频率、反应计划和升级规则是否明确 |
| 检验计划 | 把样件控制要求落到测量和测试 | 测量设备、测试方法、MSA需求、数据记录和判定准则是否明确 |
| 包装规范和检查清单 | 保护样件和量产产品在运输/储存中不失效 | 湿敏、氧化、翘曲、ESD、防潮、干燥剂、HIC、标签和追溯是否覆盖 |
| 设计评审和状态报告 | 记录决策、风险、行动项和关口状态 | 未关闭问题是否影响下一阶段，是否触发客户沟通或管理层升级 |

### DVP&R编写要点

DVP&R是第二阶段的核心工作文件之一。它既是测试计划，也是动态报告；既要证明设计目标达成，也要为后续设计评审、客户批准和PPAP准备证据。

建议字段：

| 字段 | 说明 |
| --- | --- |
| 需求来源 | VOC、QFD、客户图纸/CSR、法规标准、DFMEA、历史问题 |
| 工程目标 | 可测量的规格、公差、性能或可靠性目标 |
| 测试项目 | 尺寸、功能、材料、环境、机械、可靠性、电气或安全测试 |
| 测试条件/方法 | 测试标准、设备、夹具、环境、应力条件、循环次数和前处理 |
| 样本量与样本状态 | 数量、批次、版本、材料、工艺、是否代表量产条件 |
| 接收准则 | 通过/失败判定、性能下限、安全红线和失效定义 |
| 责任与计划 | 责任部门/负责人、开始日期、完成日期、资源和外部实验室 |
| 结果与问题 | 实测数据、偏差、失效模式、根因、行动项和关闭证据 |

编排逻辑：

- 先做零部件、材料和关键结构验证，再做整机/整板功能和破坏性或环境可靠性验证。
- 明确安全红线和客户不可接受失效，不能只写“功能正常”“外观OK”。
- 对项目周期敏感或长期可靠性要求高的项目，评估是否需要加速测试，但必须说明加速模型和适用边界。
- DVP&R结果要进入设计评审；未通过项目必须触发DFMEA、设计变更、控制计划或检验计划更新。

DVP&R与DFMEA应双向联动：DFMEA中高严重度、高风险或探测困难的失效模式必须能在DVP&R或过程控制中找到验证/预防措施；DVP&R失败项必须回写DFMEA的失效原因、现行控制和行动项。若测试失败只写“重新试验通过”，没有说明根因和设计/过程更新，不能算闭环。

### PCB/载板DVP&R示例

| 客户需求/风险 | 工程目标 | 验证项目 | 接收准则示例 |
| --- | --- | --- | --- |
| 高速信号稳定 | 阻抗、插入损耗、回损、串扰满足客户限制 | EDA仿真、coupon设计、TDR/VNA测试、关键网络评审 | 实测在客户规格内，异常网络有设计或工艺纠正措施 |
| 孔互连长期可靠 | 孔铜、微孔、叠孔和内层连接可靠 | 微切片、热应力、热循环、导通电阻监控 | 无裂纹、空洞和分层；导通变化符合客户/IPC要求 |
| 湿热和电化学迁移风险 | 绝缘系统和间距能承受环境应力 | CAF/HAST/PCT、绝缘电阻、离子污染 | 无CAF/迁移失效，绝缘电阻和污染度满足规格 |
| 焊接和表面处理可靠 | 表面处理厚度、可焊性和焊点可靠 | XRF厚度、可焊性、回流模拟、金相分析 | 镀层厚度、润湿和焊点形貌满足客户/IPC要求 |
| 装配翘曲风险 | 板/载板在回流和封装条件下翘曲可控 | 常温/高温翘曲、回流后翘曲、装配试验 | 翘曲低于客户限制，BGA/封装区域风险关闭 |
| 外观和尺寸适配 | 外形、孔位、焊盘、阻焊开窗满足装配 | 尺寸测量、AOI/AVI、首件和装配试验 | 关键尺寸在公差内，装配干涉和焊接风险关闭 |

### 设计冻结与变更控制

设计冻结不是不能改，而是任何更改必须受控。进入第三阶段前，应明确设计冻结基线：图纸版本、叠构版本、材料版本、Gerber/ODB++版本、特殊特性清单、DVP&R版本和客户批准状态。

冻结后常见变更及影响评估：

| 变更 | 需要评估 |
| --- | --- |
| 材料牌号或供应商 | 阻抗、热性能、可靠性、UL/环保、压合窗口、客户批准 |
| 线宽线距/阻抗补偿 | 阻抗、插入损耗、蚀刻窗口、AOI能力、过程能力 |
| 孔结构/背钻/填孔 | 互连可靠性、切片、X-ray、钻孔/电镀能力 |
| 表面处理 | 可焊性、接触可靠性、储存寿命、XRF程序、客户认可 |
| 拼板/治具定位 | 产能、翘曲、成型、测试治具、包装和客户装配 |

任何影响特殊特性、可靠性、客户批准状态或PPAP证据来源的设计变更，应触发DFMEA、DVP&R、PFMEA、Control Plan和PPAP影响评估。

### 行业适配判断

PCB/载板制造具有“离散订单 + 连续化学制程”的混合特征。项目管理、设计评审、样件交付和PPAP更像离散制造；电镀、蚀刻、压合、表面处理、清洗和烘烤又高度依赖过程参数、设备状态和化学体系。因此第二阶段不能只强调CAD/EDA，也要把工艺能力和过程参数纳入设计评审。

落地要点：

- 离散侧：以项目节点、BOM/图纸版本、样件批次、设计评审、ECR/ECN和客户批准为主线。
- 制程侧：以电镀厚度、蚀刻补偿、压合涨缩、材料批次、药水状态、洁净度和检测能力为主线。
- 数据侧：把仿真、DFM、FMEA、样件测试、客诉和量产SPC数据串起来，形成设计规则和失效库的持续更新。

## APQP第三阶段：过程设计开发

第三阶段连接产品设计开发和产品/过程验证。它的核心任务是把第二阶段的设计输出，转化为可以在工厂稳定执行的制造系统。换句话说，第二阶段回答“产品要做成什么样”，第三阶段回答“用什么过程、设备、人员、参数、测量和反应机制保证它被稳定做出来”。

第三阶段容易出问题，是因为图纸、仿真和设计评审无法充分暴露现场问题：设备能力是否足够、工装夹具是否合理、操作员是否理解关键控制点、测量系统是否可信、参数窗口是否稳定、异常反应是否可执行。这些问题必须在进入试生产和Run at Rate前尽量前置解决。

### 输入与输出

| 类别 | 内容 | PCB/载板关注点 |
| --- | --- | --- |
| 输入 | DFMEA、特殊特性清单、工程图纸、技术规格、材料规范、样件制造经验和DVP&R结果 | 叠构、材料、阻抗、孔结构、铜厚、表面处理、焊盘、尺寸、可靠性目标、客户CSR |
| 输出 | 过程流程图、PFMEA、控制计划、SOP/作业指导书、过程参数表、MSA计划/报告、初始过程能力计划/研究、PFMEA评审记录 | 从来料到包装的完整流程、关键工序风险、检验点、SPC项目、反应计划、设备/治具/人员准备 |
| 下游接口 | 第四阶段的试生产、Run at Rate、PPAP和量产爬坡 | 用量产材料、量产设备、量产人员和量产节拍验证过程能力 |

### 过程流程图

过程流程图是第三阶段的骨架。它要把产品从来料到出货的每一步梳理清楚，使跨职能团队对制造路径、检验点、返工路径和外协过程有一致理解。

每个工序节点至少要说明：

| 字段 | 说明 | PCB/载板示例 |
| --- | --- | --- |
| 输入 | 来料或上工序输出 | CCL、铜箔、PP、油墨、药水、上工序板件 |
| 过程参数 | 影响质量的设定值和窗口 | 曝光能量、蚀刻速度、压合温度/压力/时间、电镀电流密度、烘烤条件 |
| 关键设备/工装 | 设备、夹具、治具、程序和软件版本 | LDI、钻机、压机、电镀线、AOI/AVI、X-ray、电测治具、阻抗测试coupon |
| 检验点 | 在哪里检查、检什么、用什么判定 | AOI、微切片、孔铜厚、阻抗、镀层厚度、离子污染、尺寸、外观 |
| 输出 | 交给下工序的状态和记录 | 合格板、返工板、报废板、检验记录、批次追溯 |

PCB刚性多层板的过程流程图可以从 `04-process-quality.md` 的基本流程展开：开料 -> 内层图形 -> 内层蚀刻 -> AOI -> 棕化/黑化 -> 叠板压合 -> 钻孔 -> 除胶渣/沉铜 -> 电镀 -> 外层图形 -> 外层蚀刻 -> 阻焊 -> 表面处理 -> 成型 -> 电测 -> 终检 -> 包装。具体项目应补入背钻、控深钻、埋阻埋容、HDI激光钻孔、填孔电镀、mSAP/SAP、软硬结合、载板build-up等特殊流程。

### PFMEA

FMEA的DFMEA/PFMEA边界、实施逻辑、评分优先级和PCB/载板失效模式细节详见 `31-fmea.md`。本节保留其在APQP第二、三阶段中的位置和接口关系。

PFMEA是第三阶段的核心文件。它分析制造/装配过程可能如何导致产品失效，并把高风险项转化为预防、探测和控制措施。

DFMEA和PFMEA的边界：

| 维度 | DFMEA | PFMEA |
| --- | --- | --- |
| 分析对象 | 产品设计本身 | 制造和装配过程 |
| 典型问题 | 叠构、材料、孔结构、焊盘设计是否会导致失效 | 电镀、蚀刻、压合、钻孔、阻焊、表面处理和检测过程是否会导致失效 |
| 牵头部门 | 设计/研发，质量和工艺参与 | 工艺/质量，生产、设备、设计和供应商参与 |
| 输出接口 | DVP&R、设计变更、特殊特性 | 控制计划、SOP、MSA、SPC、反应计划 |

PFMEA常用字段包括过程编号、过程名称、潜在失效模式、失效影响、严重度、失效原因、频度、现行预防/探测控制、探测度、风险评价、推荐措施、责任人和完成日期。文章使用RPN思路；实际汽车项目需按客户CSR和所采用FMEA手册执行。若客户要求AIAG & VDA FMEA，应以Action Priority（AP）为主；若仍使用RPN，也不要只看乘积分数，严重度高的失效必须优先处理。

PCB/载板PFMEA示例：

| 工序 | 潜在失效模式 | 潜在影响 | 过程原因 | 典型控制 |
| --- | --- | --- | --- | --- |
| 内层图形/蚀刻 | 线宽偏小、短路、开路 | 阻抗偏差、电气失效 | 曝光偏差、蚀刻补偿不足、药水状态异常 | LDI参数确认、AOI、线宽测量、蚀刻速率监控 |
| 压合 | 层偏、分层、空洞、板厚异常 | 互连失效、装配不良、可靠性下降 | 涨缩补偿不准、压合曲线不当、材料批次差异 | 层偏coupon、压合曲线、板厚测量、剥离/切片 |
| 钻孔 | 孔偏、毛刺、孔壁粗糙、背钻残桩异常 | 孔可靠性下降、信号完整性问题 | 钻针磨损、叠板偏移、程序错误、定位异常 | 首件确认、钻针寿命、X-ray/切片、背钻检查 |
| 沉铜/电镀 | 孔铜不足、空洞、填孔凹陷 | 热循环开路、导通不稳 | 除胶渣不足、活化异常、电流密度不均、药水失控 | 孔铜厚、微切片、电镀参数SPC、药水分析 |
| 阻焊 | 偏位、桥连、附着不良 | 焊接缺陷、绝缘风险、外观不良 | 对位误差、曝光/显影窗口异常、油墨批次差异 | AVI、开窗尺寸、附着力、阻焊厚度 |
| 表面处理 | 镀层厚度异常、镍腐蚀、氧化 | 可焊性差、接触失效、线键合风险 | 药水老化、前处理不足、时间/温度异常 | XRF厚度、可焊性、外观、金相分析 |
| 电测/终检 | 漏检、误判、追溯断点 | 客诉、批量流出 | 治具磨损、程序错误、判定标准不一致 | 治具点检、黄金样、复判规则、数据追溯 |

### 控制计划

Control Plan的字段设计、控制项选择、反应计划和PCB/载板现场落地详见 `29-control-plan.md`。本节保留其在APQP第三到第五阶段中的位置和接口逻辑。

第三阶段的控制计划要把PFMEA识别的高风险项目，转化为现场可执行的控制点。它不是检验项目清单，而是“特性、规格、方法、频率、责任、反应计划”的系统化文件。

控制计划通常经历三个层级：

| 层级 | 用途 | 重点 |
| --- | --- | --- |
| Prototype Control Plan | 样件阶段 | 快速暴露设计/工艺风险，确认关键特性和测量方法 |
| Pre-Launch Control Plan | 试生产阶段 | 用接近量产条件验证工序、节拍、检验频率和反应计划 |
| Production Control Plan | 量产阶段 | 固化量产控制要求，并与PFMEA、SOP、SPC和检验记录一致 |

控制计划必须和PFMEA联动：PFMEA中的高严重度、高发生频度、探测弱项和特殊特性，应能在控制计划中找到对应的控制方法、抽样频率、测量系统和异常反应。若现场控制计划找不到PFMEA高风险项，说明APQP证据链断裂。

控制计划字段建议：

| 字段 | 说明 |
| --- | --- |
| 工序编号/名称 | 与过程流程图和PFMEA一致 |
| 产品/过程特性 | 明确产品特性、过程参数和特殊特性符号 |
| 规格/公差 | 来自图纸、客户规范、内部标准或工艺窗口 |
| 控制方法 | 设备参数、首件、巡检、SPC、自动检测、防错、实验室测试 |
| 测量系统 | 设备编号、治具、软件版本、MSA状态和校准状态 |
| 样本量/频率 | 首件、每批、每小时、每panel、每lot或客户指定频率 |
| 记录方式 | MES/QMS表单、设备数据、检验记录、实验室报告 |
| 反应计划 | 异常停线、隔离范围、复测、升级、客户通知、纠正措施 |

反应计划必须具体到可执行动作。只写“通知主管”“调整参数”是不够的，应说明隔离哪一批、谁批准放行、是否加严检验、是否触发8D或客户沟通。

### 关键过程参数、特殊过程与防错

PCB/载板制程中很多质量特性无法靠终检完全筛出，必须把关键过程参数纳入APQP。第三阶段应建立关键过程参数清单，并把参数来源、窗口、监控频率和异常反应写入SOP或控制计划。

| 工序 | 关键参数示例 | 防错/控制思路 |
| --- | --- | --- |
| 开料/内层 | 材料批次、铜厚、方向、裁切尺寸、内层补偿 | 条码防错、材料扫描、首件确认 |
| LDI/曝光/显影 | 能量、真空、对位、显影速度、药水状态 | 程序权限、首件线宽、AOI反馈 |
| 蚀刻 | 蚀刻速率、喷压、温度、铜离子、补偿值 | 线宽SPC、药水分析、自动补偿 |
| 压合 | lay-up顺序、升温速率、压力、真空、时间、冷却 | 叠构扫描、首件切片、压合曲线锁定 |
| 钻孔/激光 | 程序、钻咀寿命、转速进给、能量、定位 | 程序版本锁定、寿命管控、X-ray/切片 |
| 沉铜/电镀 | 除胶渣、活化、铜厚、电流密度、搅拌、药水分析 | 药水SPC、孔铜切片、设备报警 |
| 阻焊/文字 | 油墨批次、曝光、显影、固化、对位 | 限度样、首件、AVI、参数权限 |
| 表面处理 | 前处理、时间、温度、浓度、负载量、镀层厚度 | XRF、药水分析、可焊性、金相 |
| 电测/终检 | 程序版本、治具状态、探针寿命、判定标准 | 黄金样、治具点检、权限和数据追溯 |

特殊过程是指输出不易或不能通过后续监视测量完全验证的过程。PCB/载板中常见特殊过程包括压合、电镀、化学表面处理、清洗/烘烤、部分可靠性前处理和外协处理。对特殊过程应关注人员资格、设备维护、参数锁定、药水/环境监控、首件确认、周期验证和异常追溯。

### 设备、工装、人员和材料准备

第三阶段应在试生产前完成资源准备，而不是Run at Rate当天发现缺口。

| 准备项 | 检查重点 |
| --- | --- |
| 设备 | 能力、精度、维护、备件、程序权限、数据采集、产能和停机反应 |
| 工装/治具 | 电测治具、阻抗coupon、拼板夹具、切片夹具、包装治具、寿命和点检 |
| 人员 | 关键岗位培训、检验员认证、外观限度样、班次覆盖和替补机制 |
| 材料 | 供应商批准、批次追溯、储存条件、有效期、替代料规则和来料检验 |
| 环境 | 温湿度、洁净度、照明、防静电、化学品管理和实验室条件 |
| IT/MES/QMS | 料号、BOM、流程路线、检验计划、条码、数据上传和权限 |

这些准备项应连接到Run at Rate计划。若试生产使用的设备、人员、治具或材料与量产计划不一致，必须在报告中说明差异及风险。

### SOP/作业指导书

SOP是把过程设计落到操作层的文件。好的SOP不是技术报告，而是现场人员能理解、能执行、能培训、能追溯的作业标准。

SOP至少应包含：

- 目的、适用产品/工序、版本和生效日期。
- 材料、设备、工装、治具和程序版本。
- 关键过程参数及允许范围。
- 分步操作方法，优先使用图片、限度样、首件样和异常示例。
- 质量控制点、检验方法、记录要求和反应计划。
- 安全/环保注意事项、常见异常和升级路径。
- 关联文件：过程流程图、PFMEA、Control Plan、检验规范、设备点检表、ECR/ECN。

常见问题是SOP写得过长、脱离设备实际、参数与控制计划不一致，或工艺变更后没有同步更新。汽车PCB项目中，任何会影响特殊特性、过程窗口、检测方法或材料状态的变更，都应通过ECR/ECN并同步更新PFMEA、Control Plan和SOP。

### MSA与过程能力

第三阶段要确认关键测量系统是否可信，并为第四阶段试生产能力验证做好准备。文章中给出GR&R和Cpk的常用门槛，但实际项目应以客户CSR和质量协议为准。

MSA关注“测量是否可信”：

| 项目 | 说明 | PCB/载板示例 |
| --- | --- | --- |
| 重复性 | 同一人员、同一设备、同一零件重复测量的一致性 | 孔径、线宽、镀层厚度、板厚、翘曲 |
| 再现性 | 不同人员或不同条件下测量的一致性 | AOI/AVI判定、外观等级、尺寸复测 |
| 属性一致性 | 对OK/NG类判断的一致性 | 外观缺陷、阻焊桥连、镀层外观、X-ray缺陷 |
| 稳定性/偏倚/线性 | 测量系统随时间、量程和标准件的误差 | XRF、阻抗仪、离子污染测试、微切片测量 |

过程能力关注“过程是否稳定且有能力”：

- 先用控制图或其他稳定性判断确认过程受控，再评价Cp/Cpk；过程失控时，Cp/Cpk数值没有决策意义。
- `Cp`看分布宽度是否足够窄，`Cpk`同时考虑分布中心是否偏移。
- 客户常见要求是关键特性初始能力不低于 `1.33`，但不同客户、特性和阶段可能不同，必须查CSR。
- PCB可优先对阻抗、线宽线距、孔铜厚、镀层厚度、关键尺寸、翘曲、离子污染等CTQ/CTP项目做能力研究。

若Cpk不足：

| 情况 | 优先动作 |
| --- | --- |
| 过程不稳定 | 先找特殊原因，使过程受控，再谈能力 |
| 过程稳定但中心偏移 | 调整参数中心或补偿值，例如蚀刻补偿、电镀电流密度、压合涨缩补偿 |
| 过程稳定但波动过大 | 改善设备、材料批次、治具、环境、人员方法或过程窗口 |
| 测量系统不可信 | 先改进MSA，否则能力分析无效 |

### 供应商APQP和外协过程

关键材料和外协过程应纳入项目APQP，而不是等来料异常后再处理。对高风险供应商，建议要求其提交供应商APQP或等效证据。

| 对象 | 关注点 | 典型证据 |
| --- | --- | --- |
| CCL/PP/铜箔/ABF/BT | 材料规格、批次稳定性、认证状态、环保/UL、变更控制 | 材料认证、CoC/CoA、可靠性数据、变更通知协议 |
| 油墨/干膜/药水 | 化学窗口、保存期限、批次差异、兼容性 | 来料检验、试用报告、药水分析、供应商8D |
| 钻咀/铣刀/治具 | 寿命、尺寸稳定性、批次一致性 | 寿命验证、首件记录、供应商能力数据 |
| 表面处理/外协特殊过程 | 工艺窗口、检测方法、追溯、客户批准 | 外协流程、控制计划、MSA/能力、审核记录 |
| 第三方实验室 | 方法适用性、资质、样品处理、报告格式 | 实验室资质、方法确认、报告模板、样品追溯 |

供应商APQP至少要确认：要求传递、特殊特性、过程流程、PFMEA/控制计划、MSA、能力、变更控制、追溯和异常响应。客户禁止未经批准的二级供应商或外协过程时，应在采购和工艺路线中做防错。

### 第三阶段评审与进入第四阶段条件

第三阶段结束前，需要跨职能评审过程流程图、PFMEA、控制计划、SOP、MSA、过程能力准备和特殊特性传递。

评审人员通常包括项目工程、工艺工程、质量工程、生产主管、设备工程、采购/供应商质量，必要时包括客户代表。

进入第四阶段前至少确认：

| 检查项 | 要点 |
| --- | --- |
| 过程流程图 | 覆盖所有工序、外协、返工、检验和追溯路径，并与现场一致 |
| PFMEA | 高风险项有措施、责任人、完成日期和验证证据；高严重度项不被RPN低分掩盖 |
| 特殊特性映射 | 产品特殊特性已映射到过程特性、控制方法和记录 |
| 控制计划 | Pre-Launch版已批准，控制点、频率、设备、反应计划可执行 |
| SOP | 与控制计划和过程参数一致，操作人员已培训 |
| MSA | 关键测量系统已验证或有客户认可的验证计划 |
| 过程能力 | 初始能力研究计划和试生产数据采集方案明确；已有数据时按客户要求评价 |
| 第四阶段准备 | Run at Rate、试生产批量、量产材料/设备/人员/节拍、PPAP证据清单明确 |

## APQP第四阶段：产品与过程验证

第四阶段是前三阶段成果在真实量产条件下的系统性验证。它不是“再做几件样品”，而是用正式材料、正式设备、正式工装、正式人员、正式SOP和目标节拍，连续制造代表性数量的产品，验证过程能否稳定地产出符合客户要求的产品。

第四阶段的底层逻辑是：用数据说话，以事实批准，靠体系保障。任何未闭环的重大异常，都不应被带入量产。

### 第四阶段验证对象

| 验证对象 | 需要回答的问题 | PCB/载板关注点 |
| --- | --- | --- |
| 试生产/Run at Rate | 过程能否按目标节拍、目标良率和真实资源连续运行 | 瓶颈工序、WIP、返工率、一次合格率、电测不良、交期节拍和设备稼动 |
| 测量系统 | 量产环境下的测量数据是否仍可信 | 阻抗、孔铜厚、线宽、镀层厚度、翘曲、AOI/AVI、X-ray、微切片、电测 |
| 初始过程能力 | 关键特性是否稳定并具备能力 | Cpk/Ppk、SPC控制图、特殊原因、中心偏移、批间/板内均匀性 |
| 产品验证 | 试生产产品是否满足图纸、功能、性能和可靠性要求 | 尺寸、外观、电测、阻抗、热应力/热循环、CAF、HAST/PCT、离子污染、可焊性 |
| 包装和标签 | 产品能否在运输、储存和客户收货环节保持可追溯、可防护 | MBB/HIC、干燥剂、防潮、防氧化、ESD、板间隔离、批次/LOT标签、客户条码 |
| PPAP | 是否有足够证据证明量产过程可持续满足要求 | PSW、过程流程图、PFMEA、Control Plan、MSA、尺寸/材料/性能、能力研究、样品 |

### Run at Rate/试生产

Run at Rate是第四阶段最核心的验证活动。文章提到试生产数量可按客户要求连续生产数百到上千件；在PCB/载板场景中，应按客户CSR和项目特点换算成合适的单位，例如件数、panel数、批次数、面积、lot数或关键材料批次。

试生产必须尽量贴近量产：

| 要素 | 要求 | PCB/载板落地 |
| --- | --- | --- |
| 材料 | 使用量产批准材料和供应商 | CCL/PP/铜箔/油墨/药水/表面处理材料批次可追溯 |
| 设备 | 使用计划量产设备和程序 | LDI、钻机、压机、电镀线、AOI/AVI、电测、XRF、阻抗测试设备 |
| 工装/治具 | 使用量产夹具、测试治具和程序 | 电测治具、阻抗coupon、钻孔程序、拼板、载具、包装治具 |
| 人员 | 使用已培训的量产班组 | 关键岗位培训、上岗资格、检验员一致性 |
| 节拍 | 按目标节拍或客户约定节拍运行 | 瓶颈工序能力、换线时间、批量排产、烘烤/表面处理等待时间 |
| 文件 | 使用已批准的SOP、Control Plan和检验规范 | 现场文件版本与PFMEA/Control Plan一致 |

试生产过程应持续记录：

- 每小时或每批产量、一次合格率、报废率、返工率和瓶颈工序。
- 关键特性SPC数据和异常点。
- 设备停机、参数漂移、药水/耗材状态、工装磨损。
- 不良模式、发生工序、责任归属、临时措施和永久措施。
- 是否需要更新PFMEA、Control Plan、SOP、检验规范或培训材料。

Run at Rate报告建议字段：

| 字段 | 内容 |
| --- | --- |
| 运行条件 | 日期、班次、产线、设备、人员、材料批次、工装治具、文件版本 |
| 运行目标 | 计划数量、目标节拍、目标FPY、目标报废率、客户特殊要求 |
| 实际结果 | 投入、产出、合格、返工、报废、停机、瓶颈工序和达成率 |
| 质量数据 | 特殊特性、SPC、Cpk/Ppk、外观、电测、阻抗、可靠性抽样结果 |
| 异常清单 | 问题描述、影响范围、临时遏制、根因、责任人和关闭日期 |
| 结论 | 批准量产、条件批准、需重复Run at Rate或需客户偏差批准 |

若Run at Rate未达标，不应只降低节拍或删除异常批次后重算结果。应明确未达标原因是产能、良率、测量系统、材料、设备、人员、文件还是供应商问题，并更新第三阶段文件。

### MSA与过程能力补充验证

第四阶段的MSA不是重复做表，而是确认第三阶段选定的测量系统，在真实生产人员、环境、节拍和样品状态下仍然有效。若量产班组、检验员、设备、夹具、环境或测量方法发生变化，关键测量系统应重新评估。

过程能力研究应以稳定过程为前提：

- 先判稳，再计算Cpk/Ppk；过程不稳定时，能力指数没有决策意义。
- 对客户特殊特性和CTQ/CTP优先做能力研究，例如阻抗、线宽线距、孔铜厚、镀层厚度、关键尺寸、翘曲、离子污染。
- 如果Cpk/Ppk不达标，应区分过程偏移、波动过大、设备能力不足、工装磨损、原材料波动或测量系统问题。
- 客户常见门槛可能是 `Cpk >= 1.33`，但必须以客户CSR、图纸、质量协议和PPAP要求为准。

### 产品验证

试生产产品应作为产品验证和PPAP证据的主要来源。PCB/载板产品可按以下维度建立验证清单：

| 验证类别 | PCB/载板项目 | 证据 |
| --- | --- | --- |
| 尺寸/结构 | 外形、孔位、板厚、介质厚、焊盘、阻焊开窗、翘曲 | 尺寸报告、CMM/影像测量、翘曲记录 |
| 电气 | 开短路、电阻、绝缘、阻抗、插入损耗 | 电测记录、TDR/VNA报告、网络清单 |
| 金相/互连 | 孔铜厚、面铜厚、微孔填孔、层偏、背钻残桩 | 微切片、X-ray、孔铜统计 |
| 表面处理 | ENIG/ENEPIG/OSP/沉银/沉锡厚度、可焊性、镍腐蚀风险 | XRF、可焊性、外观/金相报告 |
| 可靠性 | 热应力、热循环、CAF、HAST/PCT、剥离强度、离子污染 | 可靠性测试报告、失效分析记录 |
| 外观 | IPC/客户外观标准、限度样、AQL结果 | AVI/终检记录、Master Sample、缺陷样本库 |
| 包装 | 防潮、防氧化、ESD、板间隔离、标签和追溯 | 包装验证、跌落/运输模拟、标签核对记录 |

外观和标签是PPAP退回的高频原因。应在试生产阶段固定外观Master Sample、检验照明/距离/角度、包装规格、客户标签格式、LOT追溯规则和条码扫描验证。

### 包装、标签和追溯验证

包装标签验证应作为第四阶段正式验证项，而不是出货前临时确认。PCB/载板容易出现氧化、吸湿、翘曲、划伤、混料和标签错误，包装方案必须能支撑客户仓储、运输、SMT/封装和追溯要求。

| 项目 | 检查点 |
| --- | --- |
| 防护 | 防潮、防氧化、防静电、防压伤、防刮伤、防翘曲、板间隔离 |
| 包材 | 真空袋、MBB、干燥剂、湿度卡、隔纸、托盘、泡棉、外箱和周转箱 |
| 标签 | 客户料号、供应商代码、批次、数量、生产日期、有效期、RoHS/UL、条码规则 |
| 追溯 | 原材料批次、生产lot、panel/strip、检验记录、返工记录、出货批次 |
| 验证 | 扫码验证、跌落/振动/运输模拟、开包后状态、储存寿命、客户收货测试 |

对载板、精细线路板、金面产品和长期库存产品，应特别关注防潮、防硫化、防氧化、颗粒污染、板面摩擦和洁净包装。包装变更可能影响客户批准状态，应纳入ECN和PPAP影响评估。

### PPAP准备

PPAP的提交等级、PSW、一致性评审、再PPAP触发和PCB/载板客户提交细节详见 `30-ppap.md`。本节保留其在APQP第四阶段中的证据链位置。

PPAP是第四阶段的关键输出，用来向客户证明量产过程能持续满足要求。提交等级和具体内容必须按客户CSR、采购文件和质量协议执行。文章给出的等级和清单可作为学习线索，但不能替代AIAG PPAP手册或客户要求。

常见PPAP提交等级理解：

| 等级 | 常见含义 | 管理要点 |
| --- | --- | --- |
| Level 1 | 只向客户提交PSW，其他资料内部保存 | 仍要完成内部完整证据包，不能只做PSW |
| Level 2 | PSW加样品和部分支持资料 | 确认客户指定资料清单和样品来源 |
| Level 3 | PSW、样品和完整支持资料 | 最常见，文件一致性和数据追溯要求高 |
| Level 4 | 客户指定的特殊提交要求 | 按客户系统、表单、见证或特殊项目执行 |
| Level 5 | 资料在供应商现场备查，客户现场审核 | 现场文件、样品、记录、工装和生产线必须一致 |

常见PPAP证据包括：

| 证据 | PCB/载板落地说明 |
| --- | --- |
| 设计记录/图纸/规格 | 客户图纸、Gerber/ODB++、叠构、材料、表面处理、特殊特性 |
| 工程变更和批准 | ECR/ECN、客户偏差许可、工程批准记录 |
| DFMEA | 供应商承担设计责任时尤其关键 |
| 过程流程图 | 覆盖全流程、外协、返工、检验和追溯 |
| PFMEA | 与流程图和控制计划一致，试生产新发现已回写 |
| Control Plan | Pre-Launch/Production版本，特殊特性、反应计划和记录明确 |
| MSA | 关键测量系统和属性检验一致性证据 |
| 尺寸结果 | 关键尺寸、孔位、板厚、翘曲、客户指定项目 |
| 材料/性能/可靠性结果 | 材料认证、表面处理、阻抗、可靠性、可焊性、环保合规 |
| 初始过程能力 | 客户指定关键特性Cpk/Ppk和SPC证据 |
| 合格实验室文件 | 内外部实验室资质和方法适用性 |
| 样品、Master Sample、检查辅具 | 试生产样件、限度样、电测治具/量具确认 |
| 客户特殊要求 | CSR、系统上传、特殊包装、特殊标签、年度验证要求 |
| PSW | Part Submission Warrant，零件提交保证书，通常是PPAP批准的核心签署文件 |

更完整的PPAP内部准备清单可按以下逻辑组织：

| 类别 | 文件/证据 |
| --- | --- |
| 设计与变更 | 设计记录、授权工程变更、客户工程批准、特殊特性清单 |
| 风险与控制 | DFMEA、过程流程图、PFMEA、Control Plan、SOP/检验规范 |
| 测量与能力 | MSA、尺寸结果、材料/性能结果、初始过程能力、SPC图 |
| 实验室与样品 | 合格实验室文件、样品、Master Sample、检查辅具/治具确认 |
| 合规与客户要求 | 外观批准、包装标签、环保/材料声明、CSR矩阵、PSW |

PSW签署前应做一次“证据一致性评审”：PSW料号/版本、设计记录、试生产批次、样品、尺寸报告、材料报告、能力研究、MSA和控制计划必须指向同一批准状态。任何临时偏差、未关闭问题或条件批准，都应在客户允许的格式中明确。

PPAP批准状态常见有完全批准、临时批准和拒收。临时批准应明确有效期、数量、适用批次、未关闭问题和关闭计划；不能把临时批准当成永久量产批准。

PPAP被退回的常见原因：

- 尺寸、材料、性能或可靠性结果不符合规格。
- Cpk/Ppk不达标，或过程未判稳就提交能力指数。
- MSA缺失或关键量具GR&R/属性一致性不满足要求。
- 过程流程图、PFMEA和控制计划不一致。
- 标签、包装、外观Master Sample或客户系统资料不一致。
- 文件缺项、版本错误、试生产样件来源不符合量产条件。

### 试生产问题闭环

第四阶段发现问题并不是失败，未闭环进入量产才是失败。所有问题应按严重度分级管理：

| 问题等级 | 示例 | 处理原则 |
| --- | --- | --- |
| 严重问题 | 安全/法规、基本功能、批量开短路、可靠性严重失效 | 停止试生产，根因分析，纠正后重新验证 |
| 重要问题 | 关键性能、客户CTQ、Cpk不达标、标签/包装重大错误 | 暂停或限制推进，完成纠正和验证后再提交PPAP |
| 一般问题 | 外观轻微异常、记录格式问题、局部培训不足 | 记录并闭环，必要时更新SOP/培训/检验规范 |

闭环路径：

`问题发现 -> 隔离/遏制 -> 根因分析 -> 纠正措施 -> 有效性验证 -> 更新PFMEA/Control Plan/SOP/MSA/SPC -> 客户沟通或PPAP资料更新`

### 第四阶段进入与退出条件

启动第四阶段前，应确认第三阶段基础已具备：

| 进入条件 | 检查要点 |
| --- | --- |
| PFMEA已评审 | 高风险项有措施，责任和完成日期明确 |
| Pre-Launch控制计划已批准 | 试生产控制频率、测量系统、反应计划清楚 |
| SOP已培训 | 量产班组、检验员、设备人员已培训并有记录 |
| MSA已完成或有计划 | 关键测量系统可支持试生产判定 |
| 设备/工装/材料就绪 | 量产设备、治具、程序、材料、供应商已确认 |
| 试生产计划明确 | 数量、节拍、人员、数据采集、停线规则和评审会明确 |

退出第四阶段并进入量产/第五阶段前，应确认：

| 退出条件 | 检查要点 |
| --- | --- |
| Run at Rate完成 | 产能、节拍、良率和设备状态满足目标或客户批准 |
| 产品验证通过 | 尺寸、外观、功能、材料、性能、可靠性、包装标签满足要求 |
| 过程能力可接受 | 关键特性稳定，Cpk/Ppk符合客户要求或有客户批准的计划 |
| 文件已更新 | PFMEA、Control Plan、SOP、检验规范、包装规范反映试生产发现 |
| 问题已闭环 | 严重/重要问题已完成原因分析和有效性验证 |
| PPAP批准 | PSW和客户要求的提交资料获得批准或临时批准 |

## APQP第五阶段：量产、反馈、评定与纠正

第五阶段不是PPAP批准后的行政收尾，而是APQP真正进入长期运行的阶段。汽车、服务器、通信、工控和载板项目进入量产后，产品生命周期可能持续多年；此时APQP的核心问题从“能否批准量产”转为“如何长期稳定生产，并在问题暴露后快速闭环、持续改进”。

### 第五阶段的双目标

| 目标 | 含义 | PCB/载板重点 |
| --- | --- | --- |
| 持续控制 | 维持已验证的过程状态，确保量产线按批准的控制计划、SOP、检验规范和反应计划运行 | 阻抗、孔铜、线宽线距、镀层厚度、板厚、翘曲、离子污染、电测良率等关键特性持续受控 |
| 持续改善 | 在稳定量产基础上识别趋势、缺陷、客户反馈和供应商波动，推动根因消除 | 降低报废/返工/客诉/PPM，提升FPY、Cpk/Ppk、交付稳定性和客户满意度 |

### 早期量产遏制与爬坡管理

PPAP批准后，建议在量产初期设置GP-12或类似早期生产遏制机制，直到过程稳定、客户反馈正常、关键KPI达到目标后再退出。

早期遏制可包括：

- 对特殊特性、历史高风险工序和客户关注缺陷加严检验。
- 对电测、阻抗、外观、镀层厚度、翘曲、包装标签等高风险项目设置额外复核。
- 每日评审FPY、报废、返工、SPC异常、设备停机、客户反馈和供应商异常。
- 对新增不良模式即时更新PFMEA、Control Plan、SOP、限度样和培训。
- 明确退出标准，例如连续若干批/若干周无重大异常、KPI达标、客户无投诉。

量产爬坡要关注产能和质量的同步成熟。产量上升时，班次、人员熟练度、设备负荷、药水负载、外协排程和检测能力都会改变，不能用小批试产表现直接推断高负荷量产稳定性。

### 跨职能职责

| 职能 | 第五阶段责任 |
| --- | --- |
| 质量 | 监督控制计划执行、SPC判异、客诉/8D、内审、KPI和体系闭环 |
| 生产 | 按SOP和控制计划生产，及时反馈设备、材料、人员、节拍和质量异常 |
| 工艺/技术 | 分析制程问题，调整工艺窗口，更新PFMEA、Control Plan、SOP和检验规范 |
| 采购/供应链 | 管理CCL、PP、铜箔、药水、外协特殊过程等供应商质量和变更 |
| 客服/销售/项目 | 接收并传递顾客反馈、抱怨、退货、现场失效和工程变更需求 |
| 研发/设计 | 对设计相关的量产失效、可靠性问题和客户使用问题进行设计回写 |

### 控制计划与SPC持续执行

控制计划在第五阶段是现场运行标准，不是PPAP包里的静态文件。量产现场应按控制计划规定的特性、方法、频率、样本量、记录方式和反应计划执行，并保留可追溯记录。发生下列情况时，应触发控制计划评审或更新：

- 过程参数、设备、工装、材料、供应商、生产地点或外协过程发生变化。
- 新失效模式、客户投诉、内部重大异常或审核问题暴露。
- 客户CSR、图纸、规范、检验标准或年度验证要求更新。
- PFMEA、SOP、检验规范、包装规范或MSA/SPC计划发生变化。

SPC不只是画控制图，更要判异和触发反应。常见失控信号包括点超出控制限、连续上升或下降、连续点落在中心线同侧、交替波动、多个点接近控制限等。PCB/载板场景可优先纳入电镀厚度、阻抗、线宽线距、板厚、介质厚度、镀层厚度、孔径/孔位、翘曲、离子污染等关键特性。

SPC异常的基本闭环：

`异常信号 -> 隔离可疑批次 -> 通知QE/工艺/生产 -> 根因分析 -> 临时遏制 -> 永久纠正 -> 效果验证 -> 更新PFMEA/Control Plan/SOP/培训`

### 分层过程审核与年度验证

第五阶段应通过分层过程审核（LPA）、过程审核、产品审核和年度再验证确认控制计划持续有效。审核不应只查表单存在，而要查现场是否按批准状态运行。

| 活动 | 关注点 | PCB/载板示例 |
| --- | --- | --- |
| 分层过程审核 | 关键岗位是否按SOP、控制计划、点检和反应计划执行 | 线宽首件、药水分析、钻咀寿命、电测治具点检、标签扫描 |
| 过程审核 | 过程流程、PFMEA、控制计划、设备、人员、材料和记录一致性 | 电镀线、压合、阻焊、表面处理、终检包装 |
| 产品审核 | 成品是否持续满足客户图纸、外观、尺寸、功能和包装要求 | 随机抽样复测阻抗、镀层、外观、标签和追溯 |
| 年度验证 | 客户或内部要求的周期性可靠性、材料、能力和MSA复核 | 热循环、CAF/HAST、可焊性、离子污染、Cpk、GRR |

年度验证计划应来自CSR、控制计划、历史风险和客户投诉。若年度验证失败，应按影响范围评估库存、在制品、已出货批次和是否需要客户通知。

### 顾客反馈、8D与问题闭环

8D的D1-D8、遏制、根因分析、永久措施、验证和防再发细节详见 `32-8d.md`。本节保留其在APQP第五阶段中的闭环位置。

第五阶段的顾客反馈包括正式索赔、现场失效、退货、客户关注事项、满意度调查和工程变更需求。响应时限必须以客户CSR、质量协议和项目合同为准；文章中的24小时、48小时、5天、10天等时限可作为学习参考，不能替代客户要求。

8D闭环应覆盖：

| 8D步骤 | 要点 |
| --- | --- |
| D1团队 | 组建质量、生产、工艺、供应商、设计和项目人员 |
| D2问题描述 | 用5W2H明确产品、批次、数量、地点、时间、现象和影响范围 |
| D3遏制措施 | 隔离库存、在制品、在途品和客户现场风险，必要时加严检验 |
| D4根因分析 | 区分发生原因和流出原因，使用5Why、鱼骨图、数据分层和失效分析 |
| D5永久措施 | 针对根因定义工艺、设备、材料、检验、培训或设计纠正 |
| D6实施验证 | 通过批量数据、SPC、能力、复测或客户确认验证有效性 |
| D7预防再发 | 更新PFMEA、Control Plan、SOP、检验规范、培训和供应商要求 |
| D8总结 | 固化经验教训，纳入类似产品、类似工序和下一项目APQP |

PCB/载板常见量产反馈可包括客户装配开短路、阻抗偏差、焊接不良、表面处理异常、翘曲、CAF、分层、孔铜可靠性、离子污染、外观判定争议和标签/追溯问题。每一类问题都应明确是否需要客户沟通、偏差批准、再验证或PPAP重新提交。

### 供应商质量管理

供应商质量是第五阶段稳定性的关键输入。对CCL、PP、铜箔、油墨、干膜、药水、表面处理外协、钻咀/铣刀、测试治具等，应结合风险等级执行来料检验、供应商过程数据审核、周期审核和异常闭环。

供应商异常的基本路径：

`来料/制程/客户端发现异常 -> 隔离可疑物料和相关批次 -> 通知采购与供应商质量 -> 要求供应商8D或等效报告 -> 验证纠正措施 -> 必要时现场审核、加严检验、降级或替代供应`

对高风险材料和特殊过程，不应只依赖供应商出厂报告。需要关注供应商批次稳定性、过程能力、变更控制、追溯、检验方法、客户特殊要求传递和异常响应速度。

### 过程能力与KPI

初始过程能力不是PPAP时一次性完成的数字。关键特性进入量产后仍需按风险和客户要求持续监控Cpk/Ppk趋势：安全/关键特性可按月或更高频率复核，普通关键特性可按季度复核，成熟稳定产品可按内部规则延长频率。若能力接近下限、连续下降或低于客户要求，应启动改善、遏制或再验证。

第五阶段KPI应同时看过程、客户和供应商：

| 维度 | 参考指标 | PCB/载板落地 |
| --- | --- | --- |
| 过程质量 | FPY、返工率、报废率、关键特性Cpk/Ppk、SPC异常趋势 | 不只看最终出货良率，还要看电镀、压合、钻孔、图形、阻焊、表面处理和电测等分段数据 |
| 顾客质量 | 客诉数、退货率、客户PPM、8D准时关闭率、重复问题率 | 区分批量流出、装配端失效、现场可靠性失效和文件/标签问题 |
| 供应商质量 | 来料合格率、供应商PPM、供应商8D准时率、重复异常率 | 对关键材料和外协过程建立分级管理、加严/恢复规则和年度评价 |

文章中列举的FPY、返工率、报废率、Cpk、PPM和8D准时率目标可作为参考值；实际目标必须按客户CSR、产品风险、内部经营目标和项目协议确定。

### ECN与量产变更

第五阶段任何影响产品、过程或客户批准状态的变更都要纳入ECN/ECR管理。典型变更包括设计记录、材料牌号、叠构、供应商、生产地点、设备/工装、工艺参数、检验方法、包装标签、外协过程和软件/程序版本。

推荐控制路径：

`变更申请 -> 影响评估（质量/能力/MSA/可靠性/客户批准） -> 更新图纸/PFMEA/Control Plan/SOP/检验规范 -> 小批验证 -> 必要时提交PPAP或客户批准 -> 正式实施 -> 量产跟踪`

对汽车PCB和载板项目，未经评估的材料替代、工艺窗口调整、外协切换和生产地点变化，往往比单次制程异常更危险，因为它们会破坏PPAP时已批准的过程假设。

### 再APQP与再PPAP触发

量产阶段不是只有新项目才需要APQP。下列情况通常应触发再APQP、再验证或PPAP重新提交评估，具体以客户CSR为准：

| 触发项 | 需要评估 |
| --- | --- |
| 设计记录变化 | 图纸、叠构、材料、阻抗、公差、表面处理、特殊特性 |
| 生产地点变化 | 工厂、产线、外协、关键设备搬迁或新增产线 |
| 材料/供应商变化 | CCL、PP、铜箔、ABF/BT、油墨、药水、表面处理外协 |
| 过程变化 | 压合曲线、电镀参数、蚀刻补偿、钻孔/激光程序、清洗/烘烤 |
| 检测变化 | 量具、测试程序、AOI/AVI算法、实验室、抽样频率或判定标准 |
| 长期停产后复产 | 停产时间超过客户或内部规则，设备/材料/人员状态变化 |
| 重大质量问题 | 客诉、可靠性失效、批量报废、能力长期不足、重复8D |

再提交不一定等于完整Level 3 PPAP，但必须有影响评估、验证计划和客户批准路径。内部应保留为什么需要或不需要再PPAP的判断证据。

### 经验教训回流

第五阶段的最终价值是把量产问题回流到体系和下一项目：

| 问题来源 | 回流方向 |
| --- | --- |
| 设计相关失效 | 更新VOC、设计输入、DFMEA、DVP&R和设计规则 |
| 过程相关失效 | 更新过程流程图、PFMEA、Control Plan、SOP、MSA/SPC和培训 |
| 验证不足 | 更新第四阶段验证计划、Run at Rate条件、样本量和判定准则 |
| 供应商问题 | 更新供应商准入、审核、质量协议、来料检验和变更控制 |
| 客户反馈和8D | 更新类似产品、类似工序、检查表、限度样和经验教训库 |

建议在项目生命周期关键节点、年度质量评审或产品平台复盘时，系统总结APQP前期遗漏、有效的预防措施、重复发生问题、客户审核问题和下一项目必须提前纳入的控制点。

## APQP资料包与审核追溯

面向客户审核、IATF审核或内部成熟度评审，APQP资料包应能从任意一个客户要求向下追溯到设计、过程、验证、量产和反馈。

推荐追溯链：

`CSR/VOC -> 工程指标 -> 特殊特性 -> DFMEA/DVP&R -> 过程流程图 -> PFMEA -> Control Plan -> SOP/检验规范 -> MSA/SPC/能力 -> Run at Rate/PPAP -> 量产记录/8D/ECN`

审核时常用抽样方式：

| 抽样入口 | 应能追溯到 |
| --- | --- |
| 一个客户特殊特性 | 图纸标识、FMEA、控制计划、测量系统、SPC、反应计划和PPAP证据 |
| 一条客诉/8D | 遏制、根因、纠正、有效性、PFMEA/Control Plan/SOP更新和经验教训 |
| 一个工艺变更 | ECR/ECN、影响评估、验证、客户批准、文件更新和量产跟踪 |
| 一个供应商异常 | 来料批次、使用批次、隔离范围、供应商8D、验证和供应商评价 |
| 一个Run at Rate批次 | 材料、设备、人员、参数、检验、能力、异常和PPAP样品来源 |

成熟的APQP资料包不是文件越多越好，而是每份文件能解释一个真实风险、一个控制逻辑或一个批准证据。

## 最小可执行模板包

实际落地时，可把APQP资料拆成一套最小可执行模板。不同客户和项目可裁剪，但每个模板都应有版本、责任人、批准人、关联文件和状态。

| 模板/清单 | 主要字段 | 适用阶段 |
| --- | --- | --- |
| 项目章程 | 客户、产品、范围、目标、里程碑、团队、风险、资源 | 第一阶段 |
| CSR矩阵 | 要求编号、要求内容、内部转化、责任部门、证据、状态 | 第一阶段到第五阶段 |
| APQP主计划 | 任务、责任人、计划日期、实际日期、状态、风险、升级 | 全阶段 |
| 特殊特性清单 | 来源、特性、规格、测量方法、控制方法、反应计划 | 第一阶段到第三阶段 |
| DFM/DFT清单 | 设计问题、风险、建议、客户回复、关闭证据 | 第一阶段到第二阶段 |
| DFMEA行动清单 | 失效模式、高风险项、措施、责任、日期、验证 | 第二阶段 |
| DVP&R | 需求、测试项目、条件、样本量、准则、结果、问题闭环 | 第二阶段到第四阶段 |
| 过程流程图 | 工序编号、输入、过程、输出、检验、返工、外协 | 第三阶段 |
| PFMEA行动清单 | 过程失效、原因、控制、行动优先级、措施、验证 | 第三阶段到第五阶段 |
| Control Plan | 特性、规格、方法、频率、测量系统、记录、反应计划 | 第三阶段到第五阶段 |
| SOP/检验规范 | 参数、步骤、限度样、记录、异常处理、关联文件 | 第三阶段到第五阶段 |
| MSA计划/报告 | 特性、设备、方法、样本、评价人、结果、结论 | 第三阶段到第四阶段 |
| SPC/能力计划 | 特性、控制图、抽样、判异规则、Cpk/Ppk、反应计划 | 第三阶段到第五阶段 |
| Run at Rate报告 | 运行条件、目标、实际结果、质量数据、异常和结论 | 第四阶段 |
| PPAP清单/PSW | 提交等级、证据、版本、样品来源、批准状态 | 第四阶段 |
| 早期量产遏制计划 | 加严项目、退出标准、每日评审、问题升级 | 第五阶段初期 |
| ECN影响评估 | 变更内容、影响范围、验证、客户批准、文件更新 | 第五阶段 |
| 经验教训库 | 问题、根因、措施、适用产品/工序、标准化动作 | 第五阶段和下一项目 |

模板之间应靠编号和版本关联。例如特殊特性编号应同时出现在DFMEA/PFMEA、控制计划、MSA、SPC和PPAP资料中；试生产批次编号应能连接材料批次、设备、人员、检验记录和样品。

## APQP五阶段串联逻辑

| 阶段 | 核心问题 | 主要证据 |
| --- | --- | --- |
| 第一阶段：策划/VOC | 客户真正需要什么，哪些需求是CTQ | VOC、CSR矩阵、QFD、目标、初始特殊特性、APQP主计划、Program Metrics |
| 第二阶段：产品设计开发 | 设计如何满足需求，如何验证设计 | 设计记录、DFM/DFT、DFMEA、DVP&R、样件控制计划、设计评审 |
| 第三阶段：过程设计开发 | 制造过程如何稳定做出合格产品 | 过程流程图、PFMEA、Control Plan、SOP、MSA、SPC/能力计划、供应商APQP、评审记录 |
| 第四阶段：产品与过程验证 | 真实量产条件下产品和过程是否可批准 | Run at Rate、MSA/Cpk、产品验证、包装标签验证、PPAP、PSW、问题闭环 |
| 第五阶段：量产、反馈、评定与纠正 | 如何长期稳定生产并持续改善 | 控制计划执行、SPC、早期遏制、LPA/年度验证、顾客反馈/8D、供应商质量、ECN、KPI、经验教训 |

关键原则：三张文件必须一致。过程流程图定义“有哪些工序”，PFMEA定义“这些工序怎么失效、风险多大”，控制计划定义“现场怎么控制和反应”。如果三者编号、工序、特殊特性或控制方法不一致，APQP体系很容易在审核或试生产中暴露问题。

第五阶段的关键原则是问题必须回流。客诉、SPC异常、供应商异常、ECN和内审发现如果只做临时处理，而没有更新PFMEA、Control Plan、SOP、检验规范、培训和类似项目检查表，APQP闭环就没有真正完成。

## 不同项目类型的APQP增强项

同样叫APQP，不同项目的关注点并不相同。PCB/载板项目至少应按产品和客户场景做增强。

| 项目类型 | 额外关注 | 典型证据 | 常见失误 |
| --- | --- | --- | --- |
| 消费电子HDI/SLP | 节奏快、版本变更多、外观和装配适配敏感 | 样件控制计划、快速变更闭环、装配验证、外观限度样 | 只追交期，不同步更新Control Plan和版本追溯 |
| 汽车PCB | CSR、特殊特性、PPAP、追溯、年度验证和供应商APQP | CSR矩阵、PPAP清单、能力研究、LPA、变更批准记录 | 把汽车项目按普通工业板导入，漏掉客户系统要求 |
| 服务器/AI高速板 | Dk/Df批次控制、阻抗/损耗、热和翘曲、长期稳定性 | DVP&R、TDR/VNA、材料批次管理、回流后翘曲和可靠性数据 | 只看静态图纸指标，不跟踪高速和热场景下的实测表现 |
| IC载板/ABF项目 | 洁净度、微缺陷、warpage、strip级追溯和封装接口 | build-up流程、微孔/细线路验证、封装客户端判定、strip map | 只按PCB思路做过程验证，忽略封装匹配和客户来料标准 |
| 材料替代或二供导入 | 等效性、变更批准、重新验证范围和客户接受条件 | ECN影响评估、材料认证、小批验证、必要时再PPAP | 只确认规格书相近，没有验证良率、可靠性和客户批准状态 |

项目越复杂，越不能把APQP理解成“交一套模板”。真正有价值的是：要求、风险、测量、验证、量产控制和变更批准始终使用同一条证据链。

## 常见错误

- VOC停留在访谈纪要，没有转成规格、验证条件和责任部门。
- APQP裁剪没有风险依据，只因为项目急就删掉MSA、能力、可靠性或客户批准步骤。
- CSR没有形成矩阵，客户包装、标签、变更、年度验证和系统上传要求遗漏。
- QFD做完后没有进入FMEA、Control Plan、DVP&R和PPAP证据链。
- 特殊特性识别过晚，导致样件、试产和量产控制口径不一致。
- 项目章程和RACI不清，PFMEA、控制计划、MSA和PPAP资料无人最终负责。
- 只跟踪项目时间表，不跟踪质量成熟度、风险闭环和能力数据。
- RYG状态由人工主观判断，缺少自动阈值、数据证据和升级规则。
- 使用工程样品或非量产线数据代表量产能力，导致PPAP和实际量产脱节。
- 质量目标只写PPM或良率，不定义测量系统、抽样频率、过程能力和反应计划。
- 把“图纸完成”当作第二阶段完成，忽略DFMEA、DVP&R、样件控制计划和设计评审闭环。
- 仿真、DFM和样件测试各自独立，没有把异常回写到设计规则、FMEA和控制计划。
- DVP&R只有测试名称，没有测试条件、样本量、接收准则、计划节点、责任人和失败后的反应路径。
- 数字孪生、PLM或QMS系统只作为文件库使用，没有形成需求、风险、验证、问题和变更的闭环。
- 第三阶段把过程流程图、PFMEA和控制计划分开编，导致工序编号、特殊特性和控制方法对不上。
- PFMEA由单个工程师闭门完成，没有生产、设备、质量、供应商和现场人员参与。
- 只按RPN排序，忽略严重度高、法规/安全或客户特殊特性相关的风险。
- SOP与现场实际设备、工装、参数不一致，操作员培训没有记录。
- 外协特殊过程和关键材料供应商没有纳入APQP，客户要求没有传递到二级供应商。
- MSA和过程能力研究放到PPAP前补做，发现测量系统或过程能力问题时已经来不及改过程。
- 把Run at Rate当作小批样件生产，没有使用量产材料、量产设备、量产人员和目标节拍。
- 试生产只看最终合格率，不记录小时产出、瓶颈工序、设备停机、返工率和SPC异常。
- 试生产发现问题后只修现场，不回写PFMEA、Control Plan、SOP和培训材料。
- 包装、标签、条码和追溯没有在第四阶段验证，出货前才发现客户格式不匹配。
- PPAP资料按模板凑齐，但样件来源、版本、试生产条件和数据证据不能互相追溯。
- Cpk不达标或关键问题未闭环时，为赶交期强行提交PPAP。
- PPAP批准后把控制计划束之高阁，量产现场仍按旧习惯生产和检验。
- 早期量产没有加严控制和退出标准，爬坡期间新增问题未被及时识别。
- SPC只记录数据，不判异、不隔离可疑批次、不触发反应计划。
- 客诉只做临时遏制，没有用8D更新PFMEA、Control Plan、SOP和供应商要求。
- 材料、供应商、设备、外协或工艺参数变化没有走ECN、客户批准或PPAP重新提交评估。
- 变更影响评估只看产品规格，不评估MSA、过程能力、可靠性、包装标签和客户批准状态。
- 年度验证、分层过程审核和产品审核流于形式，没有用来确认控制计划持续有效。
- KPI只看出货良率，不看FPY、返工、报废、客诉、供应商PPM和Cpk趋势。
- 量产经验教训没有回流到类似产品、类似工序和下一项目APQP策划。

## 与六大质量工具的接口

| 工具 | APQP各阶段输出如何传递 |
| --- | --- |
| DFMEA | 将客户功能、使用场景、可靠性目标、设计方案和初始特殊特性转成设计失效模式、后果和预防措施 |
| PFMEA | 将关键产品特性、历史问题、样件制造路径、过程流程图和过程敏感点转成过程失效模式、原因、控制和行动优先级 |
| Control Plan | 第一阶段形成特殊特性，第二阶段形成样件控制计划，第三阶段形成预生产/量产控制计划框架，第四阶段通过Run at Rate验证并更新，第五阶段按量产反馈和变更持续执行、评审和修订 |
| MSA | 对关键测量系统做量具R&R、偏倚、线性、稳定性或属性一致性确认；第四阶段确认量产环境下仍可信 |
| SPC | 对关键过程参数和产品特性建立控制图、能力评价和异常反应；第四阶段用试生产数据验证过程稳定性和能力，第五阶段持续判异、隔离、分析和改善 |
| PPAP | 用设计记录、设计评审、DVP&R、过程流程、FMEA、控制计划、SOP、MSA、Run at Rate、性能测试和能力数据证明量产准备度；第五阶段的重大变更或客户要求可能触发PPAP更新/重新提交 |

## 使用建议

做APQP资料、汽车PCB项目导入、客户审核或PPAP准备时，先用本页建立“VOC/QFD -> 工程指标 -> DFMEA/DVP&R -> 过程流程图/PFMEA/Control Plan/SOP/MSA/过程能力 -> Run at Rate/产品与过程验证 -> PPAP/PSW -> 量产控制/SPC/8D/ECN/持续改善 -> 经验教训回流”的链路，再结合 `04-process-quality.md` 中的IATF 16949与AIAG六大质量工具、IPC检验规范和PCB可靠性测试内容补齐标准和现场控制细节。
